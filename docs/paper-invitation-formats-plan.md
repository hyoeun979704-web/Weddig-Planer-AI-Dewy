# 종이 청첩장 포맷 택소노미 & 제작 모델 계획 (PAPER-FORMATS)

> 기준: **비즈하우스(BizHows)**. 종이 청첩장은 규격·구조·후가공이 포맷마다 다르다(엽서·포토카드·
> 2단/3단 접이식·초콜릿 포장지·티켓·바람개비·여권 등). 현재 엔진은 일부만 지원 → 포맷을 1급
> 개념으로 모델링한다. **모바일(네이티브 섹션)과 별 트랙** — 종이는 캔버스(Konva)+인쇄 규격 유지.

## 1. 현황 (있는 것 / 없는 것)

**있음** (`src/lib/invitation/types.ts:184-212`, `layout.ts`, `exportPdf.ts`):
- `product_kind`: `card | ticket | chocolate | bifold | newspaper | mobile | mobile_roll`
- 다중 페이지 `pages[]`, 페이지별 `print{wMm,hMm,bleedMm,safeMarginMm}`, 300dpi 비율, 페이지 단위 PDF.

**없음 (이 계획의 대상)**:
- 포맷 미정의: **trifold(3단)·pinwheel(바람개비)·passport(여권)·postcard(엽서)·photocard(포토카드)**.
- **구조 모델**: 접는 선/패널 경계, 다이컷/형압 윤곽(비사각), 제본(booklet), 임포지션(한 장에 패널 배치).
- **후가공 사양**: 박·형압·오시·도무송(다이컷)·미싱(절취선)·용지/평량/코팅.
- 포맷별 운영자 오서링 가이드(접는선·세이프·다이컷 윤곽 오버레이).

## 2. 포맷 택소노미 (구조 분류 — 비즈하우스 기준)

| 포맷 | product_kind | 구조 | 대표 규격(mm, 완성) | 핵심 제작 요소 |
|---|---|---|---|---|
| 엽서카드 | `postcard` | 단면 1패널 | 100×148 / 102×152 | 양면 인쇄, 모서리 라운드(선택) |
| 포토카드 | `photocard` | 단면 1패널 | 54×86(카드) / 102×152 | 사진 중심, 라운드·코팅·유광 |
| 2단 접이식 | `bifold` | 1장 1접(2패널) | 완성 128×182 → 펼침 256×182 | **오시 1선**, 표지/내지 |
| 3단 접이식 | `trifold` | 1장 2접(3패널) | 완성 100×210 → 펼침 300×210 | **오시 2선**(레터/Z폴드) |
| 초콜릿 포장지 | `chocolate` | **다이컷 전개도** + 접이 플랩 | 제품 치수 기반 net | 도무송, 접이 플랩, 풀칠선 |
| 티켓 | `ticket` | 1패널 + **절취 스텁** | 60×180 등 | 미싱(절취선), 일련번호(선택) |
| 바람개비 | `pinwheel` | **다이컷 정사각** + 접이 조립 | 150×150 등 | 도무송(대각 절개), 조립 가이드, 핀 |
| 여권 | `passport` | **제본 booklet**(표지+내지 N장) | 88×125 | 중철/실제본, 임포지션, 페이지 순서 |

> newspaper(신문형)은 기존 enum 유지. 위 포맷은 enum 확장 대상.

## 3. 데이터 모델 확장

```ts
// product_kind 확장
product_kind?:
  | "card" | "postcard" | "photocard"
  | "bifold" | "trifold"
  | "ticket" | "pinwheel" | "passport"
  | "chocolate" | "newspaper"
  | "mobile" | "mobile_roll";

// 인쇄/구조 스펙 확장 (InvitationPrintSpec 또는 page.print 에)
interface InvitationPrintSpec {
  wMm: number; hMm: number; bleedMm?: number; safeMarginMm?: number;
  // 접이식 — 접는 선(오시) 위치. 한 장(펼침)에 패널이 나뉨.
  folds?: { axis: "v" | "h"; atMm: number; type?: "score" | "perforation" }[];
  // 비사각 다이컷(바람개비·초콜릿·라운드 등) 윤곽. 없으면 사각.
  shape?: "rect" | "rounded" | "die_cut";
  dieCutPathSvg?: string;        // thru-cut 윤곽(인쇄소 도무송 라인)
  // 제본(여권) — booklet 임포지션·페이지 순서.
  binding?: "none" | "saddle_stitch" | "sewn";
}

// 후가공(공급 사양) — 포맷 무관 공통. 제휴 업체 capability 반영(아래 §8).
interface PrintFinishing {
  paper?: string;        // 용지(스노우/랑데뷰 등)
  weightGsm?: number;    // 평량
  coating?: "none" | "matte" | "gloss";  // ✅ 제휴 업체 가능
  laserCut?: boolean;    // ✅ 레이저 컷팅 — 다이컷/라운드/절취/접는 스코어 라인
  foil?: boolean;        // ❓ 박 — 미확인(파트너 확인 필요)
  emboss?: boolean;      // ❓ 형압 — 미확인(프레스 장비 필요)
}
```

- **패널 모델**: 접이식은 `pages[]` 대신(또는 병행) **한 펼침면 + folds** 로 표현해 임포지션·접는선 정합을 보장.
  (현재 `pages[]` 는 "여러 장"이라 "한 장 접힘"과 의미가 다름 — 구분 필요.)
- **다이컷/제본**은 캔버스 좌표 위에 윤곽/페이지 순서 메타로 분리(렌더는 사각, 인쇄는 다이컷 라인 적용).

## 4. 인쇄·내보내기 확장 (`exportPdf.ts`)

- **재단/접는/다이컷 마크**: crop marks + **fold lines(오시)** + die-cut outline(별색 thru-cut 레이어).
- **임포지션**: 접이식·여권은 패널/페이지를 인쇄용 한 장에 배치(여권=중철 페이지 순서 1-N).
- 색공간/해상도: 300dpi 유지, (가능하면) CMYK 안내·재단여백(bleed) 강제.
- 후가공 레이어: 박/형압 위치를 별도 마스크로 출력(인쇄소 전달용).

## 5. 운영자 어드민 (포맷별 오서링)

- 템플릿 생성 시 **포맷 선택** → 캔버스에 그 포맷의 **가이드 오버레이**(접는선·세이프·다이컷 윤곽·패널 경계) 자동.
- 포맷별 기본 규격 프리셋(위 §2 표)을 시드로 제공, 운영자는 규격·후가공만 조정.
- 미리보기: 평면(인쇄면) + (가능하면) 접힘/조립 3D 목업 미리보기.

## 6. 사용자 편집

- 종이는 **캔버스 편집 유지**(비즈하우스 = 인쇄 정밀 좌표). 단 **가이드 오버레이**(접는선/세이프/다이컷)
  를 보여 사용자가 안전영역 안에 두도록. 포맷별 다중 패널은 패널 탭으로 전환 편집.

## 7. 단계별 계획

- **P1** — 택소노미 확정 + `product_kind` enum 확장 + `InvitationPrintSpec`에 folds/shape/binding 추가 +
  포맷별 규격 프리셋 시드. (모델·시드만, 렌더 영향 최소)
- **P2** — 오서링/편집 **가이드 오버레이**(접는선·세이프·다이컷·패널 경계) + 평면 미리보기.
- **P3** — **PDF 내보내기 확장**: fold marks·crop marks·die-cut outline·임포지션(접이식/여권).
- **P4** — 다이컷·조립 포맷(바람개비·초콜릿·여권) 조립 가이드 + 후가공 사양 메타(박/형압/도무송/미싱).
- **P5** — 인쇄 파트너 연계(공급): 실제 인쇄소 사양으로 규격·후가공·발주 포맷 확정.

## 8. 제휴 업체 capability (확정) → 포맷 가능/보류

**확정**: 제휴 인쇄 업체는 **레이저 컷팅 + 코팅** 가능. → 리스크 대부분 해소.

| 포맷 | 가능 여부 | 근거 |
|---|---|---|
| 엽서카드·포토카드(라운드) | ✅ 가능 | 레이저 라운드/다이컷 + 코팅 |
| 2단·3단 접이식 | ✅ 가능 | 레이저 **스코어(오시) 라인**으로 접는 선 |
| 초콜릿 포장지(전개도) | ✅ 가능 | 레이저 다이컷 + 스코어(접이 플랩) |
| 티켓(절취선) | ✅ 가능 | 레이저 **마이크로 퍼프**(점선 컷)로 절취 |
| 바람개비(다이컷+조립) | ✅ 가능 | 레이저 대각 절개 + 조립 가이드 |
| 여권(제본 booklet) | ⏸ 보류 | **제본(중철/실제본) 미확인** — 레이저 컷팅으로 해결 불가 |
| 박(foil)·형압(emboss) 데코 | ⏸ 보류 | 별도 장비 — 파트너 확인 필요 |

→ **여권과 박/형압 데코를 뺀 전 포맷이 공급 가능**. 우선순위: 엽서·포토카드·2/3단(쉬움) →
초콜릿·티켓·바람개비(다이컷·조립) → 여권은 제본 확인 후.

## 9. 리스크 / 결정 필요

- "한 장 접힘(folds)" vs "여러 장(pages)" 의미 분리 필요 — 기존 `pages[]` 사용처 회귀 주의.
- **레이저 컷 라인 데이터 포맷**: `dieCutPathSvg`(thru-cut)·스코어 라인을 파트너 입력 규격(별색 레이어·
  SVG/AI·라인 두께)에 맞춰 내보내야 함 — 파트너 입고 사양만 확인하면 P3 착수 가능.
- 우선순위 결정: 어떤 포맷부터 공급할지(권장 §8 순서).

## 10. 현재 상태

**계획서(문서)만.** 종이 트랙은 제휴 업체 레이저 컷팅·코팅 확정으로 **여권 외 전 포맷 구현 경로가 열림**.
P1(모델·시드)→P2(가이드 오버레이)→P3(레이저 컷/스코어 PDF 출력)부터 착수 가능. 구현은 별도 지시 시.
