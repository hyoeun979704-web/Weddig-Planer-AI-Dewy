# 개인화(초개인화) 고도화 — 기회 매트릭스 + 실행계획 (260701)

> 배경: 사용자 "사용자 경험 개인화 고도화 할까?" → 병렬 감사 3(탐색·준비도구/AI·청첩장/쇼핑/커뮤니티)로
> **개인화 기회 매트릭스**를 실측(AGENTS §14 산출물). 깊이 사다리: ①없음 ②정렬/필터 ③큐레이션
> ④카피/CTA 변형 ⑤생성형 맞춤. 신호 단일 소스: `weddingPersona`(20모드)·`tasteTaxonomy`·
> `usePersonaInsights`·`useWeddingProfile`·`tipCuration`·`personaRecommendations`·`personaCoaching`.

## 2. 데이터 현실 필터 (중요 — 무엇을 지금 하나)
취향·앨범태그·행동·파트너딜·템플릿태그 데이터는 **~0**. 그래서 **지금 채워진 신호(persona·지역·D-day·
예산·스타일)로 즉시 효과 나는 것**을 먼저 한다. 취향 플래그 켜기는 1줄이나 연료(앨범 무드태그)가 없어
현재 거의 no-op → **데이터 유입 후**로 둔다(플립 자체는 무해·가드됨).

## 3. IA 결정 (사용자 확정 260701)
- **하단탭 5번째(현 커뮤니티) → "탐색/찾기(업체 마켓)"**. 커뮤니티는 아래 콘텐츠 허브로 이동.
- **꿀팁 탭 → 콘텐츠 허브**: 상단 메뉴탭 아래 **소스 서브탭** = 꿀팁영상·블로그·커뮤니티·유튜브·인스타
  (+ **카페=공식 개설 예정** → 준비 전엔 비활성/숨김, dead-end 금지). 각 소스 페르소나·지역 큐레이션.

## 4. 실행계획 (순차 커밋, 각 14차원 자기검증)
**Phase A — 데이터 즉시 가용·독립(먼저)**
- **P1 카테고리 리스트 개인화**(②→③): Venues·Studios·Honeymoon·Jewelry·Hanbok·Suit·Appliances 등
  리스트에 `excluded_categories` 필터 + `PERSONA_REC_CATEGORIES` 순서 가중을 이식(홈 패턴 재사용).
  self/스냅/노웨딩은 무관 카테고리 진입점 정리, 표준은 관련 카테고리 우선. 결과 0이면 폴백.
- **P2 견적 prefill**(②→④): `QuoteNew` 에 페르소나별 예산/규모 prefill(재혼=작은규모·budget=절약·
  luxury=호텔평균) + 업체 전달 note 컨텍스트 자동첨부(토글·수정 가능). prefill 상수 단일 소스.
- **P3 AI 스튜디오 정렬**(②→④): `AIStudio` 카드에 역할(신랑→예복/헤어, 메이크업/드레스 하향)·
  페르소나(예산형→상담할인 상단, 임신→편한 스타일 힌트) 정렬·강조. 기존 ceremony hide 유지.

**Phase B — IA 재편(더 큰 UX 빌드)**
- **P4 콘텐츠 허브 + 하단탭 탐색**: ① 꿀팁 페이지를 소스 서브탭 구조로(꿀팁영상=기존 TipsSection,
  블로그=`/blog`, 커뮤니티=기존 커뮤니티 이식, 유튜브·인스타=수집/큐레이션, 카페=예정). ② 커뮤니티를
  허브 서브탭으로 이동 + **페르소나·지역 랭킹**(tipCuration `trendingScore`에 persona/region boost).
  ③ 하단탭 커뮤니티→탐색/찾기(카테고리 마켓 진입). 라우트·딥링크·튜토리얼 앵커 정리.

**Phase C — 데이터 유입 후(이월)**
- 취향 부스트 플래그 ON(앨범 무드태그 쌓인 뒤) · 청첩장 featured_personas(마이그+태깅) ·
  행동로그 랭킹 반영 · AI플래너 오늘-미션 주입 · personaCoaching 20모드 확장.

## 5. 게이트·검증
DB 선확인·도메인 경계·PGRST 정합·최소 표적 변경·e2e(데이터 채워 확인, 불가 시 한계 명시)·14차원 +
적대적 자기검증. 신규 surface 는 이 문서 + `audit-surface-map.md` 개인화 매트릭스에 갱신.
