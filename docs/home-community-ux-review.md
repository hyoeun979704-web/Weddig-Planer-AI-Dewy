## 자동 fix (이 PR)

### Fix 1. 커뮤니티가 `?style=` 딥링크를 읽도록 (`Community.tsx`)
홈 `HomeCategoryGrid` 의 페르소나 타일("셀프 후기"→`/community?style=self`,
"스몰 후기"→`/community?style=small`)이 약속한 필터가 실제로 적용되도록 수정.

- 진입 시 `?style=self|small|general` 이 있으면 해당 스타일을 **초기 필터**로 설정
- 딥링크로 스타일이 지정된 경우 `myStyle` 자동적용/안내 토스트는 건너뜀 (의도 충돌 방지)
- 콜드스타트 폴백(해당 스타일 글 0개 → 전체)은 그대로 동작

기존 링크 계약을 코드가 지키게 만든 것뿐, 새 UI/라우트 추가 없음.

### Fix 2. 홈 "오늘의 수다"에 "더보기" 추가 (`CommunityChatterSection.tsx`)
헤더를 제목 + "더보기 ›" 행으로 변경 → `/community` 이동.
바로 아래 "실시간 인기 게시물"(`PopularPostsSection`)의 더보기 패턴과 동일하게 맞춤.

## 결정 필요 (사용자 보고)

### A. 커뮤니티 서브헤더의 뒤로가기 버튼
**현 상태**: 커뮤니티는 하단 탭의 1차 목적지인데, 서브헤더에 `navigate(-1)`
뒤로가기 화살표가 있음. 또 상단에 `HomeHeader` + 커뮤니티 헤더 2개가 겹쳐 있음.

**관찰**: 하단 탭/딥링크/새로고침으로 바로 `/community` 에 들어오면 "뒤로" 갈
의미 있는 이전 화면이 없을 수 있음.

**선택지**:
- **A1** — 그대로
- **A2** — 뒤로가기 버튼 제거(1차 탭이므로). 제목만 노출
- **A3** — 뒤로가기 대신 홈(`/`)으로 보내는 동작으로 변경

→ **추천 A2** (1차 탭에는 보통 뒤로가기가 없음). 다만 진입 경로가 다양해 보고만 함.

### B. "오늘의 수다" 이름 중복
**현 상태**: 홈 `CommunityChatterSection` 제목과 커뮤니티 페이지 트렌딩 섹션
제목이 둘 다 "오늘의 수다". 내용은 다름(홈=최신 2개, 커뮤니티=트렌딩 5개).

**선택지**:
- **B1** — 그대로 (브랜드 일관 용어)
- **B2** — 홈은 "오늘의 수다", 커뮤니티 트렌딩은 "지금 뜨는 글"/"핫토픽" 등으로 차별화

→ **추천 B1** 유지하되, 혼동되면 B2 검토.

## 그 외 발견 (작은 것, 보고만)

- **`PostListCard` 하트(찜) 토글이 로컬 state 전용**: 홈 "오늘의 수다"·"리뷰"
  카드의 하트를 눌러도 `useState` 로컬값만 바뀌고 서버 반영/인증 없음.
  상호작용처럼 보이지만 실제 동작이 없어 오해 소지. `PostListCard` 가
  `ReviewSection` 과 공유돼 변경 영향 범위가 있어 이번엔 보고만 함
  (표시 전용으로 바꾸거나 실제 좋아요 연동 결정 필요).
- **커뮤니티 검색 진입로 2개**: 헤더 검색 아이콘 + 본문 검색 박스. 중복이지만
  박스가 더 발견성이 좋아 의도된 것으로 보임. 변경 없음.
- **홈 카테고리 타일의 일부 경로 재사용**: 셀프/스몰 추가 타일이 `/store`,
  `/invitation-venues` 등 기존 라우트를 재사용 — 목적지 없는 UI 를 안 만든다는
  점에서 합리적.
