## TL;DR — 가장 큰 갭 3

1. **가치축(친환경·비건·반려동물)을 이미지/스타일 어휘가 못 잡음 — 그런데 앱엔 이미 해결책이 있다.**
   설계가 `placeValueTags`·`useFilterStore.valueTags`·`useValueTagAvailability`(0건 칩 자동 숨김 =
   dead-end 가드)를 **재사용 안 하고 누락**. 이미지로는 "비건"을 못 봄 → 값 패싯으로 합쳐야(S-2).
2. **제휴 한정 × 지방/니치 = 구조적 빈 결과(dead-end).** 부산·청주 등은 제휴 풀이 작아 유사 0건.
   폴백을 "옵션"이 아니라 **기본 ON**(제휴 우선 + 비제휴 보충 + 정직 안내)으로 못박아야(G-2·S-1).
3. **초보 진단이 카테고리마다 반복되면 오히려 마찰.** "아직 모르겠어요"가 카테고리별로 따로면
   초보가 매번 다시 고름 → **전역 무드 1회 진단 → 카테고리 상속** 구조 필요(초보·F-1 시간압박).

## 페르소나별 핵심 마찰

- **초보(central)** — §4.1 예시진단이 정확히 이 사람용(✅). 단 카테고리별 반복(⚠️ D5) + 제휴풀
  작으면 진단 후에도 결과 빈약(⚠️ D2). → 전역 무드 1회 + 폴백 기본 ON.
- **G-1 재혼·효율** — 예시진단을 "초보 취급"으로 느낌(⚠️ D1). designer/효율형은 **진단 스킵 →
  필터/업로드 우선**(설계 planning_style 분기로 일부 커버하나 재혼은 beginner 아님). 예시 풀이
  20대·초혼 편향이면 이질감(⚠️ D3) → 예시 대표성 가드.
- **G-2 부산·지방** — 제휴 한정 + 지방 → 유사 제휴 0건(❌ D2, 가장 심각). 같은-식장 포폴도 부산
  식장이 places 미등록이면 무효(⚠️ D4). → 지방은 제휴완화·반경확대·`venue_name` 폴백.
- **S-1 청주·한옥·전통** — "전통한복/혼주" 어휘가 모던 위주 예시·태그와 미스(⚠️ D3, label/value).
  부모(혼주)용 취향은 신랑신부와 별도 축(`target` 패싯 — 설계 §3.6에 있음 ✅, 단 예시진단엔 없음).
- **S-2 비건·친환경** — 이미지/스타일론 "친환경"을 못 잡음(❌ D3). **이미 있는 가치필터 재사용**이 답.
- **F-1 임산부·시간압박** — 진단이 길면 이탈(❌ D5, 세션 10분·주7회). 경량 미니퀴즈 우선(✅).
  임부 체형/편의(엠파이어·A라인·약품·식음) 어휘 부재(❌ D3) — v2 P11 연계.

## 횡단 진단

- **X1 가치축 누락 + 기존 자산 미재사용**: 설계가 `placeValueTags`/`useFilterStore.valueTags`/
  `useValueTagAvailability`(W1 dead-end 가드 내장) 를 안 씀. 이미지로 못 보는 축(친환경·비건·
  반려·종교·임부편의)은 **값 패싯**으로, 그것도 **이미 만든 단일 소스 재사용**(DRY).
- **X2 제휴 한정의 구조적 빈 결과**: 지방·니치에서 dead-end. 폴백 기본 ON + 정직 안내를 설계에 못박기.
- **X3 진단의 altitude**: 취향은 **전역 무드 1회**가 자연스럽고(카테고리 상속), 카테고리별은 옵션 세분.
- **X4 중앙 편향 전이**: v2 결론("서울·초혼·20대·정형체형 최적화")이 **예시 풀·어휘에도 전이**되면
  재혼·임부·전통·지방이 또 소외. 예시 대표성·어휘 포함성 가드 필요.
- **X5 식장 미등록**: 같은-식장 포폴 신호는 식장이 places 에 있어야 강함. 지방·소규모·한옥은
  미등록 多 → `venue_name` 텍스트 폴백 + 식장 등록 유도(설계에 폴백은 있음, 비중 강조).

## 우선 개선 (운영비 게이트 — low-cost·high-impact 순)

1. **[低] 가치 패싯 = 기존 `placeValueTags` 재사용 통합** — 후보 필터에 `valueTags.overlaps` 추가 +
   패싯 UI 는 `ValueTagChipRow` 패턴(가용성 0건 자동 숨김). 신규 어휘 만들지 말 것. (S-2, X1)
2. **[低] 제휴 빈 결과 폴백 "기본 ON"** — 제휴 우선 + 부족분 비제휴 보충 + "이 조건의 제휴가 적어
   전체에서 보여드려요" 정직 안내. (G-2·S-1, X2)
3. **[低] 전역 "웨딩 무드" 1회 진단 → 카테고리 상속** + designer/효율형 진단 스킵 분기. (초보·G-1·F-1, X3)
4. **[低] 예시 풀 대표성 가드 + 어휘 포함성** — 연령/재혼/체형/전통 다양 샘플, `target`(혼주)·임부
   편의·전통 어휘 포함. 데이터 부족 카테고리는 경량 퀴즈로 폴백. (X4)
5. **[中] `venue_name` 텍스트 폴백 비중 강화 + 식장 등록 유도** — 지방·한옥 미등록 대응. (X5)

## 결론 한 줄

> 설계의 레퍼런스/포폴/패싯 골격은 견고하나, **(a) 이미 만든 가치필터를 재사용**하고 **(b) 제휴
> 한정의 빈 결과를 기본 폴백으로 막고** **(c) 진단을 전역 1회로 올리면**, v2가 지적한 "중앙 밖
> 페르소나 소외"를 이 기능에서 반복하지 않는다.
