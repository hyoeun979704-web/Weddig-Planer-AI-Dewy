# 청첩장 인쇄 주문·결제 — 아키텍처 기획 + 정책/법무 안내

> 신규 제휴(청첩장 인쇄)용 인앱 주문·결제 설계. 플로우: 청첩장 제작 → 다운로드/바로받기 →
> 주문(용지·코팅·매수·빠른발송·배송지) → 업체 주문 알림 → 주문 확인·배송조회.
> 기준 커밋 `b779eda2`. **이 문서는 기획/검토 — 구현 전 결정사항(§6) 확정 필요.**

## 0. 핵심 결론 (먼저)
- **인쇄 청첩장 = 실물 상품 → 카카오페이 PG로 결제(기존 `kakao-pay-order-*` 레일 재사용).
  Apple/Google 인앱결제(IAP) 대상 아님 — 오히려 IAP 쓰면 리젝**(실물은 외부결제 필수/허용). §4.
- **법적 최대 분기: Dewy가 "직접 판매자"인가 "통신판매중개자"인가.** 중개면 고지·판매자정보
  제공 의무가 다르다(§5). **이걸 먼저 정해야** 약관·정산·책임범위가 결정됨.
- 주문제작(맞춤 인쇄물)은 **청약철회 예외** 가능하나 **사전 고지+동의** 필수(§5).

## 1. 사용자 플로우 (상세)
1. **제작**: 기존 청첩장 에디터(`invitation/*`)에서 디자인 완성.
2. **다운로드 / 바로 받아보기**: 디지털본(PDF/이미지) 받기. ⚠️ 유료면 IAP 이슈(§4) → **무료 권장**.
3. **주문(인쇄)**: 옵션 선택 — 용지(종류), 코팅(유/무), 매수, 빠른발송(여부), 배송지(이름·연락처·주소).
4. **결제**: 서버가 옵션→가격 계산(클라 금액 불신) → 카카오페이.
5. **업체 주문 알림**: 결제완료 → 업체 포털 + 메일/푸시 알림 + 인쇄용 고해상도 파일(서명 링크).
6. **주문 확인·배송조회**: 업체가 접수/제작/발송 상태 갱신 + 택배사·송장번호 → 소비자 배송조회.

## 2. 아키텍처

### 재사용 (이미 있음)
- **결제 레일**: `kakao-pay-order-ready`(서버 총액 계산·`orders`/`order_items` 생성·카카오 ready) →
  `kakao-pay-order-approve`(승인). **금액은 서버에서만 계산**(클라 불신) — 그대로 따른다.
- `orders`/`order_items`, 업체 포털(`/business/*`), 알림(Resend `notify-*` + DB webhook 패턴),
  청첩장 에디터·렌더(고해상도 PDF: `invitation-template-rules` 300dpi/bleed/safe-margin).

### 신규 스키마
- `print_products` — 업체가 등록하는 인쇄 상품/가격표:
  `place_id, name, base_price, paper_options[], coating_options[], qty_tiers(jsonb: 매수구간→단가),
  express_fee, shipping_fee, lead_time_days, active`.
- `print_orders` — 주문:
  `id, user_id(주문자), place_id(업체), invitation_id(디자인), print_product_id,
   options(jsonb: paper, coating, quantity, express),
   ship_name, ship_phone, ship_zip, ship_addr1, ship_addr2,
   amount(서버계산), order_id(orders FK·결제연결),
   status, courier, tracking_no, print_file_path(인쇄용 PDF), created_at, updated_at`.
- 상태머신 `status`: `paid`(결제완료) → `received`(업체 접수) → `in_production`(제작중) →
  `shipped`(발송, courier+tracking_no) → `delivered`(배송완료) / `canceled`·`refunded`(예외).
- RLS: 주문자(`user_id=auth.uid`) read, 업체(place owner) read+상태 update, 생성은 결제승인 경로(definer).

### 결제 흐름(확장)
- `kakao-pay-order-ready` 에 **상품유형 `print`** 분기 추가: print_product+options 로 서버 총액 계산
  (base + 매수단가 + 코팅 + 빠른발송 + 배송비) → orders 생성 → 카카오 ready.
- `kakao-pay-order-approve` 승인 시 `print_orders` 생성(status=paid) + 인쇄용 PDF 확정.
- **결제 금액 변조 방지**(기존 원칙), 부분쓰기/중복승인 방지(트랜잭션·멱등).

### 업체 알림 + 파일 전달
- `print_orders` INSERT → DB webhook → `notify-print-order`(Resend, mail-oauth 인프라와 별개 운영
  알림) + 업체 포털 `/business/print-orders`.
- 인쇄용 고해상도 PDF: **프라이빗 버킷 + 서명 링크**(민감·저작물) — vendor-deliveries 패턴 재사용.

### 배송조회
- `courier`(택배사 코드)+`tracking_no` 저장 → 소비자 화면에 택배사 추적 딥링크(또는 스마트택배/
  델리버리트래커 API 연동은 후속). 상태 `shipped`→`delivered` 는 업체 수동 or API.

## 3. 화면
- 소비자: 에디터 → "인쇄 주문" → 옵션/배송지 → 결제 → `/orders`(or `/print-orders/:id` 주문상세·배송조회).
- 업체: `/business/print-orders`(주문목록·접수/제작/발송 상태·송장입력·인쇄파일 다운로드).

## 4. 앱스토어 정책 (중요)
- **실물(인쇄 청첩장) = IAP 비대상**:
  - Apple App Store Review **3.1.3(e) "Goods and Services Outside of the App"** + **3.1.5(a) 물리적
    상품**: 실물 상품/서비스는 **앱 외부 결제수단(카카오페이 등) 사용 가능, IAP 금지**.
  - Google Play **결제 정책**: 실물 상품은 Google Play 결제 **사용 불가**(외부 PG 사용). 
  - → **카카오페이 PG가 정답.** 실물에 IAP 붙이면 양 스토어 모두 리젝/위반.
- **디지털 "바로 받아보기"(PDF 다운로드)** 가 **유료**면 정반대로 **IAP 대상**이 될 수 있음
  (Apple 3.1.1 / Google). → **권고: 디지털본은 무료**(또는 기존 구독 혜택 내) 제공, **유료 결제는
  실물 인쇄에만** 한정해 IAP 회피·정책 명확화. 실물+디지털 혼합 결제 금지(분리).
- 웹(PWA/Vercel) 결제는 스토어 정책 무관하나, **안드로이드 앱(Capacitor)** 내 결제는 위 기준 적용.

## 5. 법적 처리 (한국)
1. **사업 형태 결정 — 직접판매 vs 통신판매중개** *(최우선)*:
   - 직접판매(Dewy가 판매당사자): Dewy가 **통신판매업 신고**, 청약철회·환불·표시 책임 전부.
   - **통신판매중개**(업체가 판매자, Dewy는 장터): 화면·약관에 **"통신판매중개자이며 거래당사자가
     아니다"** 고지 + **판매자(업체) 신원정보 제공 의무**(상호·대표·사업자번호·연락처·주소).
     전자상거래법 §20. 정산 구조도 여기서 갈림.
2. **통신판매업 신고**: 직전년/거래 요건 충족 시 관할구청 신고, **신고번호 화면 표시**.
3. **전자상거래법 표시의무**: 판매자 사업자정보, 가격·배송비·발송예정일·결제수단, 청약철회 조건.
4. **청약철회/환불**: 일반은 7일. 단 **주문제작(소비자 사양 맞춤 인쇄물)은 청약철회 제한**
   (전자상거래법 §17②5호) — **단, 사전에 "맞춤제작이라 철회 불가"를 고지하고 동의 받아야** 적용.
   제작 착수 전 취소·하자/오배송 환불 정책은 별도 명시.
5. **결제대금 보호(에스크로/구매안전)**: 선결제 통신판매는 **결제대금예치(에스크로) 또는 소비자
   피해보상보험** 제공 의무(§13, 일부 면제요건 있음). PG사 제공 구매안전서비스 활용.
6. **PG·정산**: 카카오페이 가맹(이미 사용 중) + 업체 **정산 주기·수수료** 계약. 세금계산서/현금영수증
   (PG/업체 발행). 정산은 중개수수료 모델이면 분배 로직 필요.
7. **개인정보**: 배송지(이름·연락처·주소)는 개인정보 → **수집·이용 동의** + **제3자(판매업체) 제공
   동의**(배송 목적, 항목·보유기간 명시). 거래기록은 전자상거래법상 **5년 보관**(대금결제·공급 기록).
8. **표준약관·분쟁**: 청약철회/교환/반품 규정, 소비자분쟁해결기준 준용, 분쟁 처리 연락처.

## 7. 단계 (제안)
1. 스키마(print_products·print_orders) + 업체 인쇄상품 등록 UI.
2. 결제 확장(kakao-pay-order `print` 분기·서버 금액계산) + 주문/옵션/배송지 UI.
3. 업체 주문관리 포털 + 알림 + 인쇄파일 서명링크.
4. 배송조회(송장→추적) + 약관/고지/동의 화면(법무 §5 반영).
> 각 단계 결제·개인정보 경로라 **정적 통과만으로 완료 보고 금지**(실결제·실배송 e2e 확인).
