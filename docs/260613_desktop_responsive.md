# 260613 데스크톱 반응형 재설계 — 기반(Wave 0) + 페이지 전환 로드맵

> 목표: 모바일 세로 SPA(143 페이지)를 데스크톱/와이드에서도 제대로 보이게 한다.
> 전략: **셸 기반을 한 번 깔아 전 페이지가 즉시 혜택**받게 하고(Wave 0, 이번 작업),
> 페이지별 멀티칼럼은 추적표로 단계 전환(Wave 1+). 아키텍처 근거는
> Explore 매핑 결과(아래 §배경) 기준.

## 핵심 안전 원칙

**모든 데스크톱 변경은 `lg:`(≥1024px) 또는 `@media(min-width:1024px)` 뒤에만 둔다.**
→ 모바일/네이티브(안드로이드 APK·iOS)는 **1px도 안 바뀐다**. 브라우저 e2e 검증이 불가능한
환경에서 광범위 시각 변경의 회귀 위험을 모바일에서 0으로 격리하기 위한 규칙.

## 배경 (현 구조)

- 공유 레이아웃 셸 없음 — 143 페이지가 각자 `max-w-[430px] mx-auto` 하드코딩(**202곳/130파일**).
- `BottomNav` 가 `fixed bottom` 가로 탭바로 140 페이지에 인라인 렌더.
- 메인 페이지의 `md:`/`lg:` 사용 **0건**(순수 모바일 세로). 어드민만 일부 반응형.

## Wave 0 — 이번 작업(완료, 커밋)

| 변경 | 파일 | 내용 |
|---|---|---|
| 콘텐츠 칼럼 토큰화 | `src/index.css` | `max-w-[430px]`(202곳) → `.app-col`(= `max-width: var(--app-col-max)`). 모바일 430px 동일, 데스크톱 720px. 단일 소스化(드리프트 차단). |
| 셸 토큰 + 오프셋 | `src/index.css` | `--app-sidebar-width`(0→240px@lg), `--app-col-max`(430→720px@lg). `@media(min-width:1024px)` 에서 `#root { padding-left: 사이드바폭 }` → 좌측 사이드바 자리 확보. |
| 하단탭 → 사이드바 | `src/components/BottomNav.tsx` | 모바일 하단 가로바 그대로 + `lg:` 에서 좌측 세로 사이드바(브랜드 워드마크 + 세로 메뉴, 아이콘+라벨 가로 정렬, hover/active 배경). |
| 페이지 일괄 치환 | `src/pages/**`(130파일) | `max-w-[430px]` → `app-col` (sed, 기계적). 모바일 동작 동일, 데스크톱에서 자동으로 칼럼 확장. |

결과: 데스크톱(≥1024px)에서 **좌측 사이드바 내비 + 중앙 720px 콘텐츠 칼럼**으로 즉시 전환.
모바일은 불변. 빌드·린트(0 errors) 통과.

### 검증 한계 (must)

- **브라우저 e2e 미확인**(이 환경에 브라우저 없음). 빌드/린트/생성 CSS(`. app-col`, `min-width:1024px`)
  존재까지만 확인. **데스크톱 실제 렌더(사이드바 정렬·칼럼 중앙·스티키 헤더 폭·모달)는 사용자
  브라우저에서 확인 필요.** 특히 아래 "알려진 미세 이슈" 점검 요망.

### 알려진 미세 이슈 (Wave 1 에서 처리)

- 일부 페이지의 `fixed left-0 right-0` 요소(AI플래너 입력바 등 `safe-ai-input-offset`)는 fixed라
  뷰포트 기준 → 데스크톱에서 사이드바와 겹칠 수 있음. 해당 요소에 `lg:left-[var(--app-sidebar-width)]` 보정 필요.
- `safe-bottom-scroll`/`pb-20` 류 하단 패딩은 데스크톱에서 불필요(사이드바가 좌측) — 무해하나 정리 여지.
- 칼럼 720px 확장으로 풀블리드 히어로 이미지가 커질 수 있음 — 페이지별 점검.

## Wave 1+ — 페이지별 멀티칼럼 전환 (추적표, 미착수)

데스크톱 폭을 실제로 활용하려면(단일 칼럼 확장을 넘어) 주요 페이지에 `lg:grid lg:grid-cols-*`
도입이 필요. 우선순위(트래픽·가치순):

| # | 페이지 | 데스크톱 전환 방향 | 상태 |
|---|---|---|---|
| 1 | 홈(`Index`) | 히어로/미션/다음액션을 2~3단 그리드, 우측 사이드 요약 | ⬜ |
| 2 | 마이페이지(`MyPage`) | 프로필+통계 2단, 카드 그리드 | ⬜ |
| 3 | 스토어/상품목록(`Store`) | `lg:grid-cols-3~4` 상품 그리드 | ⬜ |
| 4 | 커뮤니티(`Community`) | 리스트+우측 사이드(인기/카테고리) 2단 | ⬜ |
| 5 | 예산/스케줄(`Budget`/`Schedule`) | 좌측 입력 + 우측 차트/타임라인 2단 | ⬜ |
| 6 | 상세(`ProductDetail`/`VendorDetailPage`) | 이미지 좌 / 정보 우 2단 | ⬜ |
| 7 | 청첩장 에디터 | 데스크톱은 이미 넓은 캔버스 선호 — 별도 검토(거대 파일 분할과 함께) | ⬜ |

> 전환 시 공통 패턴: 페이지 루트 `app-col` 을 섹션별 `lg:max-w-none lg:grid` 컨테이너로 승격하거나,
> `app-col` 안에서 `lg:grid-cols-2` 적용. 각 페이지 PR 에 이 표의 상태를 갱신.
