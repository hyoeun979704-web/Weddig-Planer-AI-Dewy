# 웨딩촬영 시안(photoshoot) — 현황·빌드 범위·결정 메모

작성: 2026-06-26 · 관련 설계서: `docs/260620_wedding_photoshoot_draft_plan.md`

## TL;DR

photoshoot 는 "배포만 하면 되는 미배포 기능"이 **아니다**. AI 스튜디오에서 **`coming_v2`로
올바르게 잠겨 있고**(사용자에게 "준비 중" 표기 — dead-end 아님), 스키마 마이그 1장 외에는
**워커·cron·생성/결과 UI·PDF 가 전부 미구현**이다. 즉 **깨진 게 없고**, "완성·배포"는 곧
**대형 AI 이미지생성 기능 신규 빌드**(이미지생성 API 비용 수반)다. 착수는 비용·범위 승인 후.

## 현재 상태 (실측)

| 구성요소 | 상태 |
|---|---|
| 스키마(`photoshoot_drafts`·`_cuts` + 버킷 + RLS) | 마이그 작성됨, **프로덕션 미적용**(history 마킹에서 의도적 제외, Task #7) |
| 프롬프트 로직 `src/data/photoshootPrompt.ts`(+test) | ✅ 존재(CUT_PLAN 등) |
| 워커 edge function(8컷 비동기 처리) | ❌ 없음 |
| pg_cron 워커 트리거 | ❌ 없음(설계서가 "후속 마이그"라 했으나 미작성) |
| 생성 입력 페이지(사진 업로드→draft/cuts INSERT) | ❌ 없음 |
| 결과/진행률 페이지(폴링) | ❌ 없음 |
| PDF 2장 조립 | ❌ 없음 |
| 클라이언트가 `photoshoot_*` 테이블 접근 | ❌ 0건 |
| AI 스튜디오 카드 `wedding-photo` | `status: "coming_v2"`(잠금/준비중) |

→ 스키마만 배포하면 **아무것도 채우지/처리하지 않는 빈 테이블**이 된다. 의미 없음.

## 빌드 범위 (설계서 기준, 착수 시)

1. **워커 edge function** — pending cut 1개씩 high 품질 이미지생성(gpt-image), 8컷 완료 시 draft 마감.
   레퍼런스는 vision 으로 텍스트화(입력한도·타인얼굴 오염 회피). 커플컷=베스트에포트.
2. **pg_cron 마이그** — 워커를 주기 호출(30분 비동기 큐). 스키마 마이그도 이때 함께 적용.
3. **입력 페이지** — 신부(헤어메이크업 완료+드레스)·신랑(헤어+슈트)·소품/레퍼런스 업로드 →
   하트 차감 → draft+cuts INSERT. (신부 헤어메이크업 사진 없으면 AI스튜디오 메이크업/헤어로 안내·차단)
4. **결과/진행률 페이지** — cut 진행률 폴링, 완료 시 PDF(1장 단독/2장 커플) 표시·다운로드.
5. **AI 스튜디오 카드 잠금 해제**(`coming_v2`→활성) + 딥링크.
6. **types.ts 재생성**(스키마 적용 후).
