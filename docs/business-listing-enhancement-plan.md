# 사업자 정보등록 → 매칭 선순환 기획서 (v2)

> 목적: 단순 "정보 등록"을 넘어 **닫힌 회로(closed-loop) 매칭 플라이휠**을 만든다. 매칭 결과·
> 피드백·품질 신호를 쌓고, 그 **기록을 업체가 직접 보고 스스로 보완**하게 해서 →
> 소비자 만족 ↑ → 업체 직업만족·업무퀄리티 ↑ → 더 좋은 매칭의 선순환을 돌린다.
> (코딩 전 기획. 근거는 코드/스키마/RPC 분석 — `docs/`·이전 분석 기반.)

## 비전: 열린 회로 → 닫힌 회로

```
[지금] 소비자 → 매칭 → 업체에 리드 → ……침묵 (결과가 안 돌아옴 → 학습·개선 없음)

[목표]
   소비자 ─매칭─→ 업체 ─응답·상담·계약─→ 결과(만족/후기) ─┐
      ↑                                                   │
      │            ┌──────── 업체가 기록을 보고 보완 ◀──────┤
      └─ 더 정확한 매칭 ◀─ 결과가 랭킹·추천·업체코칭으로 되먹임 ┘
```

**선순환 엔진 = 투명성.** 업체가 자기 성과·수요·이탈사유를 *직접 보면* 스스로 개선한다.

## 1차 원리

매칭 품질 = **데이터 완성도 × 매칭 신호 × 결과 학습**. 지금은 ①데이터가 비어 있고
②결과 되먹임이 0이며 ③업체가 자기 기록을 못 본다. 이 셋을 메우는 게 기획의 골자다.

## 5 기둥

### P-A. 등록 데이터 완성도 (기존 P1~P4 흡수)
큐레이션이 필요로 하는데 비어있는 신호를 사업자가 채우거나 자동 보강.
- 자동 지오코딩 연결(`place-geocode-backfill` 재사용) → lat/lng → 지도·거리 매칭 복구.
- 등록 완성도 미터 + 미충족 항목 딥링크(자발적 완성도↑, 정렬이 `data_completeness` 사용).
- 포트폴리오 **식장 태깅 UI**(`place_media.venue_place_id`) + `style_tags` 통제어휘 → "같은 식장" 매칭(최강 신호).
- 카테고리별 상세폼(Phase 2b) 완성 → 상세 필터 데이터.
- 가치태그·가격 구조화 → 신부 선호 매칭.

### P-B. 결과 되먹임 측정 (빠진 핵심 조각)
매칭이 학습할 연료를 모은다. **이미 머지된 `user_events`/`behavioralSignals` 위에 이벤트만 확장.**
- 추천 퍼널 이벤트: `rec_impression`(노출) → `rec_click` → `rec_feedback`(👍/👎/"안 맞아요").
- 리드 퍼널 단계 기록: 견적요청 → 업체응답 → 상담 → 계약/이탈 (각 단계 타임스탬프).
- 계약 후 후기를 **매칭 리드와 연결** → 결과 회로 닫힘.

### P-C. ★업체 인사이트·코칭 대시보드 (핵심 신규 — "기록을 업체가 보고 보완")
P-B로 쌓인 기록을 업체에게 **읽기 + 실행 가능한 형태**로 돌려준다.
- **내 리드 퍼널 & 전환율**: 받은 리드 N → 응답 N → 상담 N → 계약 N, 단계별 이탈률.
- **응답 SLA**: 평균 응답시간·응답률(느릴수록 전환·노출 손해를 수치로).
- **추천 성과**: 내 카드 노출/클릭/피드백 — "왜 안 골렸나"(👎 사유 집계).
- **내 지역 수요 트렌드**(익명 집계): 우리 지역 신부들이 요즘 찾는 스타일·예산대·날짜.
- **경쟁 대비 위치**(익명 집계): 내 가격·완성도·평점이 동지역·동급 대비 어디쯤.
- **데이터 완성도 체크리스트**: 미충족 항목 → 등록 화면 딥링크(P-A와 직접 연결).
- **놓친 이유**: 미응답/가격초과/지역불일치/날짜불가 등으로 분류 → 개선 포인트.
- **AI 개선 제안**(P-E): "응답을 N시간 내로 / Y필드 채우면 노출 N%↑ / 가격대 조정 제안".
- **프라이버시 가드(필수)**: 업체에는 **본인 리드 + 익명 집계만** 노출. 타 소비자 PII·타 업체
  식별정보 금지(RLS로 강제, 클라엔 제네릭·상세는 서버). — AGENTS.md 보안 차원.

### P-D. 품질 기반 랭킹 (광고비 아닌 품질 경쟁)
- 품질점수 = 응답속도 + 이행품질 + 후기결과 + 데이터완성도 → 기존 `partner_rank`/정렬에 주입.
- 업체가 **서비스 품질을 올려야 노출이 오름** → 소비자 더 좋은 경험 → 만족 → 리드↑ → 품질 투자.
  (선순환 폐곡선)

### P-E. AI 업체 보조 (노력↓ 품질↑)
기존 LLM 인프라 재사용: 리스팅 문구 개선·리드 응답 초안·포폴 태깅 자동 제안·인사이트 요약.

## 로드맵 (단계 — 측정 먼저)

| Phase | 범위 | 이유 |
|---|---|---|
| **1** | P-A(자동 지오코딩 + 완성도 미터) + **P-B 측정 시작**(추천·리드 이벤트 확장) | 측정 없으면 학습 없음. 최저위험으로 데이터부터 쌓기 시작 |
| **2** | **P-C 업체 인사이트 대시보드 v1**(본인 퍼널·응답 SLA·완성도 체크리스트) | 쌓인 기록을 업체가 *처음으로* 보고 보완 — 선순환 점화 |
| **3** | P-A 포폴 식장태깅·통제어휘 + **P-D 품질 랭킹** | 최강 매칭 신호 + 품질→노출 인센티브 |
| **4** | P-C v2(지역 수요·경쟁 가격 익명 집계) + P-E AI 보조 + P-A 상세폼 | 깊은 코칭·정밀 매칭 |

**권장 시작점: Phase 1.** P-A 완성도 + P-B 측정을 동시에. 대시보드(P-C)는 측정 데이터가
쌓이기 시작해야 의미 있으므로 그 뒤.

## 데이터 모델 (개략)
- **`user_events` 확장**: `rec_impression`·`rec_click`·`rec_feedback`·`lead_stage_change`
  (event_name 자유문자열이라 클라만 추가하면 시작 — 기존 인프라).
- **리드 퍼널**: 기존 `quote_requests`/`quote_responses` 상태 + 단계 타임스탬프 보강.
- **집계 뷰**: `vendor_lead_funnel`·`vendor_quality_score`·`region_demand`(익명) —
  업체 본인 RLS 또는 admin/service_role 집계. 소비자 PII 비포함.

## 검증·안전 원칙
- DB/RPC·뷰 변경은 **마이그레이션 작성 → 타입/esbuild 검증 → 실환경 적용은 사용자 확인 후**
  (sandbox e2e 불가). 컬럼/RLS 적용은 `information_schema`로 선확인(파일 존재 ≠ 적용).
- **프라이버시**: 업체 화면은 본인 리드 + 익명 집계만. RLS로 강제, 응답은 제네릭.
- 클라 로직(완성도 계산·퍼널 집계·점수식)은 **순수함수 + 유닛테스트로 고정**.
- 측정→대시보드→개선의 닫힌 회로를 깨지 않게, 각 단계가 다음 단계의 입력이 되도록 배선.
