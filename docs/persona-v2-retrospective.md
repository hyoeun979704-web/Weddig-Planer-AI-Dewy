## 발견된 정합성 경고 — "이미 만든 것 중 위험한 것"

### W1. `places.tags` 가치 태그가 비어 있을 가능성 (#7 후속)
- 신규 칩 4개(친환경/비건/반려동물/영문안내)는 클라이언트가 정의만 했고, 실제 places 행에 해당 한국어 태그가 들어 있다는 보장이 없음.
- 칩을 켜도 결과가 **0**이면 사용자 신뢰 손상 + S-2 페르소나 이중 실망.
- **조치 우선순위 高**:
  - 데이터 점검 쿼리 1회 (`SELECT count(*) FROM places WHERE tags && array['친환경','비건옵션','반려동물','영문안내']`)
  - 매칭 0~소수일 경우 (a) 칩 비활성화 (b) "곧 출시" 카피로 변경 (c) 운영 가이드 작성 + 시드 데이터 30~50건
- 운영 비용은 1인 부담 — 자동화 모델과 맞물려 신중히.

### W2. 하객 리스트 (#6)의 향후 확장 유혹
- CSV import / 청첩장 발송 / 좌석 배치 / 커플 공유는 모두 인접해 보이지만 핵심 5영역과 **직접 매핑이 약함**.
- 특히 좌석 배치는 결혼식장 운영 도구 영역 — 1인 운영 부담을 키운다.
- v0 CRUD까지가 컨셉에 맞는 안전한 선. 후속 확장은 사용자 데이터(실제 등록 수·세션 시간)가 모이기 전까지 보류 권장.

## 보류된 권고에 대한 재평가

### #5 콘텐츠 시드 (재혼·임산부·부모님 설득 매거진)
- **컨셉 정합**: 영역 ① AI 상담의 답변 풍부도와 영역 ④ 통합 정보에 직접 기여
- **운영 부담**: 1인 운영에 정면으로 충돌. 매거진 8~10편 작성·검수·업데이트는 시간 소비 ↑↑
- **결정 가이드**: 자동화 가능한 형태로만 들어가야 한다 — 즉 LLM이 정적 컨텐츠를 생성하는 게 아니라 **AI Planner 답변 시 끌어다 쓸 수 있는 캡슐형 지식 베이스**로. 매거진 페이지에 카드 N장을 직접 게시하는 형태는 ROI 낮음.

### #7 다른 카테고리(studios/honeymoon/hanbok) 확장
- venues 외 카테고리에도 valueTags 필터를 펴는 일은 W1 위험과 동일. **places 데이터가 부족한 상태에서 UI만 펴면 빈 칩**.
- venues에서 가치 칩 클릭률 + 결과 0건 비율을 며칠 보고 결정하는 게 합리적.

## 정합성 기반 다음 우선순위 — 최소 필수 (Top 3)

> 모두 새 기능 확장이 아니라 **이미 만든 것을 운영 가능 상태로 굳히는** 작업.

1. **W1 점검 — places 가치 태그 데이터 현황 확인 + 0건일 때 가드**
   - cost: 작음 · risk: 작음 · 사용자 신뢰 직결
2. **AI Planner system prompt에 "한국 도메인 캡슐 지식" 추가** (보류된 #5의 정합형)
   - 재혼·임신·부모님 설득 케이스에 답변 풍부도가 즉시 오름
   - 매거진 페이지 신설 없음 → 1인 운영 부담 0
3. **머지된 변경 5건의 사후 측정 — analytics 이벤트 부착**
   - PersonaDashboard 진입률, D-90 압축 토글, 가치 칩 클릭, 하객 추가/RSVP 변경
   - 데이터 없이 다음 결정 못 한다는 사실이 가장 큰 부채
