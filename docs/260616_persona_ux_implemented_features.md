## TL;DR

- **즉시 수정한 dead-end 1건**: 미니퀴즈가 취향 태그를 저장만 하고 **아무 데서도 소비 안 함** →
  결과 화면이 "이 분위기로 추천"이라 약속하고 `/vendors/기타`로 보내는데 **실제 정렬 반영 0**
  (거짓 약속/placeholder). → VendorList 에 **취향 태그 소프트 정렬 + "취향 맞춤 N곳" 표기** 연결해
  약속을 실제 동작으로 만듦(검증 규칙: toast/약속만 하는 CTA 금지).
- 나머지 기능은 페르소나별 큰 마찰 없음. 남은 개선은 deferred(저빈도·운영비).

## 적용한 개선 (이번 커밋)

- **미니퀴즈 → 실제 큐레이션 연결**: `VendorList` 가 `loadTasteTags()` 로 취향 태그를 읽어
  업체 `style_tags` 겹침 수로 소프트 정렬(같은-식장 1순위 → 취향 2순위 → 기존 순서).
  헤더에 "취향(로맨틱·내추럴) 맞춤 N곳" 표기 → 퀴즈가 "동작하는 척"하지 않음.

## 남은 마찰 (deferred — 저빈도·운영비/라이브검증)

- **G-2/S-1 취향·포폴 데이터 희소**: 지방·전통은 style_tags·포폴 venue 가 적어 정렬 효과 약함.
  데이터 시드/업체 입력 유도 선행(운영). venue_name 폴백은 이미 있음.
- **S-2 가치축**: 친환경·비건은 무드 퀴즈가 아니라 **기존 가치필터**(`placeValueTags`)로 — 설계
  §3.6 대로 별도 통합(이번 범위 밖).
- **F-1 결과물 알림**: 새 결과물 도착 시 알림 없음(마이페이지 진입만). 알림 연동은 후속.
- **미니퀴즈 진입 발견성**: 현재 `/vendors/*` 헤더에만. 홈/온보딩 노출은 후속(홈 편집 리스크).
- **전통/혼주 어휘**: 퀴즈 무드(클래식·모던·로맨틱·빈티지·미니멀·볼드)에 "전통" 축 보강 검토.

## 결론

> 4기능 모두 페르소나 검토 통과. 유일한 dead-end(미니퀴즈 거짓 약속)는 취향 소프트 정렬
> 연결로 즉시 해소. 남은 항목은 데이터 희소·알림·발견성으로, 운영/후속 PR 영역.
