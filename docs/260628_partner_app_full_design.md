# Dewy 사장님 앱 — 전체 설계 (260628)

> 동반 문서: 경쟁사 전수조사 `260628_partner_app_competitive_ux.md`(장점·매트릭스), 분리 로드맵 `260628_phase4_*`.
> 이 문서는 그 위에 **사장님 앱(apps/partners)의 제품 설계**(플로우·대시보드 IA·포트폴리오/개인화·취향 택소노미·
> 20 페르소나·데이터 매핑·구현 단계)를 확정한다. **§12에 적대적 자기검증→개선** 포함. 코드 변경 없음(합의용).

## 0. 실측 grounding (추측 아님 — 이 설계의 사실 기반)
- 취향 어휘 정식 소스 = `packages/lib/src/tasteQuiz.ts`의 **`MoodTag` 6개**: `클래식·모던·로맨틱·빈티지·미니멀·볼드`. `loadTasteTags()`(localStorage `dewy_taste_tags`)로 소비.
- 공급자 포트폴리오 = `place_media_albums`(앨범=폴더; 컬럼: `style_tags[]·title·venue_name·shoot_date·cover_media_id·product_id·display_order`) + `place_media`(사진, `style_tags[]`). **앨범에 공통 메타 1회 설정 → 사진 다수 귀속**이 이미 모델(`BusinessGallery` 주석).
- 매칭 소비처 = `VendorList.tsx`(카테고리 목록)만 `style_tags ∩ tasteTags`로 정렬/하이라이트. **메인 추천 `usePlaceRecommendations`는 `partner_rank` 정렬만**(취향 미사용).
- 공급자 태그 입력 = `BusinessGallery`의 **자유 텍스트(쉼표)** → 6 MoodTag와 불일치 위험(드리프트).
- 대시보드 = `BusinessDashboard` 승인 후 = 프로필·지점·제휴신청·**허영 스탯 4(조회/찜/쿠폰/사진)**·**관리 메뉴 12개 평탄 리스트(badge 전부 null)**.

## 1. 비전 — 한 문장
**"사장님이 포트폴리오를 취향 라벨과 함께 적극 등록 → 그 취향의 신부에게 우선 추천 → 고가치 리드 → 사장님 동기 → 더 많은 포트폴리오"** 의 선순환을 사장님 앱이 가동시킨다. 이 **포트폴리오→취향→초개인화 루프**가 Dewy의 심장이고, 사장님 앱의 존재 이유다(일반 셀러툴 + Dewy의 +1).

## 2. 현재 구조 진단 (고쳐야 할 3가지)
1. **대시보드가 "런처"** — 12개 메뉴 평탄 나열, `badge` 전부 null → 정보 냄새 0, 매번 뒤져야 함. 핵심 업무(리드·문의·리뷰)가 1회성 설정과 동급.
2. **태그가 자유텍스트** — 6 MoodTag와 불일치 → 취향 매칭 조용히 실패(개인화 베팅 누수).
3. **개인화 루프가 좁다** — `VendorList`에만 적용, 메인 추천은 `partner_rank`만. 취향이 본진에 없음.

## 3. 취향 택소노미 (단일 소스·다축·구체) — 이 설계의 토대
**원칙**: 공급자 태깅·소비자 콜드스타트·추천 매칭이 **하나의 어휘**를 본다(드리프트 차단). 구체성↔매칭밀도↔입력부담을 **다축**으로 푼다.

| 축 | 값 | 역할 | 출처/상태 |
|---|---|---|---|
| **무드(primary)** | 클래식·모던·로맨틱·빈티지·미니멀·볼드 | **매칭 밀도**(거친 교집합) | `tasteQuiz.MoodTag` = 정식, 그대로 사용 |
| 공간/연출 | 가든/야외·채플·호텔·하우스·스튜디오·한옥 | 세밀 랭킹·콜드스타트 다양성 | 신규(택소노미 확장) |
| 컬러톤 | 퓨어화이트·아이보리·파스텔·딥/보르도·뉴트럴 | 세밀 랭킹 | `dressFilters` 일부 재사용 |
| (카테고리 특화) | 드레스=실루엣/넥라인, 스튜디오=촬영컨셉, 홀=홀타입 | 카테고리 내 랭킹 | `dressFilters` 등 재사용 |

- **무드 6개를 1차 매칭 축으로 고정**(기존 데이터·매칭 보존). 공간·컬러는 **선택적 세밀 축**(랭킹·콜드스타트 풍부화). → "구체적이되 매칭 밀도 유지".
- **단일 소스화**: `tasteQuiz`의 MoodTag를 `packages/lib`의 공용 `tasteTaxonomy`로 승격, 공급자 피커·콜드스타트·매칭이 import. 자유텍스트 폐기.

## 4. 사용자 플로우 2종

### 4-A. 신규회원 — 가이드형 점진 온보딩 (유도 = 이점 카피 + 완성도 보상)
각 단계는 "**왜 이걸 하면 좋은가**"(이점)를 먼저 보여주고, 진행률/완성도로 보상. 상품·이벤트는 **건너뛰기 허용**(네일·아이폰스냅 등).
1. **기업회원 가입** — 가입 이유 + 제휴 장점(무료·무수수료·신혼부부 노출). (현 `BusinessOnboard`)
2. **상세정보 입력** — 업종별 분기(홀=식대·보증인원, 청첩장=디자인…). "상세할수록 검색/추천 상단" 이점 + **완성도 게이지**.
3. **포트폴리오 등록** ★ — 앨범(폴더) 만들고 **10장 일괄 업로드 + 취향 라벨(무드 피커)**. 이점: *"작품을 취향과 함께 올릴수록, 그 취향의 신부에게 우선 추천돼요"*. (포트폴리오 = 개인화 연료)
4. **상품 등록** — 구성·가격 상세 시 이점(견적 정확·전환↑). 건너뛰기 O.
5. **이벤트 등록** — 동일(노출·프로모션 이점). 건너뛰기 O.
6. **관리 가이드** — 상세정보/상품/이벤트/리뷰 관리법(현 `BusinessGuide`).
7. **대시보드 활용 안내** — 오늘 할 일·문의/리드 응대 흐름 안내 → 기존 플로우로 합류.

### 4-B. 기존회원 — 액션 우선 대시보드 (런처→업무 도구)
§5 참조.

## 5. 대시보드 IA 재편 (런처 → 액션 3단)
1. **오늘 할 일 (액션큐)** — 미응답 리드·미답변 문의(`status=open`)·미답글 리뷰(`owner_response IS NULL`)·임박 이벤트(`starts_at` 근접). **실제 카운트 뱃지**(기존 `badge` 필드 활용), 0이면 "처리할 일 없어요" 청정 상태. 푸시→탭→그 자리 처리.
2. **핵심 3숫자 + 1제안** — 허영 4개 폐기 → 의미있는 3개(예: 이번주 신규 리드·응답률·**완성도/포트폴리오 충실도**) + 제안 1개(*"포트폴리오 N장·태깅 X% → 취향 추천 노출↑"*).
3. **관리 (저빈도, 그룹핑)** — 정보/사진 · 프로모션(쿠폰·이벤트·상품) · 1회성(인수·디자인·가이드). 고빈도(리드·문의·리뷰)는 1번 액션큐로 승격.

## 6. 포트폴리오 = 개인화 연료 (앨범 폴더 · 인스타 피드)
> 사용자 확정: AI 자동태깅 폐기 → **각자 입력(통제 피커) + 10장 일괄 + 폴더링 + 인스타 피드**. 대부분 `place_media_albums`로 **이미 구현된 모델** — 델타만.

**공급자(BusinessGallery 강화)**
- **앨범 = 폴더**(이미 있음). 앨범 공통 메타(스타일·식장·촬영일) 1회 → 사진 다수 귀속(이미 있음).
- **델타①**: 단건 → **10장 멀티 업로드**(`ImageUploader` 멀티).
- **델타②** ★: `style_tags` **자유텍스트 → 무드 6개 통제 피커**(+선택 공간/컬러). 개인화 매칭 보존의 핵심.
- 보상 가시화: *"#모던 앨범이 모던 취향 신부 N명에게 노출"*.

**소비자(PlaceBusinessSections/VendorList 강화)**
- **인스타 피드** 형태로 앨범 그리드(친숙 UI). 앨범 스타일 필터 이미 있음(`PlaceBusinessSections`).
- **델타③** ★: 피드 **순서를 그 신부의 취향(taste ∩ 앨범 무드)으로 정렬** — 친숙 + 개인화.

## 7. 사장님 20 페르소나 (업종별 적응 검수용)
신규1 + 메인10 + 세분화9. (상세 표는 競합 채택문서 §2 / 본 세션 기록.) 핵심 함의: **한 구조가 고액·장기·예약형(홀·허니문·주얼리)과 저액·당일·다건(네일·부케·축의대)을 동시에 못 만족** → 대시보드 액션큐·온보딩·포트폴리오가 `service_category`로 **적응**해야 함(예: 홀=리드 파이프라인 중심, 네일=임박 예약 중심, 스튜디오=포트폴리오 피드 중심, 청첩장=디자인 판매/정산).

## 9. 컴플라이언스
- **개인정보(§13)**: 포트폴리오 인물 사진 — 외부 전송 없음(AI 폐기로 오히려 단순). 업로드 동의·보존정책만.
- **IAP/anti-steering(§8)**: 청첩장 디자인 판매·프로모션 결제는 네이티브에서 IAP 규정. 외부결제 유도 금지.
- **리뷰 무결성(§2/§10)**: 실거래(견적·계약) 검증 리뷰만. 셀프링크 어뷰징 금지.
- **추천 큐레이션 게이트**: 노출은 `is_active`·`moderation_status`·`partner_rank` + 취향. 0건이면 섹션 숨김.

## 10. 구현 단계 (재정렬 — 토대부터)
- **S0 택소노미 단일 소스화** — `tasteTaxonomy`(무드6 + 확장축) 공용화. 자유텍스트 → 피커 마이그레이션 경로(기존 free-text → 무드 매핑/정규화).
- **S1 포트폴리오 강화** — 멀티업로드 + 무드 피커 + 인스타 피드(공급/소비 양쪽). *루프의 연료.*
- **S2 대시보드 액션큐 + 완성도 게이지** — 런처→3단 재편. *기존 데이터 재배치.*
- **S3 추천 루프 확장** — `usePlaceRecommendations`에 취향 축(메인 추천 개인화). *루프를 본진으로.*
- **S4 리드/문의 상태머신 파이프라인** + 자동 첫응답/템플릿(P1).
- **S5+** 행동 랭킹·배지 / BI 퍼널+페르소나 / 구독 푸시 / 광고 2분기 (競합문서 P1~P2).
> 각 단계: web·capacitor build + lint/test/integrity 녹색 + 캡처 시뮬 + DB 선확인.

## 12. 적대적 자기검증 → 개선 (이 설계를 내가 깐다)
> "내가 사장/대표면 이거 충분한가?" 통과 못 한 지점과 그에 따른 **설계 수정**.

**R1. 🔴 양면 콜드스타트 (가장 큰 구멍).** 매칭은 *소비자 취향 캡처 ∧ 공급자 앨범 태깅* 둘 다 있어야 작동. 신규 신부(취향 0)·신규 업체(포트폴리오 0)·기존 4천 업체(자유텍스트/무태깅)면 `∩`=0 → "초개인화"가 빈손.
→ **개선**: ① 취향 0이면 `partner_rank` 폴백(현행 유지) + 콜드스타트 유도. ② **기존 free-text style_tags 정규화 배치**(무드 6 매핑) + 무태깅 업체에 "대표 무드 1개만 골라주세요" 1탭 백필 캠페인. ③ 신규 업체는 온보딩서 **무드 최소 1개 강제 아님·강력 권장**. → S0에 "기존 데이터 정규화"를 **필수 선행**으로 격상.

**R2. ⚠️ 무드 6개로 "구체적 라벨" 충분한가?** 사용자님은 넷플릭스식 구체 라벨로 만족도↑를 원함. 무드 6은 거침.
→ **개선**: 무드(매칭) + **공간·컬러 세밀축은 콜드스타트/랭킹 전용**으로 분리(매칭 밀도는 무드가 보장). 콜드스타트 레퍼런스는 **실제 앨범 이미지**(구체적 비주얼)로 보여주고 라벨은 뒤에서 다축 누적 → 사용자는 "구체적 이미지로 고른다" 체감, 시스템은 매칭 밀도 유지.

**R3. ⚠️ "포트폴리오 적극 등록" 강요가 1인 부담/반발.** 네일·축의대처럼 포트폴리오가 약한 업종엔 압박.
→ **개선**: 완성도/포트폴리오 넛지를 **업종별 가중**(스튜디오·드레스=포트폴리오 핵심, 네일·축의대=가격·예약 핵심). "오늘 3개만" 단계적, 빨간 폭탄 금지.

**R4. ⚠️ 통제 피커 = 표현 자유 상실.** 사장이 "우리만의 무드"를 못 적음.
→ **개선**: **피커(매칭용 무드) + 자유 캡션(표현용, 매칭 미사용)** 병행. 매칭은 통제어휘, 감성 카피는 자유. (label vs value 분리 원칙 그대로)

**R5. 🔴 S3(메인 추천 취향 확장)의 회귀 위험.** `usePlaceRecommendations`는 전 소비자 surface가 쓰는 핵심 — 취향 축 넣다 정렬/성능/0건 깨지면 광범위 회귀.
→ **개선**: 취향은 **재정렬 가산점(tie-breaker/부스트)**으로만, `partner_rank` 1차 정렬 보존 + 취향 0/매칭 0이면 현행과 동일. 피처플래그 뒤 점진 롤아웃 + 캡처 시뮬.

**R6. ⚠️ 두 앱 공유 영향.** `features/partners`·`BusinessGallery`·`usePlaceRecommendations`는 **소비자 웹/네이티브 + 사장님 앱이 공유** → 변경이 양쪽 파급.
→ **개선**: 각 단계 **양 빌드(web/capacitor) + 소비자 회귀 0** 검증 게이트(이미 마이그레이션서 확립한 패턴).

**R7. ⚠️ 측정 없으면 "초개인화 했다" 착각.** §11 로깅 없으면 효과 불명.
→ **개선**: S1·S3에 **매칭 노출·취향정렬 클릭·취향리드 로깅을 같은 PR에 포함**(기능과 측정 동시).

**개선 반영 요약**: S0에 **기존 태그 정규화 배치 + 무태깅 백필**을 필수로 끌어올림 · 매칭축(무드)과 표현/세밀축 분리 · 추천 취향은 **부스트로만**(회귀 가드·플래그) · 넛지 업종별 가중 · 측정 동시 출고.

## 13. 가이드형 상세정보 입력 위저드 ("하란대로 따라하면 고퀄이 완성") — 사용자 요청 반영

> 사용자 확정: *"처음 정보설정부터 가이드대로 직접 입력 → 따라하다 보면 고퀄 상세정보가 입력돼 있게."*
> 네이버 스마트플레이스·숨고·스마트스토어의 **질문형/가이드형 등록**을 Dewy에 맞춰 적용.

### 13.0 실측 진단 (지금 상세정보 입력이 왜 저퀄을 방치하나)
- `BusinessOnboard` = **사업자 인증 + 카테고리 + 제휴신청만** → "검토 중" 대기로 끝. **상세정보 입력 단계 없음**.
- 승인 후 `BusinessVendorEdit`(507줄) = **한 화면 덤프폼 + 저장 버튼 3개**(기본정보 / `BusinessListingContactForm` 연락처 / `BusinessListingDetailForm` 업종별). 사장이 스스로 길을 찾아야 함.
- **가이드·예시·좋은예/나쁜예 0** — 빈 placeholder만. 대충 적으면 저퀄 그대로 노출.
- `computeListingCompleteness`는 **기본 7필드만** 측정 → "100%"여도 상세·연락처·포트폴리오·무드 비어 얇은 리스팅 가능(silent gap).
- 업종별 상세(`DETAIL_SCHEMA` 11개 카테고리)는 **기본정보 저장(placeId) 후에야** 렌더 → 발견 갭.

### 13.1 핵심 전환: 덤프폼 3개 → 단일 가이드 위저드
**원칙**: 사장은 "무엇을 적을지" 고민하지 않는다. **질문이 끌고 가고**, 각 질문엔 *왜·예시·실시간 품질힌트* 3종이 붙고, 끝까지 따라가면 = 고퀄 리스팅 완성. 저장은 step마다 자동(draft 이미 구현), 마지막에 "검토 요청" 1회.

### 13.2 위저드 step (업종 적응 = `service_category` 분기, 필수 먼저·심화 나중)
**[1차 = 노출 최소조건, 짧게]**
1. **대표 이미지 + 업체명** — 가이드: 좋은 대표사진 예시(밝고 정면·공간/인물), 권장 비율(3:4). 나쁜예(어둡고 흔들림).
2. **상세 소개글** — ★ **업종별 골격 placeholder + 좋은예/나쁜예 + 글자수 힌트**(예: 100자↑ green). "강점 3·차별점·후기포인트" 템플릿 제시(빈칸 공포 제거).
3. **지역·매장·주소** — 매장 유무 분기(현행 로직 그대로).
4. **가격 + 키워드** — 키워드는 자유텍스트 대신 **업종별 추천 키워드 칩**(통제어휘=검색 매칭 보존, label/value 분리 원칙). → 여기까지 충족 시 **"이제 노출돼요"** 1차 완료(숨고식 핵심 먼저).

**[2차 = 고퀄·개인화, 권장·건너뛰기 가능]**
5. **업종별 상세**(`DETAIL_SCHEMA` 재사용) — 각 multi 옵션에 "왜 중요한지" 마이크로카피. 위저드 안으로 끌어와 발견 갭 제거.
6. **취향(무드) 태깅**(§3/§6 무드 피커) — 포트폴리오 개인화 연료. "이 무드의 신부에게 우선 노출".
7. **연락처·운영정보**(`BusinessListingContactForm` 재사용) — 전화·시간·SNS·주차.
8. **포트폴리오**(§6 앨범·10장·무드) — 업종 가중(스튜디오·드레스 핵심).
9. **문의 받는 방법** — 동작 검증(현행, 죽은 버튼 방지).
10. **미리보기 + 검토 요청** — 실제 `/vendor/:id` 렌더 재사용(목업 금지).

### 13.3 각 step "가이드 3종 세트" (= "따라하면 고퀄"의 실체)
- **왜**: 노출/전환에 미치는 영향("소개 100자↑면 체류·문의↑").
- **예시(좋은예/나쁜예)**: 실제 입력 샘플 — 네이버 팁 패턴. 업종별.
- **실시간 품질힌트**: 글자수·사진매수·필드충족 → green/amber inline 넛지("조금 더 적어볼까요?"). 강요 아님.

### 13.3-a 동기부여 내러티브 — "디테일·소비자친화 = 노출↑, 향후 마케팅 전환↑" (사용자 요청)
> 사용자 확정: *"디테일하고 자세하게·소비자 친화적으로 작성할수록 **노출에 유리**, 향후 **마케팅 진행 시 전환율**에도 영향."* — 위저드 전반의 **유도 카피 축**. (빈 공포·대충 입력 방지의 핵심 동력)

**근거(추측 아님)**: 큐레이션 추천 정렬이 `places.data_completeness`를 사용 → **완성도↑ = 추천 상단 노출↑**(`businessListingCompleteness.ts` 주석에 명시된 실제 동작). 취향 무드 태깅 → 그 취향 신부에게 우선 매칭(§3·§6). 따라서 "자세히·소비자친화적으로 = 노출 유리"는 **시스템상 사실**이며 사장에게 정직하게 약속 가능.

**카피 톤(중요)**: AI처럼 매끈한 설명체 금지. 토스·당근·네이버 비즈니스처럼 **짧고 담백, 구체 숫자·행동, 사장이 실제 쓰는 말**. "~돼요" 남발·군더더기 설명·작위적 비유(예: "문의 없이도 마음을 정해요") 빼고, 한 줄에 하나만.

위저드 카피 3겹(아래는 실제 문안 — 길게 풀지 말 것):
1. **노출(지금)**: *"자세할수록 위에 떠요."* / 게이지 옆: *"3개 더 채우면 노출 ↑"*(추상 % 대신 **남은 항목 수** 구체).
2. **소비자 친화(전환 토대)**: *"신부 눈높이로 적어주세요."* — 좋은예/나쁜예에 한 줄 라벨만(나쁜예: *"업력 20년 프리미엄"* / 좋은예: *"실내 가든홀, 하객 200명, 주차 100대"*). 설명 대신 **예시가 말하게**.
3. **마케팅**: *"상세페이지 꽉 찬 곳부터 광고·기획전에 올려요."* — 한 줄. *"무조건 1등"* 류 보장 표현 금지(W4·표시광고).

**작성 규칙**: 한 화면 카피 1~2줄. 명령형·체언종결 OK("적어주세요", "사진 3장"). 이모지·느낌표 남발 금지. 같은 말 반복 금지. **배치**: ① 진입 인트로 1줄 ② 고임팩트 step(소개·포트폴리오·무드) 헤더 1줄 ③ 게이지 보조 1줄. "유리/우선"은 쓰되 "보장" 금지(W4).

### 13.4 완성도 게이지 확장 (100% = 진짜 고퀄·매칭가능)
`computeListingCompleteness`를 **기본+상세+연락처+무드+포트폴리오 통합 spine**으로 확장. **업종별 가중**(스튜디오=포트폴리오↑, 네일=가격·예약↑ — R3 반영). 무드 포함 → "100%인데 매칭 0" silent gap 차단(R5/W5).

### 13.5 품질 게이트는 부드럽게 (강제 아님)
필수(이름·이미지·소개·지역·문의)만 하드(미충족 시 검토요청 비활성+무엇이 빠졌는지). 권장(상세·무드·포트폴리오)은 "건너뛰기 가능하나 노출 손해" 넛지. 1인 사장 부담·허위입력 방지(W4).

### 13.6 재사용 / 델타 (중복 금지)
**재사용**: `DETAIL_SCHEMA`·`BusinessListingContactForm`·`computeListingCompleteness`(확장)·`formDraft`(자동저장·이어하기)·`ImageUploader`·저장 RPC(`saveListing`·`upsert_my_listing_detail`·contact) **그대로**. **델타**: ① step 흐름 셸 + 가이드 카피 레이어 ② 업종별 예시·키워드칩 **단일소스 데이터**(`partnerListingGuide.ts` — 코드분기 금지, DRY/W3) ③ 완성도 spine 확장 ④ 무드 피커(S0 택소노미). **백엔드 저장경로 무변경**(UI 재배치+카피가 주).

### 13.7 구현 편입
신규 단계 **S1.5 "가이드형 상세정보 위저드"**(S1 포트폴리오·S2 대시보드 사이). 저토큰: 저장·DB 변경 최소, **재배치+가이드 카피+완성도 확장**이 핵심.

### 13.8 적대적 자기검증 → 개선 (위저드를 깐다)
- **W1 🔴 위저드 강제 = 기존 4천 업체·바쁜 사장 이탈.** → 위저드는 **신규 기본 경로**, 기존/숙련자는 **"전체 한 화면 편집"(현 `BusinessVendorEdit`) 토글 유지** — 둘이 같은 저장·완성도 공유. 단일경로 강제 금지.
- **W2 ⚠️ step 많으면 중도 이탈.** → **필수(1~4) 먼저 → "이제 노출돼요" 1차 완료 → 심화 나중**(숨고식).
- **W3 ⚠️ 업종별 예시 카피 유지비.** → 예시·키워드칩 **`partnerListingGuide.ts` 단일소스**, 코드 분기 금지.
- **W4 ⚠️ "고퀄 강요"가 허위입력·반발로.** → 필수 최소만 하드, 나머지 넛지+노출보상. "예시처럼 핵심만"(의미없는 글자수 채우기 역효과 안내).
- **W5 ⚠️ 완성도 100%인데 매칭 0(무드 누락).** → spine에 무드·상세 포함 → "100%=매칭가능" 정합.
- **W6 ⚠️ 미리보기 ≠ 실제 페이지면 신뢰 하락.** → 미리보기는 실제 `/vendor/:id` 렌더 재사용, 목업 금지.
