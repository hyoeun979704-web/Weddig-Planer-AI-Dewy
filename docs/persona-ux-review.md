## 2. surface별 검토 (마찰점 · 영향 페르소나 · 심각도)

심각도: **P0** 출시 차단/신뢰 훼손 · **P1** 핵심 흐름 마찰 · **P2** 개선 권장

### 홈 (Index.tsx + components/home/*)
- **P0 — 가짜 후기 하드코딩**: `ReviewSection`이 "처음부터 끝까지 정말 만족스러웠어요" 등 정적 데이터를 실제 후기처럼 노출. → **P18, P2, P10** 신뢰 즉시 훼손. 실제 리뷰 연동 또는 섹션 숨김 필요.
- **P1 — 페르소나 대시보드 무음 실패**: `usePersonaInsights` 등 훅이 null이면 대시보드가 통째로 사라짐(폴백 UI 없음). → **P6, P13** 핵심 가치를 못 봄.
- **P2 — 헤더/탭바 z-index 중첩**(z-50/z-40) 좁은 뷰포트 레이어 버그 가능. → **P16, P10**.

### 검색 (components/home/SearchOverlay.tsx)
- **P1 — Enter 제출이 no-op**: `handleSearchSubmit`이 최근검색 저장 후 닫기만 하고 `navigate` 없음. 키보드 Enter로 검색 불가, 결과 카드 클릭만 동작. → **P11, P10, P17** 검색 자체 실패로 인지.
- **P1 — 추천/인기 검색어 클릭이 결과로 안 감**: 칩 클릭이 query만 세팅하고 결과 뷰 전환이 안 됨 → 입력만 채워진 채 멈춤. → 전 페르소나.
- **P1 — 에러 무음**: Supabase 실패 시 스피너 무한, 재시도 버튼 없음. → **P11, P16**.
- **P2 — 인기/추천 검색어 하드코딩**(그랜드 웨딩홀 등) 트렌드 미반영. → **P18, P19**.

### 알림 (Notifications.tsx)
- **P0 — 설정이 저장 안 됨**: 5개 토글이 로컬 `useState`뿐, API/localStorage 영속화 없음. 새로고침 시 기본값 복귀, 저장 피드백도 없음. → 전 페르소나, 특히 **P7**(마케팅 동의 off 의도가 무시됨 → 동의 정책과 충돌), **P11**(껐는데 또 켜져 혼란).
- **P2 — 안내 문구 약함**: "기기 권한에 따라…"만 있어 앱 설정이 안 저장된단 사실을 가림.

### 찜 (Favorites.tsx + useFavorites.ts)
- **P1 — 파트너 전용 찜 하트 상태 모호**: 상대만 찜한 항목의 흐린 하트를 누르면 "내 목록에 추가"인지 "삭제"인지 불명확. → **P8**(공동 찜이 핵심인데 멘탈모델 깨짐).
- **P2 — 검색 토글 게이팅**: 검색창이 토글 안에 숨어 발견성 낮음. → **P12, P19**.
- **P2 — 단일 업체타입이면 서브필터 칩 미표시**(코드상 필터는 적용) → 사용자에겐 보이지 않는 필터.

### 장바구니 (Cart.tsx → Checkout → PaymentSuccess/Fail)
- 정정: 결제 흐름은 실제로 견고함(코드 재확인). `/payment/success`·`/payment/fail`·`/order-complete/:id` 라우트 모두 존재(App.tsx:229–231), `PaymentSuccess`가 `confirm-payment` 승인 후 `clearCart()` 호출(PaymentSuccess.tsx:38) 및 결제정보 누락/승인 실패 시 에러 상태+"다시 시도하기" 제공. → **P20, P15** 핵심 우려는 대체로 해소됨.
- **P2 — 결제 실패 아이콘 자리 빈 span**: `PaymentSuccess.tsx:64`에 `<span className="text-3xl"></span>` 빈 데코(이모지 제거 잔재). 아이콘 컴포넌트로 교체 권장.
- **P2 — 수량 직접입력 불가**(± 버튼만), Checkout에서 항목 수정 불가(카트로 되돌아가야 함). → **P12, P20**.
- **P2 — 비로그인 가드 위치**: Cart 자체엔 가드 없고 Checkout 진입 시 처리. 일관화 검토. → **P11**.

### 플래너 AI (AIPlanner.tsx + useAIPlanner.ts)
- 강점: 스트림 끊김 복구 카피("답변이 중간에 끊겼어요…"), 일일 한도 안내 양호.
- **P1 — 빠른질문 카드 동작 비일관**: 일부는 모달(설문), 일부는 LLM 원문 프롬프트 → 같은 그리드에서 결과 형태가 달라 예측 불가. → **P11, P17, P10**.
- **P1 — 정보입력 모달(WeddingInfoSetupModal) 스크롤/접근성**: 6단계 폼이 작은 화면에서 스크롤 컨텍스트 불명확. → **P16, P10, P7**(임신 등 민감 항목 입력 지점).
- **P2 — 업그레이드 모달이 프리미엄 혜택 미설명**. → **P15**.

### AI 스튜디오 (AIStudio + Dress/Makeup Fitting·Recommend)
- 강점: 5하트 비용을 인트로+리뷰 2회 고지, 잔액 부족 시 사전 차단, 실패 시 "하트 자동 환불" 명시. → **P15** 우려를 상당 부분 해소.
- **P1 — 결과 pending 무한 폴링**: mount 시 1회만 조회, 백엔드 지연 시 스피너 영구(타임아웃·"다시 확인" 없음). → **P13, P15**.
- **P1 — 갤러리 진입 동선 부재**: 스튜디오 홈에서 과거 결과로 바로 못 감, 새 피팅을 시작해야 갤러리 링크 노출. → **P12, P2**. (이번에 갤러리 빈 상태는 EmptyState로 통일됨)
- **P1 — 라우트 단 로그인 가드 없음**: 로그아웃 상태로 직접 진입 → 인트로 다 보고 시작 눌러야 "로그인 필요" 토스트. → **P11, P17**.
- **P2 — 생성 에러 메시지 일반적**: 무엇이 실패했는지(분석/생성/타임아웃) 구분 안 됨. → **P15, P12**.

### 꿀팁 (Tips.tsx)

## 3. 교차 패턴 (여러 surface 공통)

1. **하드코딩된 가짜 데이터 (P0, 신뢰)** — 홈 후기, 이벤트 전체, 게스트 마이페이지 통계/후기. 신뢰 민감 사용자(P18)뿐 아니라 모두에게 거짓 정보. → 실데이터 연동 또는 "준비 중" 솔직 표기로 통일.
2. **영속화/피드백 누락 (P0~P1)** — 알림 설정 미저장(가장 심각), 토글·뮤테이션 성공 피드백 부재. (결제 후 카트 초기화는 정상 동작 확인됨)
3. **에러·대기 상태 무한 스피너 (P1)** — 검색·AI결과·OrderComplete 모두 타임아웃/재시도 없음. **P11/P16**에 치명적.
4. **로그인 게이팅 비일관 (P1)** — 어떤 화면은 사전 가드, 어떤 화면은 행동 후 토스트, 어떤 버튼은 게스트도 클릭 가능. 일관된 가드+사전 안내 필요. → **P10, P11, P17**.
5. **사용자 선택이 고정 안 됨 (P1)** — 커뮤니티/꿀팁 자동필터가 수동선택을 덮음. 선호 필터 핀 기능 필요. → **P19, P3, P4**.
6. **死 UI / no-op (P1)** — 쇼핑·혜택 필터의 색상·사이즈·지역, 검색 Enter, 추천검색 칩. 동작 안 하는 컨트롤은 제거하거나 연결.
7. **접근성 (P2, 상시)** — 색만으로 상태 전달(예산 초과), 큰 폼 스크롤, 터치 타겟·대비. **P16, P10, P11**.
8. **페이지네이션 부재 (P1)** — 커뮤니티·꿀팁. 규모 확장 시 성능/탐색 문제.

## 4. 우선순위 권고

**P0 (먼저)**
- 가짜 데이터 제거/연동: 홈 후기, 이벤트, 게스트 통계·후기
- 알림 설정 서버 영속화 + 마케팅 토글을 `user_consents(marketing_v1)`와 동기화(이미 backfill 존재 → 화면도 연결)
- 스케쥴: Schedule↔MySchedule 통합 또는 명확한 상호 링크
- 커뮤니티 페이지네이션
- 예산 잔금 결제 트랜잭션 원자성

**P1**
- 무한 스피너 3종(검색·AI결과·주문완료)에 타임아웃+재시도
- 로그인 게이팅 일관화(라우트 가드 + 사전 안내)
- 자동필터 위에 수동선택 핀 보존(커뮤니티·꿀팁)
- 死 UI 제거/연결(검색 Enter, 추천칩, 필터 색상/사이즈/지역)
- AI스튜디오 갤러리 바로가기, 파트너 찜 하트 라벨 명확화

**P2**
- 접근성: 초과 경고 배너(색+텍스트), 폰트 토큰/대비, 폼 스크롤
- 예산 메인 정보구조 단순화, 맥락 없는 프리미엄 배너 조건화
- 쇼핑 이미지 캐러셀·카드 설명, 수량 직접입력
- 마이페이지 카운트 캐싱·에러 바운더리

## 5. 페르소나별 "가장 먼저 막히는 지점" 요약

- P1 알뜰신부 → 예산 잔금 트랜잭션·배분 가이드 부재
- P2 럭셔리 → 가짜 후기로 큐레이션 신뢰 저하, 상세 이미지 빈약
- P3 셀프DIY → 쇼핑 셀프탭 리셋, 커뮤니티 자동필터 덮어씀
- P4 스몰웨딩 → 꿀팁 카테고리 고착, 자동필터
- P5 주도형신랑 → (플래너 카피 중립화는 개선됨) 빠른질문 동작 비일관
- P6 위임형 → 페르소나 대시보드 무음 실패 시 가치 상실, 이벤트/혜택 분리
- P7 임신신부 → 알림 마케팅 토글 미저장이 동의정책과 충돌, 민감정보 입력 모달 접근성
- P8 장거리 → 파트너 찜 하트 상태 모호
- P9 재혼 → (커뮤니티 익명성 양호) 정보 노출 단서 명확화 필요
- P10 혼주 → 스케쥴 이중구조, 검색 Enter no-op, 정보 과부하
- P11 디지털약자 → 무한 스피너·에러 무음·게이팅 비일관·로그아웃 도달성
- P12 파워유저 → 死 필터 UI, 페이지네이션 부재, 마이페이지 잦은 재조회
- P13 초단기 → AI결과 무한대기, 날짜 자동제안 미재계산
- P14 장기준비 → 스케쥴 편집 동선 발견성
- P15 가격회의론자 → (AI스튜디오 비용고지·환불은 양호) pending 무한·에러 불투명, 업그레이드 혜택 미설명
- P16 저시력 → 색 전용 상태표시, 대비/터치 타겟, 무한 스피너
- P17 다문화 → 빠른질문 비일관, 검색 실패, 게이팅 후 토스트
- P18 신뢰민감 → 하드코딩 가짜 데이터 전반(홈·이벤트·게스트), 조회수 0
- P19 커뮤헤비 → 자동필터 덮어씀, 페이지네이션 부재, 조회수 신뢰
- P20 쇼핑충동 → "바로 구매"가 addToCart 실패를 미처리, 死 필터 UI(색상/사이즈), 상세 이미지 빈약 (결제 흐름 자체는 견고)
