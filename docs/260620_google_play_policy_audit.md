## A. 결제 — Google Play 빌링 (🔴 최우선)
**정책**: Google Play에서 배포되는 앱의 **인앱 디지털 재화·서비스 구매는 Play 빌링 사용 의무.**
가상화폐(하트), 구독, 디지털 기능이 명시 대상. **단, "획득/지급된" 포인트는** Play 빌링 없이
지급·교환 가능(= 가입 보너스·적립 하트는 무방). **"판매되는" 하트는 Play 빌링 대상.**
- **2026 변화**: Epic 합의(2026-03) 후 빌링 유연화·수수료 인하(신규 15%/기존 20%/구독 10%) 및
  외부결제 프로그램 확대. **Play 빌링 라이브러리 v8**은 2026-08-31까지 신규/업데이트 앱 의무
  (연장 시 ~2026-11-01).
- **한국 특수**: 전기통신사업법(2021)으로 **앱 내 제3자 결제(대체결제)** 허용 — 단 Google이 정한
  "개발자 제공 인앱결제(IAB)+사용자 선택" 프레임 안에서. **임의 웹 PG 우회는 별개**이며 반려 소지.

**우리 상태(확인됨)**: `src/lib/heartPackages.ts` 에 하트 패키지 **판매**(1,900~19,900원),
`/points` 충전, `/premium/subscribe` 구독. 결제는 **카카오페이 확정**(`kakao-pay-ready/approve`·
`kakao-pay-charge-*`·`kakao-pay-order-*`·`cancel-subscription` 엣지함수). 네이티브 IAP 플러그인은
없음 → **현재 네이티브 앱도 카카오페이로 흐름 = Play/Apple 정책 위반 소지**(디지털 재화).

**조치(P0)**:
1. **결제 흐름 실제 확인**: 안드로이드 앱에서 하트·구독 구매가 어떤 경로인지(웹 PG/외부브라우저/
   Play 빌링) 코드·런타임으로 확정.
2. 디지털 재화이므로 **Play 빌링 적용**(또는 한국 대체결제 프로그램 정식 신청·적용). 웹 리다이렉트
   단독은 반려 위험.
3. Play 빌링 채택 시 **라이브러리 v8**(2026-08 데드라인) 사용.
4. 구독: 가격·갱신·해지 **투명 고지**(구독 정책), 무료체험 종료 자동결제 여부 명확히
   (현재 UpgradeModal "체험 종료 후 자동 결제 없음" 문구와 실제 동작 일치 확인).

> ⚠️ 이 항목은 **Play 정책의 핵심 반려 사유**. 출시 전 반드시 현행 정책+한국 대체결제 규정으로
> 재검증. (앱 외부 웹사이트에서의 구매는 정책 영향이 다름 — 인앱 소비 디지털재화가 문제.)

## B. 생성형 AI 콘텐츠 정책 (🔴)
**정책**: AI가 요청에 따라 콘텐츠(이미지 등)를 생성하는 앱은 ① **인앱 신고/플래그**로 사용자가
앱을 벗어나지 않고 불쾌/유해 콘텐츠를 신고할 수 있어야 하고 ② 사전 **콘텐츠 필터링·모더레이션**,
③ **엄격한 사전 테스트**로 제한 콘텐츠 생성 차단, ④ AI 산출물 **라벨링**(2025 강화).
**우리 상태**: 이미지 생성 다수(드레스·메이크업·헤어·스드메·촬영시안·청첩장). **AI 결과 화면에
신고/플래그 없음**(커뮤니티 `ReportDialog`는 UGC용, AI 생성물엔 미연결).

**조치(P0)**:
1. AI 생성 결과(피팅·스드메·촬영시안·청첩장 일러스트 등)에 **인앱 신고 버튼** 추가
   (커뮤니티 `ReportDialog` 패턴 재사용 가능).
2. 프롬프트/출력 **모더레이션**(부적절 입력·결과 차단) — 엣지함수 단계에서.
3. 출시 전 **레드팀 테스트**(유해 생성 시도 차단 확인) 기록.

## C. 계정 삭제 (🟢)
**정책**: 인앱 계정 생성 가능 앱은 **인앱 계정삭제 + 웹 링크** 둘 다 제공.
**우리 상태**: ✅ `/account-deletion` 페이지 + `delete-account` 엣지함수 + Settings 진입.
**조치**: 웹에서도 동일 경로 접근 가능 확인(웹앱이라 URL 접근 가능 ✅), **Data Safety 폼의 데이터
삭제 질문**과 실제 동작 일치.

## D. Data Safety 폼 (🟡)
**정책**: 모든 제출에 필수. 수집/공유 데이터·목적·암호화·삭제요청 가능 여부를 **실제와 정확히**
일치(불일치=리뷰 플래그/반려).
**우리 수집 추정**: 계정(이메일/소셜), **사용자 업로드 사진(AI 생성용)**, 광고ID(AdMob),
사용패턴, 기기 식별자. **조치**: 항목별 정확히 선언 + 삭제 배지 위해 삭제질문 완료.

## E. 개인정보처리방침 (🟢)
✅ `/privacy`·`/terms`·`/location-terms` 존재. Play 콘솔 데이터 안전·스토어 등록정보에 방침 URL
연결 + 인앱 접근 유지.

## F. 권한 (🟢 최소)
선언 권한: `INTERNET`, `com.google.android.gms.permission.AD_ID`(광고ID). 매우 최소화 — 양호.
- **2026-04-15 신정책**: **연락처**(불필요 시 Contact Picker), **위치**(정밀위치는 location
  button 최소범위), **계정 이전**(Play 콘솔 공식 워크플로). → 우리는 연락처·위치 미사용으로 보여
  영향 적음. 사진 업로드는 시스템 피커(Capacitor camera) 사용 시 별도 미디어 권한 불요.
- 광고ID 권한 선언 시 **Data Safety에 광고ID 수집** 반드시 표기.

## G. 광고(AdMob) (🟡)
App ID 설정됨(주석상 "도입 예정"). 적용 시:
- **EEA 사용자 동의(UMP/CMP)** 필수, 광고ID Data Safety 표기, **방해성 광고**(전면광고 오남용) 금지,
- 부적절 광고 차단 설정, 보상형/전면 광고 정책 준수, 가족용 아니면 적절 연령 타게팅.

## H. UGC(커뮤니티) (🟢)
✅ `ReportDialog` 신고 기능 존재. 정책상 **신고·차단·모더레이션** 필요 — 유지. 욕설/불법 콘텐츠
처리 흐름 점검.

## I. 기타 점검
- **타깃 API**: `docs/android-target-sdk-upgrade.md`(현재 35, 17/API37 상향 계획).
- **기능 정상성/오해 소지**: 깨진 기능·placeholder CTA·과장 메타데이터 금지(Play "최소 기능" +
  내부 dead-end UI 규칙과 동일). 출시 전 페르소나 walkthrough.
- **딥링크/지적재산**: 청첩장 템플릿·이미지 저작권, 타사 상표 미오용.
- **민감정보**: 결혼=생애이벤트지만 건강/금융 민감카테고리는 아님(예산은 사용자 자기기록).
