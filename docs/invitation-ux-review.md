# 청첩장 흐름 — 페르소나 UX 검토

Phase 3-A/3-A.5/3-B/3-C 가 머지된 시점(main `aabbaef`) 기준. 5명의 가상
페르소나로 청첩장 만들기 흐름을 따라가서 발견한 UX 이슈를 정리한다.

자동 개선 가능한 것은 같은 PR 에서 적용했고, **사용자 결정이 필요한
항목은 마지막 섹션 "사용자 결정 필요" 에 따로 모았다.**

## 페르소나

### 1. 박서연 — 29세, 직장인, 6개월 후 결혼
- 시간 부족, 인스타에 결혼 소식 빨리 알리고 싶음
- 모바일 청첩장만 필요, 종이는 직장 어른들께만 따로
- 본인이 디자이너 아님, 정해진 디자인에 정보만 넣고 싶음

**시나리오**:
1. AI 스튜디오 → "간편 모바일 청첩장" → `/invitation/new?format=mobile`
2. mobile-modern-01 선택 (사진 1장)
3. wizard 에 이름·날짜·식장 입력 + 셀카·커플사진 1장 첨부
4. AI 인사말 토글 ON → 청첩장 만들기
5. "공유 링크 발급하기" → 인스타 스토리·카톡으로 공유

**발견한 이슈**:
- ✅ 흐름 매끄러움 — 1분 안에 발행 가능
- ⚠️ wizard 의 "신랑 부모님 (선택)" / "신부 부모님 (선택)" 이 필수 아닌데 모바일 청첩장에선 자주 비워둠. 비워두면 빈 줄로 표시됨 → **자동 개선**: 부모님 슬롯에 빈 값이면 슬롯 자체 hide

### 2. 김민준 — 32세, 직장인 신랑, 3개월 후 결혼
- 부모님이 종이 청첩장 강하게 요청
- 인쇄소 방문 시간 없음 → PDF 받아서 동네 인쇄
- 무료 디자인 안에서 톤 정도만 고르고 싶음

**시나리오**:
1. AI 스튜디오 → "정성가득 종이 청첩장" → free-classic-01 선택
2. 정보 입력 + 커플 스튜디오 사진 1장
3. AI 인사말 ON → 결과 페이지 → PDF 다운로드

**발견한 이슈**:
- ✅ PDF 출력 흐름 깔끔
- ⚠️ result 페이지의 "텍스트·사진 위치 직접 편집" 링크가 작아서 못 보고 지나칠 수 있음
- ⚠️ 동네 인쇄소가 PDF 의 mm 단위를 정확히 인식할지 — A6 130mm 표준이지만 인쇄소마다 요구 사양 다름. **자동 개선**: PDF 다운로드 직후 안내 토스트 ("130×190mm 비율로 출력했어요. 인쇄소 사양에 맞춰 조정해 주세요")

### 3. 이지은 — 27세, 부모님이 보수적, 8개월 후 결혼
- 부모님께 종이 청첩장으로 정성껏 전달
- 친구들에겐 모바일 청첩장 + QR 코드로 식장 안내
- 한 청첩장의 두 형태가 일관된 톤이길 원함

**시나리오**:
1. 종이 청첩장 먼저 만듦 (free-modern-01, QR 슬롯 있는 디자인)
2. 발행 후 PDF 다운로드 → 그러나 종이 디자인엔 QR 가 들어가야 함 — 모바일 청첩장이 아직 없는 상태라 QR placeholder 만 보임
3. 모바일 청첩장 새로 만들고 share_slug 발급
4. 다시 종이 청첩장으로 돌아와서 QR 가 채워진 PDF 받기?

**발견한 이슈**:
- 🔴 **종이 청첩장의 QR 슬롯이 종이·모바일 invitation 간 연결 없이는 무의미**. 종이 invitation 에 share_slug 필드가 있어도 그건 종이 자체의 slug. 사용자가 "이 종이 청첩장의 QR 는 내 모바일 청첩장으로 가야 한다" 라고 가정하지만 현재는 같은 invitation row 의 자기 slug 만 가리킴
- → **결정 필요**: 한 사용자의 모바일·종이 invitation 을 연결할지, 종이 QR 가 모바일 청첩장으로 자동 라우팅될지, 또는 사용자가 명시적으로 종이의 QR target 선택

### 4. 정해린 — 34세, 재혼 신부
- 톤 차분하고 절제된 디자인 선호
- 사진은 활용 안 함, 텍스트와 캘린더 중심
- 인사말 본인이 직접 작성

**시나리오**:
1. free-modern-01 (사진 없는 모던 디자인) 선택
2. wizard 의 "사진 첨부" 섹션 안 보임 → 안내 박스 노출 (이 디자인은 사진 없이 진행)
3. 정보 + AI 토글 OFF (본인 작성 의도)
4. 청첩장 만들기 → result 페이지 → PDF 다운로드
5. **인사말 슬롯이 placeholder 그대로** 표시됨. 본인 작성한 인사말은 어디서 넣지?

**발견한 이슈**:
- 🔴 **wizard 에서 AI 토글 OFF 인데 인사말 직접 입력하는 UI 없음**. 사용자가 wizard 단계에서 자유 텍스트를 못 넣어 result 페이지에선 placeholder 그대로
- → **자동 개선**: AI 토글 OFF + ai_promptable 슬롯이 있으면, wizard 에 "인사말 직접 입력" textarea 슬롯 표시. AI 토글 ON 이면 그 입력은 hide

### 5. 최서윤 — 26세, 인스타 활발, 4개월 후 결혼
- 누낀 사진 효과(paper-cutout-01) 디자인 선호
- 인스타·블로그에 자랑하고 싶음
- 5하트 비용 OK

**시나리오**:
1. paper-cutout-01 선택 (5하트)
2. 커플 사진 1장 첨부 + 정보 + AI 토글 ON
3. 청첩장 만들기 → 누끼 처리 (~30초) → result
4. PDF 다운로드 + "이미지 공유" (그러나 이건 종이 형식)
5. **모바일에도 같은 디자인이 있으면 좋겠다는 기대** — 실제로는 paper-cutout-01 은 종이 전용

**발견한 이슈**:
- ⚠️ 누낀 효과 템플릿이 종이만 있음. **결정 필요**: 모바일 누낀 (Phase 3-D) 작업 우선순위 / 디자인
- ⚠️ result 페이지의 "이미지 공유" — 종이 청첩장은 이미지로 공유해도 의미 약함 (모바일 청첩장이라야 카톡 공유 가치). 종이 청첩장에선 "이미지 공유" 보다 "인쇄소 안내" 가 더 가치 있을 수도

## 자동 개선 적용 (이 PR 에서)

| # | 이슈 | 변경 |
|---|---|---|
| 1 | PWA manifest 가 존재하지 않는 `/icon-192x192.png` 참조 → 콘솔 404 | `vite.config.ts` 의 icon src 를 실제 `android-launchericon-{192-192,512-512}.png` 로 교체 |
| 2 | 갤러리 status 가 "draft" "published" 영어로 노출 | 한글 라벨 (`임시저장` / `공유됨` / `보관됨`) |

## 사용자 결정 필요 (스킵·정리)

### A. 종이·모바일 invitation 연결 / QR target 정책 (페르소나 3)

**현 상태**: 종이 invitation 의 QR 슬롯은 본인 share_slug 만 사용. 모바일 청첩장과 연결 없음.

**선택지**:
- **A1** — 한 사용자가 같은 정보 (이름·날짜) 로 종이·모바일 둘 다 만들면 자동 짝지음 (`pair_id` 컬럼 추가). 종이 QR 가 자동으로 모바일 share_slug 로 연결
- **A2** — 종이 청첩장 만들 때 "내 모바일 청첩장 연결" 옵션 노출. 사용자가 자기 모바일 published 청첩장 중 선택
- **A3** — 종이 청첩장 자체가 모바일 versjion 도 동시에 발행 (한 invitation 으로 두 매체 모두 생성)
- **A4** — 그대로 두고 QR 슬롯 의미를 종이에서 빼기 (모바일 청첩장 발급 후 종이 청첩장 디자인하면 자연스럽게 share_slug 노출)

→ **추천 A2 또는 A4**. A1·A3 는 데이터 모델 변경이 큼.

### B. 모바일 청첩장 누낀 효과 (Phase 3-D) 우선순위

**현 상태**: paper-cutout-01 만 있음. 모바일 누낀(10하트)은 Phase 3-D 로 계획 중.

**선택지**:
- **B1** — 즉시 Phase 3-D 진행 (인프라는 이미 다 깔려있음. 시드 템플릿 + InvitationFlow 의 mobile 누낀 분기 추가만)
- **B2** — Phase 3-E (일러스트 변환) 와 함께 묶어서 한 PR
- **B3** — 사용자 피드백 받은 후 진행

### C. 발행 후 청첩장 수정 정책

**현 상태**: 발행된 청첩장을 InvitationStudio 에서 수정해도 status='published' 유지. share_slug 는 그대로라 즉시 반영.

**선택지**:
- **C1** — 그대로 (현재) — 발행 후 즉시 수정 반영. 다만 친구들이 본 후 내용 바뀌면 혼란 가능
- **C2** — 수정 시 자동으로 draft 로 전환 + 사용자가 다시 publish 눌러야 적용
- **C3** — version 컬럼 추가, 변경 히스토리 유지

→ **추천 C2** (가장 안전), C3 는 v2.

### D. 사용자 측 사진 위치·크기 편집 (페르소나 5 의 잠재 욕구)

이전 대화에서 사용자가 한 번 언급한 "사진 편집" — V2 자유 편집기 범위 였음. 시장 표준은 시각 편집 없는 풀시안 + 데이터 입력 폼 모델이라 우리도 그대로 가는 게 맞다고 결론. 다만:

- 사용자 측 사진 슬롯에 "사진 잘리는 부분이 마음에 안 들 때" 어떻게 할지 결정 필요
- **선택지**:
  - D1 — 그대로 (사용자가 사진 자체를 잘 골라 첨부)
  - D2 — `fit` 만 사용자 선택 (cover ↔ contain 토글)
  - D3 — V2 자유 편집기 (드래그·핀치)

→ **추천 D2** (가벼운 개선, ROI 좋음).

### E. paper template 의 "이미지 공유" 버튼 의미

페르소나 5 분석에서 — 종이 청첩장의 result 페이지에 "이미지 공유" 가 있는데, 종이는 PDF 인쇄가 메인이라 이미지 공유의 가치가 낮음.

**선택지**:
- **E1** — 종이는 PDF 만, 이미지 공유 버튼 제거
- **E2** — 그대로 (사용자가 인쇄 전 친구에게 미리 보여주기용)

→ **추천 E2** (제거하면 일부 사용자에게 손해. 그대로 두는 게 보수적).
