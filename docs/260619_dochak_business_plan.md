# 1. 문제 인식 (Problem)

## 1-1. 창업아이템의 배경 및 필요성

**(1) 시장 배경 — 고비용·고관여 의사결정인데 정보는 표준화되어 있다**

한국의 결혼 준비는 1인당 평균 **6~12개월**, 비용 **3,000만~5,000만원**(통계청 인구동향조사·결혼정보업체 자료)이 드는 대표적 고관여 의사결정 영역이다. 그러나 기존 플랫폼(웨딩북·아이웨딩·요즘웨딩 등)은 **'표준 결혼식' 사용자를 전제로 동일한 정보를 일괄 제공**하여, 다음 사용자층이 사실상 방치된다.

**(2) 해결해야 할 문제 (Pain Point)**

| 핵심 문제 | 영향 받는 사용자 |
|---|---|
| 정보 과다 → 의사결정 피로 | 표준 결혼식 준비자(약 50%) |
| 비표준 케이스 가이드 부재 | 스몰·셀프·노웨딩(약 30%) |
| 상황 특수성 미대응 | 임신·재혼·국제결혼·1인 진행(약 15%) |
| 지역·예산별 맞춤 정보 부재 | 지방 거주자·저예산층 전반 |
| 결과물(드레스·메이크업) 사전 확인 불가 | 전 사용자 — 고가 계약 전 시각 정보 부재 |

**(3) 기술·사회 변화가 만든 기회**

- **AI 상용화**: LLM·이미지 생성 모델(gpt-image, Gemini)이 상용 수준에 도달했으나 **결혼 도메인에 특화·신뢰성 검증을 적용한 서비스는 부재**.
- **소비 행태**: 모바일 기반 의사결정 70%+, 시각자료·후기 중심.
- **사용자 구조 변화**: 스몰웨딩·셀프웨딩이 2015년 대비 **3~4배 증가** — 표준형 정보의 효용 하락, 초개인화 수요 급증.

→ **"AI로 각 사용자에게 맞는 정보만 골라주고, 결과물을 미리 보여주는" 초개인화 버티컬 AI**의 필요성이 분명하다.

## 1-2. 목표시장(고객) 현황 분석

| 시장 | 규모(추정) | 산정 근거 |
|---|---|---|
| TAM (디지털 웨딩 도구) | 약 1,500~2,000억원 | 연간 혼인 19~20만건 × 1인당 디지털 지출 30~80만원 |
| SAM (AI·플래너 디지털) | 약 200~400억원 | TAM 중 AI·플래닝 도구 세그먼트 |
| SOM (1년차 도달 가능) | 약 3~5억원 | 모바일·SEO·커뮤니티 유입 가능 세그먼트 |

**핵심 고객**: ① 비표준 케이스(스몰·셀프·임신·재혼 등) — 기존 서비스 사각지대, ② 정보 피로를 겪는 표준 준비자, ③ 지방·저예산층. **확장 고객**: 결혼 완료 후 신혼·기념일·육아 라이프스테이지로 자연 이동.

# 2. 실현 가능성 (Solution)

## 2-1. 창업아이템의 현황(개발 준비 정도)

**이미 '출시 완료' 상태에서 도약을 신청한다** (2026.05.22 라이브). 추측이 아니라 실물로 검증된 제품이다.

| 영역 | 구축 현황 |
|---|---|
| 서비스 | dewy-wedding.com 웹·PWA 정식 운영 |
| 규모 | 프론트엔드 110개 페이지 / 라우트 112개 / DB 마이그레이션 76개 |
| AI | AI 기반 Edge Function 13종(플래너·이미지생성·웹검색·콘텐츠) |
| B2B | 사업자 콘솔(가입→인증→정보등록→검수→대시보드→정산) 완비, **제휴업체 36곳 입점** |
| 기반 | 인증·결제(카카오페이)·권한 3계층·SEO/SSR(메타·OG·JSON-LD·sitemap) |

**핵심 기술 자산(기술성·혁신성)**

1. **L1~L5 다층 환각차단 RAG 아키텍처(자체 설계)** — ① 의도 분류 후 DB 직답(LLM 미호출) ② 16개 시도×7항목 평균가·실업체 데이터 근거주입("목록 밖 생성 금지") ③ 모든 금액에 출처태그 강제 ④ 출력 후처리 감사 ⑤ 사용자 메모리 검증 UI. → **"LLM에 맡기지 않고 구조적으로 환각을 차단"** = 규제·신뢰성 도메인으로 이전 가치가 큰 IP.
2. **20모드 페르소나 초개인화 엔진** — 임신·국제결혼·재혼·스몰·셀프·1인진행 등 상황별 + 성향별 분류(결정적·멱등). DB 트리거와 클라이언트 동기화.
3. **멀티모달 이미지 생성 파이프라인** — 사용자 사진+레퍼런스 합성(드레스·메이크업·헤어 시뮬). e-커머스 가상시착으로 확장 가능.
4. **ML 분류(KLUE-RoBERTa)** — 콘텐츠 토픽 멀티라벨 분류(규칙→신경망 진화).
5. **멀티프로바이더 추상화 + 평가 인프라** — OpenAI/Gemini 교체 용이, 프롬프트 캐싱(입력비 ~50%↓), A/B 평가(eval) 자동화.

## 2-2. 실현 및 구체화 방안 (협약기간 내 개발 목표)

| 과제 | 협약기간 내 목표 | 산출물 |
|---|---|---|
| AI Studio 라인업 완성 | 출시 1종 → 6종(드레스·메이크업·헤어·촬영시안·청첩장·식전영상) | 시뮬레이션 6종 운영 |
| 환각차단 고도화 | 2-tier 라우팅 본격화, 의도 라우터 커버리지 40%+ | 운영비·환각율 추가 절감 |
| ML 분류 배포 | KLUE 모델 프로덕션 적용 | 자동 큐레이션·모더레이션 |
| 데이터 자산 확대 | 지역·업체·후기 데이터 정합성 강화 | 추천 정확도 향상 |
