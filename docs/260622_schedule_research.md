# 결혼 준비 일정(추천일정) 리서치 — 다방면 자료 수집 (2026-06-22)

> 목적: 일정 개인화(P1) 강화를 위한 **외부 자료 다방면 조사**. 한국 결혼 준비 표준 타임라인 +
> 업체 리드타임/성수기 + 법·행정(혼인신고) + 페르소나별(임신·스몰·셀프·재혼) 변수 + 경쟁 서비스
> 패턴. 출처는 §끝. ⚠️ 패턴·사실관계만 참고(복사 금지). 현 코드: `src/lib/schedule.ts`(5단계),
> `src/lib/personaPlanProfile.ts`(페르소나 델타).

## 0. 한눈에 — 현 코드 대비 보강 포인트

현 `STANDARD_PHASES`(5단계×4할일)에 **빠진 핵심 태스크**가 리서치로 드러남:
- **상견례**(6~10개월 전) — 현 일정에 없음. 양가 방향 합의가 식장/플래너 상담 **전**에 와야 함.
- **예단·예물 협의**(상견례 전 부부 사전 조율) — 현 "예물 선택"만 phase-3에 있음(시기·맥락 부족).
- **본식 사진·영상(DVD) 예약**, **드레스 가봉**(D-1~1.5개월), **혼인신고 서류**(셀프·노웨딩 필수).
- **하객 인원 확정 → 식대·답례품 수량**(스몰·식당형에서 특히 중요).

## 1. 표준 타임라인 (업계 합의 — 12개월 기준)

| 시기 | 핵심 태스크 | 근거 |
|---|---|---|
| **D-12~10개월(이상)** | 예산 큰 틀·결혼 시기 확정 / **웨딩홀 조사·투어·계약**(성수기·토요일 프라임은 1년~1년반 전) / **상견례**(양가 방향 합의) | 웨딩홀 인기·성수기 1년+ 전 마감; 상견례 6~10개월 전 |
| **D-9~6개월** | **스드메 계약**(스튜디오·드레스·메이크업) / 본식 사진·영상 예약 / 예단·예물 부부 협의 후 부모 조율 / 신혼여행 큰 틀 | 스드메 계약 6~9개월 전 가장 많음 |
| **D-6~3개월** | 웨딩 촬영 진행 / 청첩장(모바일+실물) 제작·발송 / 하객 명단 정리 / 신혼집·혼수 진행 / 식순(입장·서약·축가·퇴장) 확정 | 청첩장·촬영·하객정리 3~6개월 전 |
| **D-2개월~2주** | **드레스 가봉**(체형 변화 고려 1~1.5개월 전) / 본식 리허설·동선 / **하객 인원 확정→식대·답례품 수량** / 답례품 준비 | 가봉 식 1개월~1.5개월 전 |
| **D-2주~당일** | 최종 점검 / 잔금 정산(예식 당일) / 당일 준비물·동선 | 잔금 보통 예식 당일 |

> **결제 흐름**: 웨딩홀 계약금 ≈ 총비용 10~20%(평균 ~300만원), **잔금은 예식 당일**. 스드메 패키지
> 중간가 ~290만원. → 예산/일정 연계 시 "계약금 시점 ≠ 잔금 시점" 반영 여지(P2 예산과 연결).

**현 5단계와 매핑**: phase-1(시작)에 **상견례** 추가, phase-2(계약)에 **본식 사진·영상 예약** 추가,
phase-3(혼수·예물)에 **예단/예물 협의** 맥락화, phase-4(허니문·청첩장)에 **하객 인원 확정** 추가,
phase-5(최종)에 **가봉(1~1.5개월 전)**·잔금 정산 추가. (※ base 변경은 전 사용자 영향 — §5 결정 필요.)

## 2. 페르소나별 변수 (리서치 근거 — personaPlanProfile 보강안)

### 임신 (차수별 — 의료 근거 강화)
- **입덧**: 4~5주 시작 → **11~13주 최심** → 14~16주 지속 → **22주경 호전**. 안정기 전 무리한 일정 회피.
- **초기(1st)**: 외관 변화 적어 준비 자체는 수월하나 태아 미정착 → 과로·스트레스 회피, 사전 조사 위주.
- **안정기(2nd, ~20주 전후)**: 임신 티 거의 안 남 → **본식 촬영·가봉 적기**(집중 배치).
- **만삭(3rd)**: 드레스 **디자인보다 착용감**, **식순 간소화**, **당일 휴식 공간 확보**, 사진 촬영 조절,
  **신혼여행 안정기/출산 후로 미루거나 육로 간소화**(항공 제약).
- → 현 `pregnancyProfile` 차수 분기와 정합. 보강: 만삭에 "휴식 공간 확보" 명시, 중기에 "촬영 적기" 근거 강화.

### 스몰 (형식·규모 변수)
- **레스토랑/식당형 스몰웨딩** 트렌드(호텔 레스토랑 포함). 20~40명, 800만원~ 사례.
- **인원 파악이 핵심**: 식대는 보증인원 기준 — 적게 잡으면 하객이 식사 못 하고, 많이 잡으면 전액 부담.
  → 스몰 일정에 **"하객 인원 확정 → 보증인원·답례품 수량"**을 강조(현 small_intimate 규모 분기와 정합).
- 직접 준비 항목 多(대관·음식·의상) → 셀프 성격 짙음.

### 셀프·노웨딩 (혼인신고 행정)
- **혼인신고 준비물**: 혼인신고서 1부 + 양 당사자 신분증 + **증인 2인 서명** + 가족관계/기본/혼인관계증명서.
  시·군·구청 또는 주민센터, **평일 09~18시**(주말·공휴일 접수 불가). 당사자 1인 신고 가능.
  → 현 SELF/NO_WEDDING 의 "혼인신고 서류 확인/접수"를 구체화(증인 2인·증명서·평일 방문).

### 재혼·작은 가족식
- 가족·친지 **식사로 갈음**하거나 소규모 식. 인원 파악·식대가 동일하게 중요.
- → 현 REMARRIAGE(예물↓·혼인신고·가족 자리배치)와 정합. 보강: "하객 인원 확정→식사 규모".

## 3. 경쟁 서비스 패턴 (기능 벤치마크 — 복사 아님)

- **아이웨딩**: 체계적 웨딩 체크리스트 + 예산 관리.
- **요즘웨딩**: **배우자 연동 · 진행률 · 디데이 관리 · 맞춤 추천**(올인원).
- **웨딩북**: 결혼준비 필수 앱(체크리스트·정보).
- → Dewy 는 이미 커플 연동·진행률·디데이·페르소나 맞춤을 보유. **차별점은 "페르소나별 추천일정"**
  (경쟁 앱은 대체로 단일 표준 체크리스트). 본 P1 이 그 차별화를 강화.

## 4. 도출된 개선안 (우선순위)

1. **표준 추천 태스크 보강(전 사용자 — 표시 전용)**: STANDARD_PHASES `defaultTasks` 에 상견례·본식
   사진/영상 예약·가봉·하객 인원 확정·혼인신고(해당 페르소나)를 추가/명확화. **표시만**이라 회귀 0.
2. **personaPlanProfile 근거 강화**: 임신 만삭 "휴식 공간", 스몰 "보증인원·답례품 수량", 셀프 "증인
   2인·증명서·평일" 등 리서치 디테일 반영(이미 골격 있음 — 문구·정확도 향상).
3. **(후속) 예산-일정 연계**: 계약금/잔금 시점 차이를 P2 예산에 반영(숨은비용·결제 타이밍 경고).

## 5. 결정 필요 (구현 전)

- **표준 phase 태스크 보강 범위**: ① 모든 사용자의 "추천 할 일"에 상견례·가봉 등 추가(표시 전용,
  회귀 0) vs ② 페르소나 델타로만. → **①(표준 보강) 권장** — 상견례·가봉은 거의 모든 결혼에 공통.
- **base STANDARD_PHASES 직접 수정 vs 별도 추천 레이어**: 시드 로직(신규 사용자 user_schedule_items)에
  영향 주는지 확인 필요(현재 시드가 defaultTasks 를 쓰는지). 영향 있으면 추천 표시 전용 레이어로 분리.
