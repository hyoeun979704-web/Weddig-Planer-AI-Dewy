# 1. 문제 인식 (Problem)

## 1-1. 창업아이템의 배경 및 필요성 — "결혼의 진입장벽" 문제

**(1) 시장 배경 — 고비용·고관여인데 정보는 표준화되어 있고, 그 부담이 결혼을 늦춘다**

한국 결혼 준비는 1인당 평균 **6~12개월·3,000만~5,000만원**(통계청·결혼정보업체)이 드는 대표적 고관여
의사결정이다. 그런데 ① 비용·정보 부담 자체가 **결혼 지연·포기의 사회적 장벽**이 되고 있고(국토연구원:
주거·결혼 비용 부담이 혼인·출산에 직접 악영향), ② 기존 플랫폼(웨딩북·아이웨딩·요즘웨딩)은 **'표준
결혼식' 사용자를 전제로 동일 정보를 일괄 제공**해 비표준·정보피로 사용자층이 방치된다.

**(2) 해결해야 할 문제 (Pain Point)**

| 핵심 문제 | 영향 사용자 | Dewy의 해결 |
|---|---|---|
| 정보 과다 → 의사결정 피로 | 표준 준비자(~50%) | 20모드 페르소나로 **필요한 정보만 선별 노출** |
| 비표준 케이스 가이드 부재 | 스몰·셀프·노웨딩(~30%) | 상황별 전용 페르소나·체크리스트 |
| 상황 특수성 미대응 | 임신·재혼·국제·1인(~15%) | 임신/재혼+자녀/국제결혼 전용 모드 |
| 지역·예산별 맞춤 부재 | 지방·저예산층 | 16지역×7항목 평균가 + 4,378 업체 큐레이션 |
| 결과물 사전확인 불가(고가계약 전) | 전 사용자 | **AI 이미지생성**(드레스·메이크업·헤어·촬영) |

**(3) 기술·사회 변화가 만든 기회**

- **AI 상용화**: LLM·이미지 생성(gpt-image·Gemini)이 상용 수준 도달, 그러나 **결혼 도메인 특화 +
  신뢰성 검증을 적용한 서비스는 부재**.
- **결혼 시장 회복기**: 2024년 혼인 **22.2만 건(+14.8%, 13년 만의 첫 두 자릿수 반등)**, 미혼 결혼의향
  **62.2%(2021 대비 +11.4%p)**, 25~29세 여성 57.4%(+9.2%p) — **수요가 다시 늘고 있는 타이밍**.
- **사용자 구조 변화**: 스몰·셀프웨딩 2015년 대비 3~4배 — 표준형 정보 효용 하락, 초개인화 수요 급증.

→ **"AI로 각자에게 맞는 정보만 골라주고 결과물을 미리 보여줘 결혼 준비의 비용·불안·정보비대칭을 낮추는"
초개인화 버티컬 AI**의 필요성이 분명하다.

**핵심 기술 자산(기술성·혁신성)**

1. **L1~L5 다층 환각차단 RAG(자체 설계)** — `supabase/functions/ai-planner/`. ①의도 분류 후 DB 직답(LLM
   미호출, 상위 30~40%) ②16지역×7항목 평균가·실업체 근거주입("목록 밖 생성 금지") ③모든 금액 출처태그
   강제 ④출력 후처리 감사 ⑤메모리 검증 UI. → **"LLM에 맡기지 않고 구조적으로 환각 차단"** = 금융·의료 등
   신뢰성 도메인 이전가치 큰 IP.
2. **20모드 페르소나 초개인화 엔진** — `weddingPersona.ts` 단일 레지스트리(임신·국제·재혼·스몰·셀프·1인 등),
   결정적·멱등, DB 트리거 parity, **18개 테스트**로 reachability·priority·parity 검증.

# 3. 성장 전략 (Scale-up)

## 3-1. MAU 성장성 — "검증된 퍼널을 마케팅으로 증폭"

**(1) 현재 퍼널(무마케팅 오가닉, §0-B 실측)**

```
누적 49명 ─→ MAU 49 ─→ WAU 19(39%) ─→ 유료 전환(하트구매 28건·포인트 192건)
  ↑광고비 0원      ↑리텐션 양호        ↑결제 실작동 검증
```

- **해석**: 작은 표본이지만 **가입→재방문→결제**의 전 구간이 **광고 없이** 작동. 즉 "구멍 난 퍼널에 돈을
  붓는" 단계가 아니라 **"작동하는 퍼널을 채널로 키우는"** 단계다.

**(2) 성장 엔진 (자금 투입 시)**

| 채널 | 레버 | Dewy 고유 강점 |
|---|---|---|
| **SEO/AIO(오가닉)** | 16,131 콘텐츠 + 4,378 업체 페이지 = **거대한 검색 surface** | 이미 색인 자산 보유 → 한계비용 0에 가까운 유입 |
| **마케팅 자동화** | 블로그·인스타 카드뉴스 AI 생성 파이프라인(원소스 6채널) | 콘텐츠 생산 단가↓ → 같은 예산으로 더 많은 노출 |
| **퍼포먼스 광고** | CAC≤LTV/3 검증 채널 집중 | 페르소나별 카피 변형으로 타깃 효율↑ |

## 3-2. 시장성 — "쇠퇴 시장이 아니라 회복·구조변화 시장"

- **수요 회복**: 2024 혼인 +14.8%(13년 만), 결혼의향 62.2% — 표적 시장이 **다시 커지는 중**.
- **구조 변화 = 우리의 기회**: 스몰·셀프·비표준 결혼 급증 → 표준형 경쟁사가 못 잡는 세그먼트를 선점.
- **디지털 전환**: 결혼 준비 의사결정 70%+ 모바일 → 디지털 웨딩 도구 TAM 1,500~2,000억, AI 세그먼트 미성숙(공백).
- **확장성**: 동일 AI 엔진으로 신혼·육아·인접 vertical(숙박·뷰티) 이전 → SAM을 넘는 멀티도메인 천장.

## 3-5. 사회환원성(ESG) — "결혼의 장벽을 낮춰 저출산 국가 의제에 기여"

> 정부·공공 심사에서 가장 강력한 축. Dewy의 미션은 사회적 가치와 사업적 가치가 일치한다.

- **혼인율 → 출산율의 선행지표**: 한국은 출생의 절대다수가 혼인 가구에서 발생 → **혼인 증가는 1.5~2년
  시차로 출산 증가로 이어진다**(2024 혼인 반등이 2025~2026 출산율 반등 기대의 핵심 동인). 결혼의 비용·정보·
  심리 장벽을 낮추는 서비스는 **저출산 대응의 입구(funnel top)에 직접 작용**한다.
- **결혼 비용·정보비대칭 완화**: 평균가 데이터·페르소나 큐레이션·결과물 사전확인으로 **과지출·정보부족으로
  인한 결혼 지연/포기 요인**을 줄인다(국토연구원: 비용 부담이 혼인·출산에 악영향).
- **지역 균형**: 충남 천안 소재 1인·여성 창업. 지방 거주자·저예산층을 위한 맞춤 정보로 **수도권 집중 정보
  격차 해소**. 지역 소상공인 웨딩업체 디지털화 지원.
- **여성·1인 창업 모델**: 여성 대표가 AI 자동화로 1인 운영하는 성장 모델 제시 → 창업 생태계 다양성 기여.

> **상담 메시지**: "정부가 수십조를 쓰는 저출산 대응의 가장 앞단은 '결혼'이다. Dewy는 결혼 준비의
> 비용·불안·정보 장벽을 AI로 낮춰 **혼인율 회복을 보조**한다 — 사회적 미션과 BM이 같은 방향이다."

## 4-2. "문제해결에 진심을 다한" 실행력 — 정량·정성 증거

> 단순 '개발'이 아니라 **검증·문서화·자동화의 엔지니어링 체계**를 혼자 구축했다. 모두 레포에서 확인 가능.

**(1) 검증에 대한 집요함 — 실패를 기록하고 규칙으로 만든다**
- `docs/verification-lessons.md`(169줄) — 실제 발견·수정한 **회귀 8건**을 원인·영향·교훈으로 정식 기록. 예:

  - **업체 승인 100% 불능**(RPC 인자 불일치 PGRST202) → "RPC 시그니처 교차검증" 규칙화.
  - **dead-end CTA**(미입점 업체 '문의'가 토스트만 — 4천여 곳 영향) → 감사에 'dead-end UI' 차원 신설.
  - **검색 0건**(label "충남" vs value "충청남도" 불일치) → "표시값 ≠ 매칭값 분리" 규칙화.
- **태도**: "작동한다 ≠ 검증됨" — e2e 불가 시 "검증함" 보고를 금지하고 한계를 명시하는 자기 규율.

**(2) 스스로 만든 14차원 품질 게이트** — `AGENTS.md`(멀티툴 단일 소스). 모든 코드 변경을 정확성·보안·성능·
테스트·유지보수·아키텍처·**iOS/사파리·출시적합성(스토어)·안정성·법규·비용·접근성·개인정보·초개인화** 14개
관점으로 자기검증. 단순 가이드가 아니라 회귀 재발을 구조적으로 차단하는 게이트.

**(3) 지속적 감사 자동화** — `.github/workflows/`에 **주간 전체감사(weekly-audit)·월간 e2e
시뮬레이션(monthly-e2e-simulation)·일일 에러 트리아지(daily-error-triage)** 구축. "매번 빼먹는" 문제를
사람 의지가 아니라 **시스템**으로 해결.

**(4) 분석·기획의 깊이** — `docs/` **195개 문서**(코드리뷰·보안감사 **63건**, 경쟁사 분석 3건, 페르소나 UX

**(5) 완성도에 대한 집착** — 청첩장 템플릿을 눈대중이 아니라 **좌표 그리드 수치 측정**으로 레퍼런스와 정밀
대조, 환각차단을 L1~L5로 다층 설계, 페르소나를 20모드 단일 레지스트리 + 테스트로 관리.

> **요약**: 빌드 통과를 완료로 보지 않고 → e2e·실DB 검증 → 경쟁사 대조 → 회귀 기록 → 규칙 갱신 →
> 자동화까지 가는 루프를 **혼자, 반복적으로** 돌렸다. 이것이 "진심을 다한 노력"의 실물 증거다.

## 4-3. 채용 및 외부 협력

- **채용(자금 투입 시)**: 개발 1명·마케팅 1명 — 고용 창출 + 1인 의존 리스크 완화.
