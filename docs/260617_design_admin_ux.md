## ① 업체정보 수정 — 전체 확인/수정 불가 + 단일 가격
**현황**
- 운영자 `AdminPlaceEdit.tsx`: 보여주는 필드 = 업체명·대표이미지·시도·구군·위경도·**최소가격(min_price)**·설명·태그 **8개뿐**(L93 select). 카테고리별 상세(place_details)·상품(business_products)·이벤트·쿠폰·포트폴리오·문의채널 **미표시**.
- 기업 `BusinessVendorEdit.tsx`: 기본정보 + `BusinessListingDetailForm`(카테고리 상세) + 문의채널은 있음. 상품/이벤트/쿠폰/사진은 **각 전용 페이지**(탭 분리).
- 가격: `places.min_price` **단일**. 실제 상품가는 `business_products`(상품별)에 따로.

**문제**: 운영자가 업체 "전체"를 한 화면에서 못 봄/검수 못 함. min_price가 "그 업체의 가격"처럼 보여 다상품과 안 맞음("노답").

**설계안**
- (A) `AdminPlaceEdit`에 **읽기전용 요약 패널** 추가: 이 업체의 상품 N개(이름·가격)·이벤트·쿠폰·포트폴리오 수 + 상세(place_details) 표시. 편집은 핵심 필드 유지(운영자는 검수·교정 중심).
- (B) `min_price` 라벨 → **"최소가(시작가)"** + 도움말 "개별 상품 가격은 상품 탭에서 관리". 기업 폼·소비자 표기도 "최저가~"로 일관.
- ❓**결정**: 운영자가 상품/상세까지 **편집**도 해야 하나, **보기(검수)**만 충분한가?

## ② 사용자 관리 ↔ 마이페이지 정보 불일치
**현황**: `AdminUsers`가 `nickname: p.display_name`(L106)로 매핑 → "닉네임" 컬럼에 **실명(display_name)**을 표시. 사용자가 실제 설정하는 커뮤니티 닉네임은 **`community_nickname`(별도 필드)**. 등급(member_tier)·소속은 동일 출처(profiles/RPC).

**문제**: 운영자가 보는 "닉네임"(=실명)과 사용자가 마이페이지에서 설정한 커뮤니티 닉네임이 달라 **불일치로 오인**. 마이페이지엔 내 등급/소속 표시 없음.

**설계안**
- AdminUsers 컬럼 분리: **"이름"(display_name)** + **"커뮤니티 닉네임"(community_nickname)** 둘 다 표시(현재 community_nickname 미조회 → select에 추가).
- (선택) 마이페이지에 **내 등급/소속 배지** 노출(운영자 화면과 같은 값 = 정합).
- ❓**결정**: 마이페이지에 등급/소속 표시 추가할까요?

## ③ 오류 모니터링 — 무슨 말인지 모름
**현황**: `AdminErrorLogs`는 digest 그룹 + **원문 message + stack(접이식)**. 한글화는 `SOURCE_LABELS`(source=렌더크래시/런타임)만. message는 개발자 영문 그대로.

**문제**: 운영자가 message("Cannot read properties of undefined" 등)를 봐도 의미·영향·조치 모름.

**설계안**: 흔한 패턴 → **평이한 한글 설명 + 추정 영향 + 조치** 매핑(상위 N개부터):
| 패턴 | 한글 | 조치 |
|---|---|---|
| Load failed / Failed to fetch | 네트워크/서버 연결 끊김 | 일시적, 반복 시 서버 점검 |
| undefined is not ... / null | 특정 화면 데이터 누락 | 해당 화면 데이터/쿼리 확인 |
| ChunkLoadError | 새 배포 후 캐시 불일치 | 새로고침 안내 |
그룹 카드 상단에 "추정 원인" 배지 + 빈도. 원문은 접이식 유지.
- ❓**결정**: 매핑 패턴을 우선 10개 정도로 시작 OK?

## ④ 어드민 좌측 메뉴 ↔ 대시보드 불일치
**현황**: 사이드바(`AdminLayout` L44~64) ≈ **20개**. 대시보드 빠른액션(`AdminDashboard`) = **다른 부분집합 ~10개**. 대시보드는 `/admin/tip-instagrams`(Instagram 큐레이션)인데 사이드바엔 `/admin/instagram-posts`(인스타 카드뉴스) — **둘 다 존재하지만 라벨/경로가 엇갈림**. 미완(준비중/1단계) 표기는 사이드바에만.

**문제**: 두 진입점이 안 맞아 "어디서 뭘 하는지" 혼란.

**설계안**: **단일 메뉴 소스**(`adminNav` 배열 한 곳)에서 ① 사이드바 전체 ② 대시보드 "자주 쓰는" 핵심 N개를 `featured: true` 플래그로 파생 → 라벨·경로·배지 항상 일치. tip-instagrams/instagram-posts는 **하나로 통합 or 둘 다 명확히 구분**.
- ❓**결정**: 대시보드 빠른액션은 (a) 핵심 선별(featured) (b) 사이드바 전체 미러 중?
- ❓tip-instagrams와 instagram-posts는 **다른 기능**인가요(둘 다 유지) 아니면 **중복**(하나 제거)인가요?

## ⑤ CTA 버튼(보드추가·인수신청·정보수정) 불만
**현황**: 업체 상세에 보드추가(primary 카드)·인수신청(중립 카드+버튼, 기업·미입점)·운영자 정보수정(muted 스트립, 어드민). 직전 위계 정리에도 여전히 거슬림.

**문제**: 취향/배치 불만(구체 미상).

**설계안(옵션 — 택1 또는 조합)**:
- (A) **업체 상세에선 "보드추가"만** 노출. 운영자/인수는 **오버플로 메뉴(⋯)**나 역할 전용 위치로 이동(소비자 화면 깔끔).
- (B) 스타일만 더 차분히(테두리·아이콘 제거, 텍스트 링크화).
- (C) 위치 이동(상단 누적 → 하단 고정 영역/별도 섹션).
- ❓**결정**: 무엇이 거슬리나요 — **개수(너무 많음)** / **위치(상단 누적)** / **스타일(너무 튐)**? 한 가지만 짚어주시면 그 방향으로.
