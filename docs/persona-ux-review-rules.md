# 페르소나 UX 검토 규칙 (방법론)

> AGENTS.md "페르소나 UX 검토" 포인터의 상세. 페르소나 관점 UX 검토를 **재현 가능한
> 규칙**으로 codify 한 것. code-review-rules.md 의 UX 짝. 산출물 예: `docs/persona-ux-review*.md`.

## 목적

페르소나 작업의 핵심은 **사용자 친화적 UX 검토** — "이 페르소나가 이 화면에서 막히지
않는가, '내 결혼식 같다'고 느끼는가"를 현재 코드 기준으로 점검한다. 분류 엔진은 수단이지
목적이 아니다.

## 정식 페르소나 소스 (단일)

- **`src/lib/weddingPersona.ts` 의 `PERSONA_REGISTRY`(20모드)가 정식.** 검토 문서는
  이 레지스트리를 기준으로 한다. 과거 P1~P20 시트가 2종(인물이 다름) 공존했는데,
  **병렬 P-시트를 새로 만들지 말 것** — 페르소나를 추가/수정하려면 레지스트리부터 고치고
  검토는 그걸 참조한다(드리프트 방지).
- 페르소나가 어떤 입력으로 분류되는지는 레지스트리의 `match` 와 우선순위(배열 순서)가 답.

## 5대 규칙 (must)

1. **코드 근거(code-grounded)** — 추측 금지. 각 판정은 실제 코드·DB 스키마·게이팅·시스템
   프롬프트를 읽어 *"현재 코드가 이 페인을 해결하는가"* 로 검증. 동작/필드가 코드에 있어야 ✅.
2. **friction-first** — "기능이 있나"가 아니라 *"이 페르소나가 여기서 막히나·이질감을
   느끼나"*. 같은 화면이라도 페르소나마다 마찰점이 다르다(예: 임신=동선/입덧, 재혼=톤,
   지방=권역 큐레이션, 신랑=호칭/여정).
3. **컨셉 정합 + 1인 운영 비용 게이트** — 발견된 개선을 `docs/business-plan.md` §1 핵심
   5영역에 매핑하고, **자동화 1인 운영** 비용으로 거른다(사람이 매번 만져야 하는 콘텐츠·
   운영은 ROI↓). 컨셉 이탈/운영부담 큰 확장은 보류.
4. **stale 스냅샷 불신** — 검토는 *시점 스냅샷*이다. 문서 상단에 **기준 커밋 SHA + 검증일**을
   박고, 코드가 바뀌면 재실행. 옛 ❌를 현재로 착각하지 말 것(과거 "6개 모드 코드에 없음"
   지적이 이후 구현으로 무효화된 전례).
5. **label vs value 분리** — UX 카피(표시)와 백엔드 매칭 키워드를 구분(AGENTS.md 검증 규칙과
   동일). 표시 라벨 통일이 검색·분류를 깨지 않는지 확인.

## 검토 대상 surface (현행)

온보딩 · 홈/탭 · 검색 · 알림 · 찜 · 장바구니/결제 · AI 플래너 · AI 스튜디오 · 꿀팁 ·
이벤트/혜택 · 쇼핑 · 스케줄 · 예산 · 커뮤니티 · 청첩장 · 커플 도구 · 마이페이지 · 프리미엄.

## 산출 포맷

1. **TL;DR** — 가장 큰 갭 3가지(횡단 시스템 한계 우선).
2. **매트릭스** — 페르소나(20) × 페인 5개를 ✅/⚠️/❌ 로. 요약 카운트.
   - ✅ 코드에 명확히 존재 · ⚠️ 부분 지원(데이터만/한 단계만) · ❌ 부재.
3. **횡단 진단** — 여러 페르소나 공통 시스템 한계(모드 소비 여부·지역 입도·신호 미사용 등).
4. **우선순위** — 컨셉 정합 × 운영 비용으로 정렬한 "최소 필수 Top N". 신기능 확장보다
   *이미 만든 것을 운영 가능 상태로 굳히기*(가드·측정·AI 컨텍스트)를 우선.

## 자주 나오는 횡단 함정 (체크)

- **모드는 분류되지만 하위 surface 가 소비 안 함** — `persona_mode` 는 계산되는데 큐레이션/
  스케줄/예산/커뮤니티에서 분기가 없어 비표준 페르소나가 "내 결혼식 같지 않다" 느끼는 경우.
- **수집했는데 안 쓰는 신호** — `pregnant`/`marital_history`/신규 `planning_style`·`has_children`
  등을 DB 에 저장만 하고 UI/큐레이션이 안 읽는 상태(데이터만 ⚠️).
- **빈 결과의 신뢰 손상** — 가치 칩/필터를 켰는데 데이터가 0건(places 태그 미시드 등) →
  페르소나 이중 실망. UI 펴기 전 데이터 현황 쿼리 1회로 가드.
