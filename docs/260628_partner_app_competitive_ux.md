# 사장님 앱 경쟁사 UX 전수조사 → Dewy 채택 계획 (260628)

> 목적: **숨고(고수)·네이버 스마트스토어 판매자센터·네이버 스마트플레이스·쿠팡 윙(Wing)** 의 공급자(셀러/사장님)
> 측 UI·UX·시스템을 전수조사하고 **장점만** 뽑아 **Dewy 사장님 앱(apps/partners, Phase 4-B)** 에 적용하는 계획.

## 0. 조사 방법·한계 (먼저 명시 — AGENTS 검증 규칙)
- 4개 플랫폼을 **병렬 리서치 에이전트**로 셀러측 7대 영역(온보딩·리드/주문·대시보드/통계·리뷰·마케팅·알림·품질지표) 조사.
- ⚠️ **이 환경에서 WebFetch가 전면 403 차단** → 모든 보고가 **WebSearch 요약 스니펫 + 공식 도움말·셀러 블로그 2차 자료** 기반. **실제 셀러 화면(픽셀 단위 UI·버튼 워딩)은 미캡처** → 화면 단위 단정은 "추정" 표기. 정확 검증이 필요하면 **실계정 walkthrough**(각 센터 로그인) 권장.
- 수치(과금 단가·등급 기준·정책 일수)는 2025~2026 개편으로 변동 가능 → 차용 시 원문 재확인.

## 1. TL;DR — Dewy 사장님 앱이 가져올 핵심 (4개 공통 베스트)
양면 플랫폼 셀러 도구는 결국 **같은 5개 원리**로 수렴한다. 이게 Dewy 사장님 앱의 설계 베팅이다.

1. **단일 모바일 허브 + "오늘 할 일" 액션큐** — 흩어진 메뉴가 아니라, 미답글 리뷰·신규 문의·미응답 리드·임박 일정을 **한 화면 할 일 목록**으로. (스마트플레이스·스마트스토어 앱·쿠팡 윙 공통) → 1인 운영의 전제.
2. **상태머신 파이프라인 — "다음 액션 1개"** — 주문/리드가 `신규→확인→상담→견적→계약→행사`로 고정 단계화, 각 단계가 버튼 하나로 전이. (쿠팡·스마트스토어 주문 공통) → "지금 뭘 해야 하나"를 화면이 대신 기억.
3. **정보 완성도 → 노출 보상 루프** — "채울수록 검색/추천 상단"을 **게이지+체크리스트+카피로 명시**. (숨고 50% 게이트·스마트플레이스 2000자·스마트스토어 SEO 코치 공통) → 사장이 자발적으로 정보를 채움.
4. **행동·품질 기반 랭킹 + 신뢰 배지** — 응답속도·이행률·리뷰 같은 **실제 행동**을 노출 신호로. (쿠팡 아이템위너 판매자점수 4지표·스마트스토어 굿서비스·스마트플레이스 행동신호·숨고 빠른응답 공통) → 가격이 아니라 행동으로 경쟁 유도.
5. **자동화로 1인 부담 제거** — 자동 첫 응답(스마트견적)·견적 템플릿·방치/임박 자동 리마인더. (숨고 스마트견적·쿠팡 지연 리마인더 공통) → 사장이 현장에 있어도 매칭이 굴러감.

> **Dewy 차별화 한 줄**: 위 5개는 업계 표준. Dewy만의 +1 = **페르소나 20모드·예산·D-day·지역 신호를 사장 대시보드에 결합** → "내 업체를 보는 부부는 주로 어떤 유형인가"까지 보여주는 **공급자용 초개인화 BI**(네이버 비즈어드바이저보다 한 단계 위).

## 2. 플랫폼별 핵심 장점 (압축)

### 숨고 (고수/공급자)
- **열람 기반 과금 + 미열람 자동환불**: 견적 보낼 때 캐시 쓰지만 **고객이 열람했을 때만 과금**, 48h 미열람분 자동환불 → "돈 날린다" 불안 제거. (유료 리드 모델 쓸 때만 차용; 무료면 **열람 표시**만)
- **스마트견적/바로견적(자동 발송)** ★: 조건+일일예산 설정 → 매칭 요청에 **시스템이 자동 견적 발송**(수동보다 저렴·우선노출·"바로 답변 가능" 표시). **1인 운영 최대 레버.**
- **자주 쓰는 견적 템플릿**: 반복 입력 제거.
- **프로필 완성도 50% 게이트 → 노출 ON**: "채우면 노출"의 명확한 보상.
- **안심번호+인앱채팅**: 개인번호 비노출(개인정보 §13 합치).

### 네이버 스마트스토어 판매자센터
- **온보딩**: "본인인증 휴대폰 명의=정산계좌 명의" 강제 + **1~2시간 초고속 승인**. 서류 요건 사전 고지로 반려 왕복↓.
- **상품등록 = SEO 코치**: 적합도+인기도+신뢰도 알고리즘을 **공개**하고 폼이 제목 길이·반복어·카테고리·이미지 규격을 실시간 가이드.
- **비즈어드바이저 = 판매자용 무료 BI**: 유입채널·전환·재구매·고객등급(RFM).
- **판매자 등급 + 굿서비스 배지** 게임화: 주문이행·배송품질·CS를 점수화 → 자발적 응대 가속.
- **모바일 셀러앱 완결**: 주문 푸시→발송처리→통계까지 폰에서.

### 네이버 스마트플레이스 (지역 업체 — Dewy와 가장 유사)
- **업종별 서류 분기 + 1사업자 1업체 + 인증 우선노출**: 클레임(내 업체 인수) 모델이 Dewy '인수'와 정확히 대응.
- **정보 완성도 = 노출** 명시(상세 2000자·대표키워드·편의시설 체크).
- **알림받기 = 광고비 0 단골 푸시 채널** + 월 1회 무료 마케팅 메시지.
- **2025 알고리즘 = 실제 고객 행동(클릭·체류·예약클릭·전화·길찾기)이 노출 결정** → 조작 억제, 진정성 보상.
- **사장님 답글 = 활성도 신호**; **키워드 리뷰**(작성 허들↓, 구조화 데이터로 노출/필터).
- **통계 = 노출→유입→전환(전화·길찾기·저장)** 90일.

### 쿠팡 윙 (마켓플레이스 셀러)
- **주문 상태 체인 + 2버튼 진행**(결제완료→상품준비중→송장입력→배송중→완료): 상태=할일.
- **출고소요일 → 예정일 자동 산정**(Dewy는 **D-day 역산**이 더 적합).
- **지연 자동 감지 + 확인요청 리마인더(24h 응답 의무)**: 사장이 "깜빡"하기 전에 시스템이 찌름.
- **반품/취소 자동화 + 사장은 판단만**(귀책·증빙·금액). 미확인 주문 자동취소.
- **광고 자동(매출최적화)/수동 2분기** + **일예산=상한이지 소진보장 아님** 안심카피 + 권장 프리셋.
- **쿠폰 4파라미터**(정률/정액 × 최대할인캡 × 최소주문 × 와우전용), 다운로드형=리타게팅 자산.
- **아이템위너**: 가격+배송+출고기간+**판매자점수(정시출고·정시배송·주문이행·24h응답)** 종합 → 1명 대표 노출. "운영 잘하면 무료 1등".
- **표준 광고 5지표**(노출→클릭→CPC→ROAS→14일 전환) 예시 포함.

## 4. 우선순위 로드맵 (컨셉 정합 × 1인 운영비용 × 기존 surface 활용)
> "이미 만든 surface를 운영 가능 상태로 굳히기" 우선(persona-ux §4). 신규 대발명보다 **재집약·자동화·가시화**.

**P0 — 1인 운영의 골격 (저비용·고효과, 기존 재배치)**
1. **대시보드 액션큐 재집약**(A) — 분산 메뉴 → "오늘 할 일" 단일 허브. 기존 데이터 재배치라 빠름.
2. **리드/문의 상태머신 파이프라인**(B) — `신규→상담→견적→계약→행사` 카드 + "다음 액션 1개".
3. **완성도 게이지 → 노출 보상 루프**(C) — 기존 `REQUIRED_FIELDS` 체크를 게이지+카피로 격상.

**P1 — 매칭·자동화 레버 (Dewy 핵심 베팅)**
4. **자동 첫 응답(스마트견적형) + 견적 템플릿**(E) — 1인 운영 최대 레버, 매칭 직결.
5. **방치 리드/임박 일정 자동 리마인더**(E/K) — "깜빡" 방지(쿠팡 패턴, 단 페널티 없이 알림까지).
6. **행동 기반 추천 랭킹 + 신뢰 배지**(D) — `partner_rank`+응답률·전환·리뷰. 큐레이션 인프라 재사용.

**P2 — 성장·리텐션·차별화**
7. **공급자 BI 퍼널 + 페르소나 결합**(G) — Dewy만의 초개인화 BI("우리 업체 보는 부부 유형").
8. **무료 단골 채널(찜/알림받기→소식·쿠폰 푸시)**(F).
9. **광고 간편/직접 2분기 + 표준 5지표**(H), **쿠폰 정형 모델**(I), **온보딩/인수 위저드 + 인증 배지**(J), **환불 규정 데이터화**(K).

## 5. 안티패턴 — 그대로 복제 금지 (Dewy 원칙과 충돌)
- 🚫 **숨고 앱 30% 회피용 웹결제 유도** → iOS **anti-steering 위반(§8)**. 네이티브 빌드는 IAP 규정 준수, 패턴(예산 기반 노출)만 차용.
- 🚫 **리뷰 셀프 작성 링크 / 영수증 리뷰**(숨고·네이버 조작 논란) → Dewy는 **실거래(견적·계약) 검증 리뷰만**, 셀프 어뷰징 차단(§2 보안·§10 법적).
- 🚫 **쿠팡식 강제 페널티·노출정지·무응답 처리** → 너무 강압적. Dewy는 **알림+대시보드 가시화까지만**(톤).
- 🚫 **지표 과다 대시보드** → 1인 압도. **"이번 주 핵심 3숫자 + 1개 제안"** 으로 압축.
- 🚫 **완성도 빨간 체크리스트 폭탄** → "오늘 3개만 채우기" 단계적 넛지로.
