# 260611 국내 웨딩앱 경쟁 분석 — 요즘웨딩 · 아이웨딩 · 웨딩북 (+스냅포스트)

> 조사 방법: 멀티에이전트 웹 리서치(앱스토어/플레이스토어 페이지, 공식 사이트, 언론 보도 fetch 후 주장 추출 → 3표 적대 검증). **[확인]** = 교차검증 통과(2-0/3-0), **[출처만]** = 원문 인용은 확보했으나 검증 단계가 세션 한도로 중단되어 미교차검증. 수치는 조사 시점(2026-06-11) 기준.

## 1. 앱별 프로필

### 웨딩북 (Weddingbook) — 최대 경쟁자
- **규모 [확인]**: Google Play 50만+ 다운로드(내부 수치 ~64.6만), 평점 4.2/5, 리뷰 3,030개. **1점 리뷰가 382개(12.6%)** 로 불만 꼬리가 두껍다. 2015년 출시, 2026-04 최종 업데이트(활발), '광고 포함'.
- **기능 [확인]**: ① 웨딩플래너 1:1 매칭 ② 웨딩홀·스드메 비교 + 즉시예약·잔여타임 검색 ③ 실사용자 후기 14만 건·업체 6천 곳 DB ④ 실시간 채팅 견적 ⑤ 신혼살림 커머스 ⑥ 웨딩홀·예물·예복·한복·예단 결제액 **최대 10% 현금 캐시백**.
- **모바일 청첩장 [확인]**: **무제한 무료 생성** + 축의금 현금펀딩·카드결제 + RSVP 참석확인 내장. [출처만] 2025.11 '펀딩형 청첩장' 출시 보도.
- **AI [출처만]**: 2026.1 'AI 웨딩플래너'(상담형) 출시, AI 상담자의 20~25%가 업체 방문예약으로 전환된다고 대표 인터뷰에서 주장.
- **수익모델 [확인]**: 벤더 입점/제휴(입점 문의 채널 명시) + 광고 + **B2B ' 웨딩북Biz'**(노출 마케팅·고객관리·일정·통계·실시간 채팅, "10만 고객·1,000명 플래너" 홍보). [출처만] B2B SaaS(웨딩홀 고객관리)로 시작해 B2C 앱으로 전환한 이력, 2019년 가입자 20만·월 신규 1만.

### 요즘웨딩
- **규모 [확인]**: iOS 평점 3.8/5, **평가 수 12개** — '올인원 플랫폼' 포지셔닝 대비 앱 사용자 기반은 매우 초기.
- **기능 [확인]**: D-365→D-day 단계별 준비 타임라인, 업체 탐색·견적 비교(홀/드레스/사진/메이크업), 스마트 체크리스트 + **배우자 동기화**, 커뮤니티, 가입 시 1만 포인트. 스토어 설명 기준 AI·모바일청첩장·RSVP 없음.
- **UX 약점 [확인]**: "인터넷 사이트 보는 듯, 앱으로 연결하라는 팝업이 반복되고 오류까지" — webview 래퍼형 아키텍처로 추정되는 대표 불만 리뷰.
- **수익모델 [출처만]**: 제휴업체 중개 디렉토리 중심 + 계약 시 현금성 페이백(180만~222만원 상당) + '웨딩안심보험' + 오프라인 박람회·컨설팅. 디지털 도구는 홈페이지에 비노출.

### 아이웨딩 — 전통 강자 (전 항목 [출처만], 검증 중단분)
- iOS 평점 4.7/5, 평가 약 6.2천 개 — 경쟁군 중 최대 리뷰 볼륨·최고 평점.
- 기능: 날짜·지역·예산 매칭 무료 웨딩홀 상담, **투명가격 스드메 패키지**, 업체 포트폴리오, 인앱 메시징 상담. 반복 칭찬 키워드 = "가격 공개·투명성".
- AI·청첩장·RSVP·게임류는 스토어 설명에 없음.

### 스냅포스트 (모바일 청첩장 전문 — RSVP 벤치마크, [출처만])
- RSVP 기본 수집 무료 + 응답 수 제한 기반 freemium.
- 수집 항목: **신랑측/신부측 자동 구분, 성인·어린이(3~12세)·유아 인원, 식사 여부**, 메시지, 연락처. **엑셀 일괄 내보내기** 지원 → 이 카테고리의 기대 기본기.

## 2. 시장 구조 시사점

1. **3사 모두 본질은 벤더 중개업**(커미션·입점·캐시백). 체크리스트·예산 같은 도구는 유입 훅이고 돈은 B2B에서 번다. 웨딩북Biz가 그 정점.
2. **생성형 이미지 AI는 시장 공백.** 웨딩북의 'AI 플래너'도 상담 챗봇이지, Dewy의 AI 스튜디오(드레스/헤어/메이크업 생성) 같은 비주얼 생성은 3사 모두 없다.
3. **모바일 청첩장은 웨딩북이 '무제한 무료'로 가격 파괴.** 청첩장 자체 과금은 어렵고, RSVP 데이터가 다른 기능(식대·하객·예산)으로 흘러가는 연동 가치로 차별화해야 한다.
4. **중개형의 그늘 = 평판 리스크.** 웨딩북 1점 12.6%, 요즘웨딩 webview 불만 — 영업 피로·앱 품질이 반복 불만 축.

## 3. Dewy 개선 권장사항

### P0 — 직접 경쟁 대응 (단기)
1. **RSVP → 하객명단 → 예산리포트 파이프라인 연결.** Dewy는 이미 세 조각을 다 갖고 있다: 공개 청첩장 RSVP 수집(`invitation_rsvp`), 수기 하객명단·식대/좌석 추정(`src/lib/guestList.ts` 의 `expectedHeads`), 예산리포트 식대 방어율. 지금은 서로 단절 — RSVP 제출분을 하객명단으로 가져오기(중복 매칭 포함) + 리포트에 "보증인원 vs RSVP 집계" 비교를 넣으면 **경쟁 3사 누구도 없는 연동**이 된다.
2. **RSVP 필드 보강:** 신랑측/신부측 구분(현재 스키마에 없음), 성인/자녀 인원 구분, 작성자용 집계 대시보드 + CSV/엑셀 내보내기. 스냅포스트가 이미 '기본기'로 정의한 기대치라 없으면 비교 열위.
3. **AI 스튜디오를 포지셔닝 전면에.** 스토어 설명 첫 줄·온보딩 첫 화면·마케팅 카피의 1순위로. 시장 공백이 닫히기 전(웨딩북이 상담 AI에서 비주얼로 확장하기 전)의 선점 윈도우.

### P1 — 수익·리텐션 (중기)
4. **이벤트/혜택 탭 → 벤더 제휴 채널화.** `business_events_coupons` 기반이 이미 있다. 웨딩북Biz의 라이트 버전(업체 셀프 등록 + 쿠폰/프로모션 노출)부터 — 시장의 검증된 수익원은 B2B다.
5. **D-day 단계별 타임라인 강화.** 요즘웨딩의 코어 UX(D-365 단계 안내). Dewy는 `checklistTemplate` + Schedule 페이지가 있으므로, 페르소나 엔진(20모드)으로 단계 안내를 개인화하면 "동기화 체크리스트"를 넘어설 수 있다.
6. **앱스토어 평판 설계.** 베타 초기 평점이 수년을 좌우한다(요즘웨딩 3.8/12개가 반례). 만족 모먼트(청첩장 발행 직후, AI 생성 성공 직후)에 리뷰 유도 + 포인트 보상. AdMob 빈도는 절제 — '광고 포함' 1점 리뷰 패턴 회피.
7. **축의금 결제(펀딩형)는 중장기 검토만.** PG·정산·세무 부담이 크고 웨딩북이 선점. 단기는 계좌 복사 UX(금번 안전화 완료) + 송금 딥링크 수준이 합리적.

### P2 — 기반 (장기)
8. **후기 콜드스타트 해소.** 웨딩북의 해자는 후기 14만 건. `place_reviews` 시딩 전략(커뮤니티 글→후기 전환 유도, 작성 보상)을 별도 설계.
9. **네이티브 체감 품질.** 요즘웨딩 webview 불만의 반면교사 — Capacitor 전환 애니메이션·오프라인 캐시·푸시로 "앱다움" 확보.
