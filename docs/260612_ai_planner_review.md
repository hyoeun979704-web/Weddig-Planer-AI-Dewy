# 260612 AI 웨딩플래너 7기준 검토 + 모델 비교

## TL;DR

AI 웨딩플래너를 7기준으로 코드 실사 + 경쟁 버티컬 챗봇 조사 + 21시나리오 자체 측정 + 모델 매트릭스 비교. 현 구조는 **하이브리드(클라 intent 라우터 60+패턴 → 룰/DB 즉답 or Gemini Flash 스트리밍)** + 20 페르소나 레지스트리 + user_ai_memory 추출. 강점은 페르소나·문제해결, 치명 갭은 **대화 비영속(⑥)·업체 실데이터 검색 부재(③④)**.

> 배포 부수 발견: `prompt.ts` 템플릿 리터럴에 비이스케이프 백틱이 있어 **ai-planner 가 그 줄 도입 이후 배포 실패 → 프로덕션이 구버전 프롬프트로 구동** 중이었음. 수정·재배포 완료(커밋 f84efb2).

## 2. 경쟁 버티컬 챗봇 장점 (조사)

- **웨딩북 AI 플래너**: 실시간 슬롯/예식 데이터 자연어 검색(W1) · 복합조건 검색(W2) · 대화→상담예약→계약 완결(W3) · 자체 10년 데이터를 신뢰 출처로(W4) [출처: 테크월드]
- **트립닷컴 TripGenie**: AI 경유 예약 400%↑(T1) · 멀티모달 사진 인식 재방문율 2배(T2) · 구조화 비교로 클릭 80%↓(T3) [출처: 벤처스퀘어/트래블데일리]
- **버티컬 일반**: RAG 로 자체 데이터 인용해 신뢰도 확보(G1)
- **Dewy 우위**: 20 페르소나·도메인 캡슐·0환각 DB 즉답 — 경쟁 공개자료에 없는 차별점

## 5. 개선 백로그 (우선순위)

- **P0 ⑥b 대화 영속화**: `ai_conversations`/`ai_messages` + RLS + useAIPlanner 복원
- **P0 ③a+④a 업체 검색 RAG**: places DB 조건 검색을 함수에 통합, 응답에 실데이터 카드+출처
- **P1 ③c 행동 연결**: 응답에 찜/문의/일정추가 액션
- **P1 ⑤a 복합조건 파싱**: LLM 함수콜 기반 구조화 검색
- **P2**: 메모리 관리 UI(⑥d) · 비교표 출력 포맷(⑦b) · 메모리 확인 칩(④d)
