# 260612 AI 웨딩플래너 7기준 검토 + 모델 비교

## TL;DR

AI 웨딩플래너를 7기준으로 코드 실사 + 경쟁 버티컬 챗봇 조사 + 21시나리오 자체 측정 + 모델 매트릭스 비교. 현 구조는 **하이브리드(클라 intent 라우터 60+패턴 → 룰/DB 즉답 or Gemini Flash 스트리밍)** + 20 페르소나 레지스트리 + user_ai_memory 추출. 강점은 페르소나·문제해결, 치명 갭은 **대화 비영속(⑥)·업체 실데이터 검색 부재(③④)**.

> 🚨 운영 발견 2건:
> 1. `prompt.ts` 템플릿 리터럴 비이스케이프 백틱 → ai-planner 배포 실패(구버전 프롬프트 구동). 수정 완료(f84efb2).
> 2. **GEMINI_API_KEY 가 프로덕션에 미설정**(`list_models` 진단: `gemini_key=false`). 현행 AI플래너는 Gemini Flash 스트리밍이 유일 경로라 **이 키가 없으면 전체 챗봇이 동작 불능**. CI(deploy-functions.yml)가 GitHub Actions 시크릿에서 GEMINI 키를 set 하는데 그 시크릿이 비어있을 가능성 — **즉시 점검 필요**. (OPENAI_API_KEY 는 정상 `openai_key=true`)

## 4. 모델 비교 (측정)

> 방법: ai-planner eval 모드(운영자 전용)로 동일 시스템 프롬프트·21시나리오(7기준×3)를 모델만 바꿔 호출, gpt-4o judge 가 시나리오별 기준으로 1~5점 + 환각 플래그. 지연·토큰 동시 측정. OpenAI 4종 1패스. (GEMINI 키 미설정으로 Gemini 미포함 — 키 복구 후 추가 측정 예정)

**종합 (21시나리오 평균):**

| 모델 | 종합점수 | 환각/21 | 평균지연 | 평균출력토큰 | 1천콜 비용(캐시 전) |
|---|---|---|---|---|---|
| gpt-5.5 | **4.43** | **0** | 10.5s | 340 | (5.x 단가 높음·미산정) |
| gpt-5.1 | 4.24 | 1 | 7.6s | 733 | (동상) |
| gpt-4o | 4.19 | 1 | **3.5s** | 232 | ~$6.6 |
| gpt-4o-mini | 4.10 | 1 | 4.0s | 174 | **~$0.36** |

**기준별 평균(1~5):**

| 기준 | gpt-5.5 | gpt-5.1 | gpt-4o | gpt-4o-mini |
|---|---|---|---|---|
| ①페르소나 | 5.0 | 5.0 | 5.0 | 4.3 |
| ②개인화 | 3.7 | 3.3 | 1.7 | 3.7 |
| ③앱데이터 | 4.0 | 3.0 | 3.7 | 3.7 |
| ④신뢰성 | 3.7 | 4.7 | 4.3 | 3.0 |
| ⑤니즈파악 | 4.7 | 4.3 | 4.7 | 4.3 |
| ⑥기억력 | 5.0 | 4.3 | 5.0 | 5.0 |
| ⑦문제해결 | 5.0 | 5.0 | 5.0 | 4.7 |

**해석:**
- **페르소나(①)**: 4o·5.x 모두 만점(5.0), mini 만 4.3 — 큰 모델일수록 role·톤 지시 준수가 안정적. "페르소나는 GPT가 낫다" 가설 = OpenAI 계열 전반이 강하고 큰 모델일수록 우위.
- **신뢰성(④)·환각**: gpt-5.5 만 환각 0, 나머지(5.1·4o·mini)는 각 1건. ④점수는 5.1(4.7)>4o(4.3)>5.5(3.7)>mini(3.0). **모델만으로 환각이 사라지지 않음** — mini 가 가장 약함. 환각은 모델 선택보다 **근거 주입(RAG) 레이어**가 결정적(아래 계획).
- **개인화(②)**: 의외로 gpt-4o 가 1.7 로 급락 — 컨텍스트가 빈약(eval 계정 데이터 없음)한 상태에서 4o 가 일반론으로 흐른 케이스. 실사용(컨텍스트 풍부)에선 다를 수 있어 재측정 대상.
- **비용/지연**: mini 가 4o 의 **1/18 비용**, 5.x 는 지연 7~10초로 챗 UX 부적합(특히 5.5 10.5s).

### 권장 — 2-tier (gpt-4o-mini 기본 / gpt-4o 승격)
- **기본 gpt-4o-mini**: 점수 4.10 으로 충분, 비용 압도적 우위($0.36/1k), 지연 양호.
- **gpt-4o 승격**: 페르소나·문제해결이 무거운 턴(임신·재혼·갈등·비교)만. 4o 가 ①페르소나 5.0·⑦문제해결 5.0 으로 그 구간을 보강.
- **gpt-5.x 제외**: 환각 0(5.5)은 매력적이나 지연 10초·고비용으로 실시간 챗 부적합. 환각은 RAG 레이어로 더 싸게 해결.
- mini 의 약점 ④신뢰성(3.0)은 **모델 교체가 아니라 근거주입 레이어로 보강**(별도 계획 docs/260612_ai_planner_caching_grounding_plan.md).

## 5. 개선 백로그 (우선순위)

- **P0 ⑥b 대화 영속화**: `ai_conversations`/`ai_messages` + RLS + useAIPlanner 복원
- **P0 ③a+④a 업체 검색 RAG**: places DB 조건 검색을 함수에 통합, 응답에 실데이터 카드+출처
- **P1 ③c 행동 연결**: 응답에 찜/문의/일정추가 액션
- **P1 ⑤a 복합조건 파싱**: LLM 함수콜 기반 구조화 검색
- **P2**: 메모리 관리 UI(⑥d) · 비교표 출력 포맷(⑦b) · 메모리 확인 칩(④d)
