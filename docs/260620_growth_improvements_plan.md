## 1. 추천인 초대 (referral) — 기존 스캐폴드 완성
**분석 결과(현황)**: 테이블은 있으나 **보상·UI·가입연결 미배선**.
- `referral_codes(user_id, code)` — 사용자별 초대코드.
- `referrals(code, referrer_user_id, referee_user_id, redeemed_at)` — 사용 이력. **보상 컬럼 없음.**
- 초대코드 발급/공유 UI·가입 시 코드 입력·보상 지급 로직 **부재**.

**설계(완성 항목)**:
1. **코드 발급**: 가입 시(또는 최초 진입) `referral_codes`에 사용자 코드 생성(없으면). 짧은 영숫자.
2. **공유 UI**: 마이페이지/설정에 "친구 초대" — 코드+딥링크(`https://dewy.kr/?ref=CODE`)를 **원탭 공유**
   (`navigator.share`/`@capacitor/share` 재사용, §2와 동일 경로).
3. **가입 연결(attribution)**: `?ref=CODE`를 가입 플로우까지 보존(localStorage), 가입 완료 시 `referrals`
   insert(referee=신규, referrer=코드 주인).
4. **보상 지급**: redeem 시 **양쪽에 하트 지급**(`earn_hearts` RPC 재사용). 금액은 상수화(예: 초대자 +N,
   피초대자 +M). **부정 방지**: 자기추천 차단, 1인 1회(referee unique), 신규계정만, 일일 상한.
   지급 시점은 "가입"보다 **"피초대자 첫 유효행동(예: 첫 AI 생성)"** 권장(어뷰징↓).
5. **멱등**: `referrals`에 referee unique + 지급 트랜잭션 멱등키.

> ★ **정책 보너스**: 추천 보상 하트는 **"지급분"이라 IAP 30% 대상 아님**(Google/Apple 모두 earned
> currency는 스토어 빌링 불요). 바이럴 + 수수료 회피 동시 효과.

## 2. AI 결과 원탭 공유 (워터마크 없음)
**현황**: `navigator.share`가 `MakeupFittingResult`·`ShareCodeCard`·`PartnerLinkCard`·`Budget`에 이미 사용.
AI 결과 일부만 공유 가능.
**설계**:
- **모든 AI 결과**(드레스 피팅·스드메·헤어·촬영시안·청첩장)에 **원탭 공유 버튼** 일관 적용.
- 공유 내용: 결과 이미지(+간단 텍스트/딥링크). **워터마크는 넣지 않음**(결정 2).
- **네이티브 신뢰성**: 웹 `navigator.share`는 Capacitor WebView에서 불안정할 수 있어 **`@capacitor/share`**
  플러그인으로 네이티브 공유시트 보장(웹은 Web Share 폴백). 단일 공유 유틸로 묶어 재사용(`src/lib/native/`).
- 부수효과: Apple "네이티브 기능"(4.2) 충족에도 기여.

## 3. 무료체험 1개월 + 데모 2개월
**현황**: `subscriptions.trial_ends_at` + `useSubscription`(isTrialActive/trialDaysLeft) 동작. trial은
`kakao-pay-approve`에서 설정. `kakao-pay-ready`의 trial=100원 카드인증.
**설계**:
- **기본 체험 = 1개월**: trial 승인 시 `trial_ends_at = now + 1개월`로 설정(현 위치 `kakao-pay-approve`).
  상수화(`TRIAL_DAYS=30` 등 단일 소스).
- **데모 테스터 2개월**: 리뷰어 고정계정(또는 화이트리스트 이메일)에 `trial_ends_at = now + 2개월` 부여
  (관리자/시드 경로). 일반 사용자와 분리.
- **네이티브(IAP) 체험**: 스토어 **인트로 오퍼(1개월 무료)** 로 등록(Apple/Google 각각). 데모 2개월은
  스토어 오퍼가 아니라 **서버 권한 부여**로 처리(스토어와 무관).
- **정책 정합(중요)**: 무료체험은 **자동결제 전환 고지**가 핵심(Apple 3.1.2 / 한국 전자상거래) →
  체험 화면에 "1개월 후 자동결제, 기간/가격/해지" 명시 + 약관·개인정보 인앱 링크(G3와 동일). 현재
  UpgradeModal "체험 종료 후 자동 결제 없음" 문구가 **실제 동작과 일치하는지 재확인**(불일치=반려·분쟁).

  정합 확인만 수행.
