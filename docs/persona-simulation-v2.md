## 1. G-1 황지수 (일반·재혼·호텔)

**상황** — 1년 만에 다시 준비. 첫 결혼 때의 노하우는 있지만 시장이 바뀌었다고 느낌. 평일 자기 전 25분 + 점심 10분.

| 세션 | 시간 | 동선 | 막힌 지점 |
|---|---|---|---|
| 1 | 12분 | Splash → 홈 → `WeddingInfoSetupModal`: general / 호텔 / D-220 / "정보 알아보는 중" 입력. 시드 체크리스트 32개. | 모달에서 **"재혼" 컨텍스트가 없음** — 청첩장 발송 범위·하객 수 안내가 일반인 기준. |
| 2 | 18분 | AI 플래너 → "**재혼 호텔 1.4억** 가능?" 자유 입력. STYLE_OVERRIDES.general의 "양가 분담" quick card는 본인 case와 어긋남 (이미 자녀 1명, 양가 분담 안 함). | quick question 4개 모두 본인과 어긋남 → 첫 메시지 자유입력으로 우회. |

| 5 | 14분 | CoupleVote는 안 씀(상대가 앱 안 깔음). 매거진에서 "재혼 노트" 검색 → 결과 0건 → 외부 이탈. | **"재혼" 검색 결과 0 → 이탈 트리거.** |

## 9. 우선 처리 권고 (low-cost · high-impact 순)

1. **모달 확장 — marital_history + pregnant 플래그 + small 프리셋의 한복 분리** (P1·P2·P3)
   - 영향: G-1, F-1, S-1 — 첫 세션 이탈을 막음
   - 비용: 모달 1~2 step 추가, schema 컬럼 2개
2. **AI 플래너 system prompt 컨텍스트 주입** (P8)
   - `weddingSettings`에 marital_history / values_tags / pregnancy_week / region 추가 후 system prompt에 전달
   - 6인 전부에서 만족도 향상 — 가장 ROI 높음
3. **D-90 미만 사용자에 "압축 체크리스트 모드"** (P10)
   - `usePersonaInsights`에서 daysUntilWedding < 90일이면 시드 항목 자동 priority 분류·접기
4. **셀프웨딩 평균 시세 분리** (P12)
   - 데이터 라벨링 1회 + 카테고리 평균 쿼리 분기
5. **매거진/커뮤니티 콘텐츠 시드 — 재혼 · 임산부 · 부모님 설득** (P7)
   - 콘텐츠 8~10편 기획. 각 페르소나의 검색 0건 → N건으로 전환.
6. **하객 리스트 v0** (P5)
   - 별도 페이지로 큼. 그러나 G-2 같은 케이스의 외부 이탈을 막는 핵심. 단순 CRUD + CSV import면 충분.
7. **가치 기반 필터 (친환경/비건/반려동물/종교)** (P4)
   - 답례품·베뉴·케이터링 카테고리에 tag 추가. S-2 같은 가치 중심 페르소나 잡음.

## 10. 시뮬레이션 결론 한 줄

> **현재 Dewy는 "서울·초혼·신부 단독·정형 체형·약한 가치축"인 정중앙 페르소나에 최적화돼 있고, 그 바깥 케이스로 한 발만 벗어나도 세션이 절반으로 짧아진다.** 우선순위 1~4번을 처리하면 6인 평균이 21분 → 32분(+52%) 까지 회복 가능할 것으로 추정.
