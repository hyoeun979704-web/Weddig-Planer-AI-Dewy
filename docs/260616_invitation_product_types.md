# 청첩장 상품 종류 모델 — 디자인 / 디자인+인쇄 / 셀프디자인(인쇄)

> 앞선 두 기획을 통합하는 **상품 종류(product_type)** 모델. 업체가 가진 역량(디자인/인쇄)이
> 달라 상품을 3종으로 나눈다. 짝 문서: `260616_invitation_design_marketplace.md`(디자인),
> `260616_invitation_print_order_design.md`(인쇄). 기준 커밋 `11518a3c`.

## 0. 핵심 — 종류마다 결제 정책이 갈린다 (제일 중요)
| 상품 종류 | 구성 | 디자인 출처 | 결제 성격 | 결제 레일 | 공급 업체 |
|---|---|---|---|---|---|
| **디자인** | 디지털 디자인만 | 작가 | **디지털** | 디지털(하트/프리미엄과 동일 — 카카오페이 선례) | 작가(design) |
| **디자인+인쇄** | 작가 디자인 + 인쇄·배송 | 작가 | **실물(혼합→실물 취급)** | 실물 PG `kakao-pay-order` (IAP 금지) | 작가+인쇄(design_print) |
| **셀프디자인(인쇄)** | 사용자가 만든 디자인 인쇄·배송 | **사용자 본인** | **실물** | 실물 PG `kakao-pay-order` | 인쇄소(print_only) |

- **디지털(디자인만)** ↔ **실물(인쇄 포함)** 의 경계가 IAP 정책선이다(디지털=IAP 영역이나 Dewy
  는 카카오페이 선례 / 실물=외부PG·IAP 금지). **혼합(디자인+인쇄)은 실물 산출물이 있으므로 실물
  주문으로 취급**해 외부PG·인쇄 약관을 적용(디지털 라이선스는 그 주문에 부수 grant).

## 1. 업체 역량(capabilities) 모델
업체(작가/인쇄소)가 제공 가능한 종류를 선언 — `offers` 다중 플래그:
- `design` : 디지털 디자인 판매(작가).
- `design_print` : 자기 디자인을 인쇄·배송까지(작가+인쇄 가능).
- `print_only` : **남의/사용자 셀프 디자인을 인쇄·배송**(인쇄소 — 디자인 안 만듦).

→ 한 업체가 `design`+`design_print` 둘 다일 수도, `print_only`만일 수도 있다(요청: "인쇄까지 가능"
vs "인쇄만"). `business_profiles`/`places` 또는 `print_products` 에 `offers text[]`.

## 2. 종류별 플로우
- **디자인**: 마켓에서 작가 디자인 구매 → 라이선스 grant → 에디터 사용 → (원하면 나중에 인쇄는
  별도 셀프디자인 인쇄 주문). 디지털 결제.
- **디자인+인쇄**: 작가 디자인 상세에서 "디자인+인쇄" 선택 → 인쇄 옵션(용지·코팅·매수·빠른발송·
  배송지) → **실물 주문**(서버 금액=디자인료+인쇄비) → 작가(또는 작가의 인쇄 파트너)가 제작·발송.
  결제 즉시 디자인 라이선스도 grant(에디터에서 문구 편집 후 인쇄 확정 가능).
- **셀프디자인(인쇄)**: 사용자가 에디터에서 직접 디자인(무료/기본 템플릿 or 구매 디자인) → "인쇄
  주문" → **인쇄소(print_only) 선택** → 인쇄 옵션·배송지 → 실물 주문 → 인쇄소 제작·발송.

## 3. 스키마 델타 (두 짝 문서 통합)
- **업체 역량**: `offers text[]`(`design|design_print|print_only`) on 업체.
- **`designer_designs`**: `sellable text[]`(`design`, `design_print`) — 작가가 이 디자인을 디지털만/
  인쇄포함까지 파는지. `design_print` 면 인쇄 가격/옵션(자체 또는 연결된 print_product) 지정.
- **`print_products`**(인쇄 가격표): `print_only` 업체 + `design_print` 작가가 공용. 셀프디자인 인쇄의
  공급 풀.
- **주문 통합** `orders`(+ print_orders): `product_type('design'|'design_print'|'print')`,
  `design_id?`(디자인/디자인+인쇄), `invitation_id?`(셀프디자인=사용자 작업물), 인쇄옵션·배송지(실물일 때).
  결제 성격 분기: `design` → 디지털 grant만 / `design_print`·`print` → 실물 주문+배송 + (해당 시)라이선스.
- `design_purchases`(라이선스): `design`·`design_print` 결제 승인 시 grant(멱등).

## 4. 결제 분기 (서버 — 클라 금액 불신)
- `design` : 디지털 결제(기존 하트/프리미엄 레일) → `design_purchases` grant.
- `design_print` : 실물 주문(`kakao-pay-order`) 금액 = 디자인료 + 인쇄비(옵션) + 배송비 → 승인 시
  print_order 생성 + 라이선스 grant.
- `print`(셀프디자인) : 실물 주문 금액 = 인쇄비(옵션) + 배송비. 라이선스 불필요(본인 저작물).

## 5. 정책·법무 차이 (요지 — 상세는 짝 문서)
- **디자인(디지털)**: IAP 정책(기존 디지털 선례 따름), 저작권/이용허락(작가 보유·재판매 금지),
  디지털 청약철회 예외(사용 개시·사전동의).
- **디자인+인쇄 / 셀프디자인(실물)**: 외부PG·IAP 금지, 통신판매/배송/맞춤제작 청약철회 예외(사전고지),
  에스크로, 배송지 개인정보·제3자(인쇄소) 제공, 5년 보관.
- **셀프디자인 인쇄**: 사용자가 올린 디자인을 인쇄 → **사용자가 저작권/사용권 보유 보증**(타인 디자인
  무단 인쇄 방지 고지). 작가 디자인을 셀프로 인쇄하려면 그 디자인 라이선스에 인쇄 허용 범위 포함돼야.

## 6. 확정 필요
- [ ] 업체 역량 플래그 위치(`places`/`business_profiles`/`print_products`).
- [ ] `design_print` 인쇄 주체: 작가 직접 vs 작가가 지정한 인쇄 파트너(정산 2자/3자).
- [ ] 디자인 라이선스에 **인쇄 허용 범위**(본인 인쇄 무제한? 매수?) — 셀프 인쇄와 충돌 방지.
- [ ] 통화 모델(디지털 디자인: 하트/포인트 vs KRW) — 디자인 마켓 문서 §6과 동일.
- [ ] 중개 vs 직접판매(전 종류 일관).

## 7. 단계 (제안 — 통합 로드맵)
1. 업체 역량 `offers` + `designer_designs.sellable` + `print_products`(역량별 등록 UI).
2. 마켓/에디터에서 **상품 종류 선택 UI**(디자인 / 디자인+인쇄 / 셀프디자인 인쇄).
3. 주문 통합 + 결제 분기(디지털 grant / 실물 PG) — 서버 금액·멱등.
4. 작가·인쇄소 포털(주문/제작/배송/정산) + 약관·라이선스·고지(법무).
> 결제·인가·실물 배송 경로 — 정적 통과만으로 완료 보고 금지(실결제·실배송·라이선스 grant e2e).
