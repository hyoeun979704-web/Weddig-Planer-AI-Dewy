> 범위: **청첩장 제품 한정** — ① 모바일 발행 뷰어(하객) ② 편집 에디터(사용자) ③ 운영자 어드민
> ④ 생성 위저드. 기준: **모바일=메이크디어, 종이=비즈하우스**. 연계 문서: `mobile-invitation-revamp.md`
> (뷰어 아키텍처), `mobile-editor-plan.md`(에디터 구조). 이 문서는 **UI/UX 디테일·품질**에 집중한다.

## 1. 모바일 발행 뷰어 (하객 경험) — 가장 중요

### 현재(캔버스) 문제
- **타이포 래스터라이즈**: 캔버스 텍스트라 고DPI 흐릿·선택 불가·스크린리더 불가(접근성 0).
- **모션 빈약**: 슬롯 트윈뿐, 섹션 등장·패럴랙스·매끄러운 전환 없음.
- **사운드 부재**: BGM 구조적 불가.
- **인터랙션 단편적**: 지도=외부앱 딥링크, 계좌=텍스트, 갤러리만 라이트박스.

### 개선 (네이티브 전환 = revamp.md, 여기선 UX 디테일)
| 항목 | 개선 | 우선 |
|---|---|---|
| 타이포 | 웨딩 세리프+산세리프 조합, 한글 자간/행간 토큰화, 본문 16px·행간 2.0 | P0 |
| 여백·리듬 | 섹션 상하 여백 스케일(80/64/48) 일관, 풀블리드 커버 | P0 |
| 모션 | 섹션 스크롤 등장(Reveal)·커버 패럴랙스, `prefers-reduced-motion` 존중 | P0 |
| 사운드 | BGM 토글(우상단), iOS 제스처 언락, 재생 상태 시각피드백 | P0 |
| 갤러리 | 풀스크린 스와이프+카운터+`좌우로 넘겨보세요` 힌트, 핀치줌(차기) | P1 |
| 계좌 | 아코디언+**복사 버튼**+복사 토스트(은행·예금주 분리 표기) | P1 |
| 지도 | 지도앱 바로열기(카카오·네이버) + (차기)임베드 지도+핀 | P1 |
| D-day | 미니 달력(예식일 강조)+실시간 카운트다운 | P1 |
| 공유 | 카카오 공유 썸네일/제목 커스텀(og) — 링크만 던지지 않기 | P1 |

### 상태·엣지(놓치기 쉬운 dead-end)
- **로딩**: 폰트 FOUT 방지(로드 전 본문 opacity 0→1) — Phase 1 적용됨. 사진 lazy + 스켈레톤.
- **빈 데이터**: 데이터 없는 섹션은 **숨김**(빈 박스·placeholder 금지 — 큐레이션 원칙).
- **에러**: 잘못된 slug → "청첩장을 찾을 수 없어요" 친절 안내(현재 적용). 사진 로드 실패 폴백.
- **마감**: RSVP 마감 시 폼 대신 안내(I8-B 적용).

### 모바일웹/iOS·접근성
- safe-area(`--safe-bottom`) 적용, 탭 타깃 ≥44px, 가로 스크롤 차단.
- 아이콘 버튼 `aria-label`(BGM·닫기 등), 이미지 `alt`, 명도 대비 충족.
- **RSVP 폼 입력 보존**: iOS 탭 폐기 시 입력 유실 → **draft 자동저장**(`useTextDraft`) 적용 *(현재 미적용 — P1)*.

## 2. 편집 에디터 (사용자) — 핵심 난제

### 현재(캔버스 Studio) 문제
- 드래그/리사이즈가 모바일에서 **조작 어렵고 결과 들쭉날쭉**(품질 비일관의 주원인).
- "하객이 보는 화면"과 편집 캔버스가 분리 안 됨 → 최종 모습 예측 어려움.
- 텍스트/사진/폰트 변경이 슬롯 단위라 직관성↓.

### 개선 (섹션 폼 + 라이브 프리뷰 — editor-plan.md, 여기선 UX 디테일)
| 항목 | 개선 | 우선 |
|---|---|---|
| 라이브 프리뷰 | 상단 폰 프레임에 **실제 뷰어 body**를 draft로 즉시 반영 | P0 |
| 섹션 카드 | on/off 토글·드래그 순서·삭제·추가(custom)·펼침 인라인 폼 | P0 |
| 편집 범위 | 스티커추가·사진/BGM변경·폰트/내용변경(위치·크기 **제외**) | P0 |
| 입력 검증 | 필수값(이름·날짜·예식장) 인라인 검증, 빈 섹션 경고 | P1 |
| 저장 피드백 | 자동저장 인디케이터·발행 진행 상태(이미지 서명 중…) | P1 |
| 사진 가이드 | 권장 비율(3:4)·해상도 안내, 업로드 진행률, 재정렬 DnD | P1 |
| 온보딩 | 첫 진입 1회 코치마크(섹션 토글·미리보기 설명) | P2 |
| 스티커 | 프리셋 앵커(좌상/우상/하단/중앙) 선택 — 자유 좌표 없음 | P1 |
| 실수 복구 | 섹션 삭제 Undo(토스트 '되돌리기'), 폰트/내용 변경 undo | P2 |

### 상태·엣지
- 빈 청첩장(데이터 0) 미리보기에서도 섹션 골격+안내, 발행 전 필수값 미충족 시 발행 차단+무엇이 빈지 표시.
- 큰 입력 폼은 `formDraft` 로 이탈 보호(iOS).

## 3. 운영자 어드민 (`AdminInvitationTemplates` / `AdminTemplateEditor`)

### 현재 문제
- 모바일 템플릿도 **좌표 슬롯 편집(Konva)** → 운영자가 픽셀 좌표를 다뤄야 해 비효율·실수 유발.
- 템플릿 품질이 운영자 손기술에 의존(일관성↓).

### 개선
| 항목 | 개선 | 우선 |
|---|---|---|
| 모바일 템플릿 | 좌표 편집 → **테마 프리셋 관리**(테마·기본 섹션셋/순서·샘플·추천폰트·썸네일) | P0 |
| 미리보기 | 템플릿 저장 시 실제 뷰어로 즉시 미리보기(샘플 콘텐츠) | P1 |
| 종이 분리 | 캔버스 에디터는 **종이 전용**(진입 분기), 모바일 진입 차단 | P0 |
| 에셋/폰트 | 스티커·폰트 관리 UI 유지·정돈(카테고리·활성 토글 일관) | P2 |

## 5. 공통 — 디자인 토큰·일관성

- **토큰화**: 색 역할(ink/inkSoft/accent/surface/bg), 타입 스케일, 여백 스케일, 모션 듀레이션/이징을
  `mobileThemes` 토큰으로 단일화(현재 템플릿마다 색 하드코딩 → 드리프트). 신규 테마=토큰만 추가.
- **테마 일관**: 뷰어·에디터 프리뷰·어드민 미리보기가 **같은 토큰/같은 body** 사용(드리프트 차단).
