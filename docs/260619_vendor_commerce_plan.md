## 3. 결정사항 (▶ = 권장 기본값, 별말 없으면 이대로 진행)
1. **이벤트 노출 위치** ▶ 홈 상단 **'이벤트' 탭(`/deals` 전체 혜택)** 유지(주신 스크린샷 화면). *아니면 정확한 위치 지정.*
2. **이벤트 '신청하기'** ▶ 웨딩은 상담/예약 중심(경쟁사 §8) → **인앱 문의/신청**(`PlaceInquirySheet`)로 연결. 결제형 이벤트(참가비)면 카카오페이(Phase C 이후).
3. **상품 '결제 지원' 판별** ▶ **상품별 토글**(`payment_enabled`). 업체가 즉시결제 적합한 소액·부가 상품만 ON. (고액 홀·스드메는 문의/예약 권장.)
4. **카카오페이 수수료율** ▶ **일반 1.62% + VAT 10%**를 안전 안내치로 **설정 상수**화(실계약 확정 시 조정). 머니 1.45%/영세 0.38%는 참고.
5. **카테고리 핵심 스펙(기본정보 노출)** ▶ 감사문서 표 그대로(아래 §4.1). *가감 원하면 표시.*
6. **#362 처리** ▶ 완료된 안전 항목(R1·R3·R5·R6·R9 등)은 **머지**, 이벤트 위치·신청하기만 위 ①②로 확정 후 포함. *원하면 분리 머지.*
7. **정산 구조** ▶ **카카오페이가 업체에 직접 정산**(플랫폼 자금 미보유) → PG 등록 의무 회피(§9).

## 4. 정보구조 / UX 설계
### 4.1 기본정보 탭 — 카테고리별 '핵심 스펙' 블록(신규, R10)
- 컴포넌트 `PlaceKeyFacts`: 카테고리→`[{label,value}]` 맵. 값 없으면 자동 생략(빈칸 금지). 위치: 갤러리/태그 아래, 혜택 위.
- **업체 소개**(`place.description`) 공통 노출(현재 어느 탭에도 없음).
- 카테고리별 핵심 스펙(감사문서 §, 경쟁사 §8 반영):
  - 웨딩홀: **식대(1인)·대관료·최소보증인원**·홀타입
  - 스튜디오: **시작가·원본/수정컷·작가등급**·촬영시간
  - 드레스: 시작가·대여/맞춤·피팅수·벌수 / 메이크업: 시작가·리허설포함·원장급·출장
  - 한복: 시작가·대여/맞춤·액세서리 / 예복: 시작가·맞춤/기성·피팅수
  - 허니문: 평균예산·국가·박/일·항공/호텔등급 / 혼수가전: 패키지가·카드할인·무료배송설치·보증
  - 예물: 시작가·금속·다이아4C·평생보증 / 청첩장모임: 수용인원·코키지·룸·시그니처
  - **공통**: 추가비용·주차(발렛)·영업시간·위치 (아이웨딩 패턴). 가격은 "시작가~ +추가금" 투명표기.

### 4.2 상품 상세페이지 `/product/:id`(신규, R11~R15)
- 노출: 썸네일(없으면 업체 대표이미지)·**상세 이미지 다중**·설명·가격·관련 포트폴리오.
- 하단 CTA(스티키):
  - `payment_enabled` ON → **[구매하기]**(카카오페이) + **[문의하기]**
  - OFF → **[문의하기]**
  - 문의하기 → **전화(tel:)** / **인앱 채팅(`PlaceInquirySheet`)** 선택(액션시트).
- 상품 카드(`PlaceBusinessSections`) 클릭 → 이 페이지로 이동(모달 대체).

## 6. 결제(카카오페이) 설계 (R14, Phase C — 신중)
- 원칙: **업체 직접 정산**(플랫폼 자금 미보유), 서버에서 금액 산정(클라 금액 불신).
- edge function `kakao-pay-order-ready`(+approve) 확장: `products`에 없으면 **`business_products`** 조회(`moderation_status=approved` AND `payment_enabled=true`)해 name·price로 주문 생성. order_items엔 스냅샷 저장.
- 플로우: 상품상세 [구매하기] → ready(주문 생성) → 카카오 결제 → PaymentSuccess→approve → 완료/주문내역.
- 검증: esbuild 번들 + 타입/RPC 인자 교차확인. **실결제 e2e는 sandbox 불가 → 실환경 검증 필수**(§11).

## 7. 비용 안내 가이드 (R16, Phase D)
- 모델: **수령액 = 판매가 − 결제수수료 − (결제수수료의 VAT 10%) − 플랫폼수수료(초기제휴 0)**.
- 안전 안내치: 결제수수료 **1.62% + VAT**(설정 상수 `PAYMENT_FEE_RATE`). 예) 10만원 → 수수료 1,620원 + VAT 162원 → **수령 약 98,218원**, 플랫폼 수수료 0.
- 컴포넌트 `FeeGuide`(상품/이벤트/쿠폰 등록 폼 공용): 입력 가격에 따라 **정산 예상금액 실시간 미리보기**(스마트스토어 패턴). "초기 제휴 업체 입점 수수료 무료" 명시.
- 부가세 주의(§9): 판매대금은 업체 매출(업체 신고), PG 수수료 부가세는 PG가 업체에 세금계산서. 플랫폼 수수료 0 → 플랫폼 과세매출 0.

## 8. 경쟁사 분석 (260619 웹조사) + 적용 (R17)
- **상세 정보구조**: 아이웨딩=추가비용·발렛비·영업시간 + 스드메 **장바구니·쿠폰 투명가격**; 웨딩북=실시간 상담·비교견적·후기·즉시예약. → §4.1 핵심스펙·투명가격 반영.
- **웨딩홀 스펙**: 식대(1인)+대관료+**최소보증인원**(미달도 그 인원 식대) 3대 결정요소.
- **결제 문화**: 계약금(~300만원)+잔금 분할이 일반 → **고액=상담/예약금, 즉시결제=소액·부가**. → §3-③ 상품별 토글 타당성.
- **수수료**: 카카오페이 카드 1.62%/머니 1.45%/영세 0.38%. 마켓은 결제+판매수수료 분리(네이버 1.98~3.63%+α, 카카오선물 ~10%대). → 우리는 판매수수료 0.
- **셀러도구**: 이미지 기반 상세 + **정산 예상 미리보기** 표준 → §7 반영.
- 출처: iwedding.co.kr · weddingbook.com · ikidari.co.kr · gongysd.com · partner.kakaopay.com · zdnet.co.kr · windly.cc · allra.co.kr · sellerking.io · ebn.co.kr

## 9. 규제·세무 체크리스트 (일반정보 — 확정은 공식기관/전문가) (R18)
1. **통신판매중개 — 별도 신고/면허 불필요**. 기존 **통신판매업 신고로 충분**. 단 **중개자 고지**("당사자 아님" 표시) + **판매자 신원정보 제공** 의무(미이행 시 연대책임). PG 사용과 무관. → **앱에 중개자 고지 문구 삽입(운영 작업)**.
2. **PG(전금법)**: 외부 PG 사용 + **정산 직접 안 함 → 등록 의무 없음**. 정산대행하면 등록 필요. → §3-⑦ 직접정산 원칙.
3. **부가세**: 판매대금=업체 매출(업체 신고). 플랫폼 수수료 0 → 과세매출 0. 수수료 받기 시작하면 신고. PG 수수료 부가세는 PG가 업체에 세금계산서.
4. **에스크로**: 카카오페이 등 안전결제 사용으로 충족.
- 문의처: 공정위 1670-0007·관할 구청·정부24(중개) / 금융위·금감원·핀테크지원센터(PG) / 국세청 126·홈택스·세무사(세무).

## 10. 단계별 구현 (각 Phase = 별도 PR, 순차)
### Phase A — 상품 상세페이지 + 문의하기 (결제 무관, 즉시 검증)
- 작업: `ProductDetailPage`(`/product/:id`) 신규(이미지·설명·가격·관련 포트폴리오), 문의하기 CTA(전화/`PlaceInquirySheet`), `PlaceBusinessSections` 카드→상세 이동, App 라우트. 이벤트 신청하기=인앱문의 연결(§3-②).
- 수용기준: 카드 클릭→상세 진입, 문의하기→전화/채팅 동작, 썸네일 폴백, 결제미지원 상품은 문의하기만.
- 검증: build·lint·UI e2e(목 로그인). 결제 무관 → 완전 검증 가능.

### Phase B — 기본정보 구조화 (R10)
- 작업: `PlaceKeyFacts`(카테고리별 핵심스펙 맵) + 업체 소개 노출. 값 없으면 생략.
- 수용기준: 각 카테고리 기본정보에 핵심스펙·소개 노출(없는 값 빈칸 안 생김).
- 검증: 카테고리별 렌더 e2e(목).

### Phase C — 카카오페이 구매 (R14·R15)
- 작업: 마이그(payment_enabled·detail_images) + types 동기화 + `BusinessProducts` 폼(토글·다중이미지) + edge function 확장 + 상품상세 [구매하기]→결제.
- 수용기준: 토글 ON 상품에 구매하기 노출→ready 호출까지 동작(정적·esbuild).
