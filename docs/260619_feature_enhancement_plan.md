# 260619 기능 추가·고도화 계획 (경쟁사 전수조사 선행)

> AGENTS.md "분석 선행 — 경쟁사 분석 선행" 원칙에 따라 **타 업체 전수조사를 먼저** 수행한 뒤 작성.
> 근거: ① 경쟁사 기능 매트릭스(이 문서 §1) ② 라이브 DB 실측(`docs/260619_codereview.md` §라이브 DB).
> 전수조사 원본(출처 링크·상세)은 같은 감사 세션에서 수집했으며 핵심만 본 문서로 응축.

## 0. 두 가지 핵심 전제 (의사결정 기준선)

1. **Dewy 의 진짜 해자 = 생성형 AI 비주얼 스택**. 국내 경쟁사(웨딩북·메링·아이웨딩·오딩·신부야)의 AI 는
   전부 **조건형 추천 · 자동화 디렉팅 · 챗봇 상담** 수준. Dewy 만 **드레스 피팅·헤어 프리뷰·메이크업·사진 보정·
   퍼스널컬러 컨설팅 리포트**를 실제 구현(코드 실측). "상상→실행" 퍼널을 유일하게 점유 가능.
2. **라이브 DB 는 pre-launch**: 활성 업체 4,213(콘텐츠는 충실)이나 **partner_rank>0 = 0 · 활성 partner_deals = 0 ·
   community_posts = 0 · inquiries = 0**. → **공급(제휴·딜·리뷰) 충전과 입점 세일즈가 기능보다 먼저인 병목**.
   "또 새 기능"보다 **이미 구현된 레이어를 데이터로 채워 작동시키는" 고도화가 ROI 가 높다.

## 2. 고도화 로드맵 (우선순위 = 임팩트÷비용, pre-launch 전제 반영)

### P0 — 지금 당장(이미 구현된 걸 "작동"시키기, 신규 개발 최소)
가장 큰 레버리지는 신규 기능이 아니라 **빈 레이어 충전 + AI 비주얼→견적 연결**.

- **A. AI 비주얼 → 견적/예약 딥링크 퍼널 (해자 활성화, 차별화 최대)**
  - 현재: 드레스 피팅·헤어 프리뷰·메이크업 결과가 **종착점**(저장/공유까지). 경쟁사가 못 가진 "상상→실행"을 완성하려면
    결과 화면에 **"이 스타일 가능한 업체 보기 → 견적 요청"** CTA 를 붙여 기존 Quote/디렉터리로 딥링크.
  - 비용: 낮음(기존 Quote·디렉터리·placeMappers 재사용). 신규 테이블 불필요.
  - 검증 게이트: 추천은 **반드시 큐레이션 필터**(is_active·moderation·partner_rank 정렬, 0건이면 섹션 숨김 — AGENTS.md).
- **B. 제휴/딜 공급 충전 + 노출 (partner_rank=0, deals=0 해소)**
  - 라이브 DB 에 partner_rank·partner_deals 가 비어 혜택 피드가 사실상 공백. **입점 세일즈 + 어드민 충전 워크플로**가
    선행돼야 이미 만든 Deals·쿠폰·이벤트 UI 가 가치 발생. (기능이 아니라 **운영/콘텐츠** 과제 — 이게 진짜 병목.)
- **C. 캐시백/포인트 "얼마 돌려받나" 명시 (인프라O, 노출 갭)**
  - 포인트·하트·레퍼럴 인프라는 있으나 경쟁사처럼 "최대 N% / N만P" 가 사용자 의사결정 지점(상세·견적)에 안 보일 수 있음.
    상세/견적 화면에 **예상 적립/환급 배지**를 단일 소스(priceFormat)로 노출. 비용 낮음.

### P1 — 다음(저비용·데이터 재사용 갭 메우기)
- **D. AI 좌석배치(seating chart)** — RSVP/하객 데이터가 **이미 있음** → Zola 형 자동 배정(테이블 수·관계·노쇼 확률)
  을 AI 로. 데이터 재사용이라 저비용, 해외만 가진 기능을 국내 선점.
- **E. 크라우드소싱 견적 투명성** — 신부야 포켓/웨딩의여신처럼 **익명 실계약 견적 공유 DB**. 한국 시장 1순위 페인(가격
  불투명) 직격. 단 **DB 신설 → "DB 작업 전 테이블 선확인" 게이트** 필수(기존 quotes 재사용 가능성부터 분석).
- **F. 맥락 누적형 LLM 플래너 강화** — AI 플래너가 개인 맥락(예산·취향·본 업체·일정)을 누적해 **동적 체크리스트 +
  다음 액션 제안 + 견적 시뮬레이션**까지 끝까지 수행. The Knot ChatGPT앱/Zola 태스크분담의 한국형 상위호환.

### P2 — 탐색(검증·현지화 필요)
- **G. 실시간 홀 슬롯 연동** — 웨딩북 핵심 해자지만 **공급사(웨딩홀) 캘린더 연동 = 영업·통합 난이도 최상**. 먼저
  Dewy 현 예약이 메시징 기반인지 슬롯 기반인지 **코드 재확인** 후 ROI 판단(섣불리 착수 금지).
- **H. 레지스트리 현지화** — 한국 축의금 문화상 직이식 불가 → "신혼 위시리스트 + 축의 디지털 송금/감사장" 형태로
  재설계 필요. 세무/법무(대금 예수금) 확인 선행(디자인 마켓 OFF 와 동일 사유).

## 3. 착수 전 필수 게이트 (AGENTS.md 준수)
- **DB 신설 기능(E·G·H)**: 착수 전 `src/integrations/supabase/types.ts` → migrations → 기존 `.from()`/hook
  순으로 **기존 테이블/컬럼 재사용 가능성 분석**(중복 생성 금지). 라이브 DB 교차확인.
- **추천/관련 섹션(A·D·F)**: 큐레이션 게이트(활성·partner_rank·다양성·0건 숨김) 의무.
- **결제 연계(C 환급·H)**: 머니패스 실 호출 검증 동반 — 정적 통과만으로 완료 보고 금지.
- **모든 신규 CTA**: placeholder/toast-only 금지(페르소나 walkthrough 로 끝까지 동작 확인).
