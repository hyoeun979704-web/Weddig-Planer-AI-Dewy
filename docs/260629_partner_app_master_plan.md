## 1. 한 문장 비전
**"사장님이 포트폴리오를 취향 라벨과 함께 적극 등록 → 그 취향의 신부에게 우선 추천 → 고가치 리드 → 더 많은 포트폴리오"** 의 **선순환 루프**를, 1인 운영도 굴릴 수 있게 만드는 게 사장님 앱이다.
- 업계 표준 5원리(허브·파이프라인·완성도보상·행동랭킹·자동화)는 **갖추되**,
- Dewy의 +1 = **취향(무드)·페르소나 신호를 사장 도구에 결합**한 **공급자용 초개인화**.

## 2. 배경·전제 (왜 두 앱인가)
- **백엔드 1개(Supabase) 공유, 네이티브 앱은 2개** 출시: 소비자 / 사장님(기업회원 전용). 쿠팡·배민·네이버처럼 역할별 분리.
- 소비자 안드로이드는 **이미 출시**(연속성 필수), 소비자 iOS·사장님 앱은 미출시.
- 코드 구조: 모노레포(`apps/* + packages/*`). 공유 코드는 `packages/{lib,db,ui,hooks,shared}`, 사장님 전용은 `apps/partners` + `src/features/partners`.
- 원칙: **안전한 분리 우선**(기능 추가보다), 각 단계 **web·capacitor 양빌드 + 소비자 회귀 0 + lint/test/integrity 녹색 + DB 선확인**.

## 3. 경쟁사에서 가져온 5원리 (입력① 요약 — 이 기획의 베팅)
1. **단일 허브 + "오늘 할 일" 액션큐** — 흩어진 메뉴 대신 미응답 리드·미답변 문의·미답글 리뷰·임박 일정을 한 화면 할 일로. (1인 운영 전제)
2. **상태머신 파이프라인 — "다음 액션 1개"** — 리드/문의를 `신규→상담→견적→계약→행사완료`로 고정 단계화, 버튼 하나로 전이.
3. **완성도 → 노출 보상 루프** — "채울수록 검색/추천 상단"을 게이지+체크리스트+카피로 명시. *(근거: 추천 정렬이 `places.data_completeness` 사용 — 실제 동작)*
4. **행동·품질 기반 랭킹 + 신뢰 배지** — 응답률·이행률·리뷰를 노출 신호로(가격이 아니라 행동으로 경쟁).
5. **자동화로 1인 부담 제거** — 자동 첫 응답·견적 템플릿·방치/임박 자동 리마인더.

**채택 매트릭스 A~K / 우선순위 P0~P2** 는 입력① §3·§4 그대로 유효(중복 게재 생략, 단일 소스 유지).

## 4. 현재 구조 진단 (고쳐야 할 3가지 — 실측)
1. **대시보드가 "런처"** — `BusinessDashboard` 메뉴 12개 평탄 나열·`badge` 전부 null → 정보 냄새 0. 핵심 업무(리드·문의·리뷰)가 1회성 설정과 동급.
2. **상세정보 입력이 덤프폼** — `BusinessVendorEdit`(507줄) 한 화면에 저장 버튼 3개(기본/연락처/업종별), **가이드·예시 0** → 대충 적으면 저퀄 그대로 노출. 완성도 게이지는 **기본 7필드만** 측정(상세·포트폴리오·무드 누락 → "100%"가 거짓 가능).
3. **개인화 루프가 좁다** — 취향 매칭이 `VendorList`에만 적용, 메인 추천 `usePlaceRecommendations`는 `partner_rank` 정렬만 → 취향이 본진에 없음. 공급자 태그는 자유텍스트라 6 무드와 드리프트.

**원칙**: 공급자 태깅·소비자 콜드스타트·추천 매칭이 **하나의 어휘**를 본다(드리프트 차단).

| 공간/연출 | 가든/야외·채플·호텔·하우스·스튜디오·한옥 | 세밀 랭킹·콜드스타트 다양성 | 신규 확장 |
| 컬러톤 | 퓨어화이트·아이보리·파스텔·딥/보르도·뉴트럴 | 세밀 랭킹 | `dressFilters` 일부 재사용 |

## 6. 사용자 플로우 2종

### 6-A. 신규회원 — 가이드형 점진 온보딩
각 단계가 "왜 좋은가"(이점)를 먼저 보여주고 완성도로 보상. 상품·이벤트는 건너뛰기 허용.
1. **기업회원 가입** — 제휴 장점(무료·무수수료·신혼부부 노출). (현 `BusinessOnboard`: 사업자 인증+카테고리+제휴신청)
2. **상세정보 입력** — **가이드형 위저드**(§8)로 진입. 업종별 분기 + 완성도 게이지.
3. **포트폴리오 등록** ★ — 앨범(폴더) + 10장 일괄 + 무드 피커. (개인화 연료)
4. **상품 등록** — 견적 정확·전환↑ 이점. 건너뛰기 O.
5. **이벤트 등록** — 노출·프로모션 이점. 건너뛰기 O.
6. **관리 가이드** — 상세/상품/이벤트/리뷰 관리법(현 `BusinessGuide`).
7. **대시보드 안내** — 오늘 할 일·리드 응대 흐름 → 기존 플로우 합류.

### 6-B. 기존회원 — 액션 우선 대시보드
런처 → 업무 도구. §7 참조.

## 7. 대시보드 IA 재편 (런처 → 액션 3단)
1. **오늘 할 일 (액션큐)** — 미응답 리드·미답변 문의(`status=open`)·미답글 리뷰(`owner_response IS NULL`)·임박 이벤트(`starts_at` 근접). **실제 카운트 뱃지**, 0이면 "처리할 일 없어요" 청정 상태. 푸시→탭→그 자리 처리.
2. **핵심 3숫자 + 1제안** — 허영 4개(조회/찜/쿠폰/사진) 폐기 → 의미있는 3개(이번주 신규 리드·응답률·완성도/포트폴리오 충실도) + 제안 1개("포트폴리오 N장·태깅 X% → 취향 추천 노출↑").
3. **관리 (저빈도, 그룹핑)** — 정보/사진 · 프로모션(쿠폰·이벤트·상품) · 1회성(인수·디자인·가이드). 고빈도(리드·문의·리뷰)는 1번 액션큐로 승격.

## 8. 가이드형 상세정보 입력 위저드 ("따라하면 고퀄이 완성")
> 네이버 스마트플레이스·숨고식 질문형 등록을 Dewy에 적용. **덤프폼 3개 → 단일 가이드 위저드.** 사장은 "무엇을 적을지" 고민하지 않고, 질문이 끌고 가고, 끝까지 따라가면 고퀄 리스팅 완성.

### 8.1 위저드 step (업종 적응 = `service_category` 분기, 필수 먼저·심화 나중)
**[1차 = 노출 최소조건, 짧게]**
1. **대표 이미지 + 업체명** — 좋은 대표사진 예시(밝고 정면), 권장 비율(3:4), 나쁜예(어둡고 흔들림).
2. **상세 소개글** ★ — 업종별 골격 placeholder + 좋은예/나쁜예 + 글자수 힌트(100자↑ green). 빈칸 공포 제거.
3. **지역·매장·주소** — 매장 유무 분기(현행 중복판정 로직 그대로).
4. **가격 + 키워드** — 키워드는 자유텍스트 대신 **업종별 추천 키워드 칩**(통제어휘=검색 매칭 보존). → 여기까지 충족 시 **"이제 노출돼요" 1차 완료**.

**[2차 = 고퀄·개인화, 권장·건너뛰기 가능]**
5. **업종별 상세**(`DETAIL_SCHEMA` 재사용) — 각 옵션에 "왜 중요한지" 마이크로카피. 발견 갭 제거.
6. **취향(무드) 태깅** — §5 무드 피커. "이 무드의 신부에게 우선 노출".
7. **연락처·운영정보**(`BusinessListingContactForm` 재사용).
8. **포트폴리오**(앨범·10장·무드) — 업종 가중(스튜디오·드레스 핵심).
9. **문의 받는 방법** — 동작 검증(죽은 버튼 방지).
10. **미리보기 + 검토 요청** — 실제 `/vendor/:id` 렌더 재사용(목업 금지).

### 8.2 각 step "가이드 2종" + 실시간 힌트
- **왜**: 노출/전환 영향("소개 100자↑면 체류·문의↑").
- **예시(좋은예/나쁜예)**: 업종별 실제 샘플 — 설명 대신 **예시가 말하게**.
- **실시간 품질힌트**: 글자수·사진매수·필드충족 → green/amber inline 넛지("조금 더 적어볼까요?"). 강요 아님.

### 8.3 동기부여 카피 — "디테일·소비자친화 = 노출↑" (담백체)
**근거(사실)**: 완성도↑ = 추천 상단 노출↑(`businessListingCompleteness.ts` 주석에 명시된 실동작) + 무드 태깅 = 그 취향 신부 우선 매칭. → 정직하게 약속 가능.
**톤**: AI식 설명체 금지. 토스·당근·네이버 비즈니스처럼 짧고 담백, 구체 숫자·행동. "~돼요" 남발·작위적 비유 빼고 한 줄에 하나.
1. **노출(지금)**: *"자세할수록 위에 떠요."* / 게이지 옆 *"3개 더 채우면 노출↑"*(추상 % 대신 남은 항목 수).
2. **소비자 친화**: *"신부 눈높이로 적어주세요."* — 나쁜예 *"업력 20년 프리미엄"* / 좋은예 *"실내 가든홀, 하객 200명, 주차 100대"*.
- 작성 규칙: 한 화면 1~2줄, 명령형/체언종결 OK, 이모지·느낌표 남발 금지. "유리/우선"은 쓰되 "보장" 금지.

### 8.4 완성도 게이지 확장 (100% = 진짜 고퀄·매칭가능)
`computeListingCompleteness`를 **기본+상세+연락처+무드+포트폴리오 통합 spine**으로 확장 + **업종별 가중**(스튜디오=포트폴리오↑, 네일=가격·예약↑). 무드 포함 → "100%인데 매칭 0" 차단.

### 8.5 품질 게이트는 부드럽게
필수(이름·이미지·소개·지역·문의)만 하드(미충족 시 검토요청 비활성). 권장(상세·무드·포트폴리오)은 "건너뛰기 가능하나 노출 손해" 넛지. 강제·허위입력 방지.

### 8.6 재사용/델타 (중복 금지)
**재사용**: `DETAIL_SCHEMA`·`BusinessListingContactForm`·`computeListingCompleteness`(확장)·`formDraft`(자동저장·이어하기)·`ImageUploader`·저장 RPC(`saveListing`·`upsert_my_listing_detail`·contact) 그대로. **델타**: ① step 흐름 셸 + 가이드 카피 레이어 ② 업종별 예시·키워드칩 **단일소스 `partnerListingGuide.ts`**(코드분기 금지) ③ 완성도 spine 확장 ④ 무드 피커. **백엔드 저장경로 무변경**.

## 9. 포트폴리오 = 개인화 연료 (앨범 폴더 · 인스타 피드)
> AI 자동태깅 폐기 → **각자 입력(통제 피커) + 10장 일괄 + 폴더링 + 인스타 피드**. 대부분 `place_media_albums`로 이미 구현된 모델 — 델타만.

**공급자(BusinessGallery 강화)**
- 앨범=폴더(있음), 앨범 공통 메타 1회 → 사진 다수 귀속(있음).
- 델타①: 단건 → **10장 멀티 업로드**. 델타②★: `style_tags` **자유텍스트 → 무드 통제 피커**(+선택 공간/컬러).
- 보상 가시화: *"#모던 앨범이 모던 취향 신부 N명에게 노출"*.

**소비자(PlaceBusinessSections/VendorList 강화)**
- **인스타 피드** 형태 앨범 그리드(친숙 UI). 앨범 스타일 필터 있음.
- 델타③★: 피드 순서를 **그 신부의 취향(taste ∩ 앨범 무드)으로 정렬**.

## A. 적대적 분석 → 개선 (이 기획을 내가 깐다)
> "내가 사장/대표/투자자면 이거 충분한가?" 통과 못 한 지점과 **설계 수정**. (설계 R1~R7 + 위저드 W1~W6 + 신규 M1~M3 통합)

### A.1 루프·매칭 리스크
- **R1 🔴 양면 콜드스타트** — 매칭은 *소비자 취향 ∧ 공급자 태깅* 둘 다 있어야 작동. 신규 신부·신규/기존 4천 무태깅 업체면 `∩`=0 → "초개인화"가 빈손.
  → **개선**: ① 취향 0이면 `partner_rank` 폴백. ② **기존 free-text 정규화 배치(무드 매핑) + 무태깅 업체 "대표 무드 1개" 1탭 백필**을 S0 필수 선행으로. ③ 신규는 온보딩서 무드 최소 1개 강력 권장(강제 아님).
- **R2 ⚠️ 무드 6개로 "구체 라벨" 충분한가** — 넷플릭스식 구체성 기대 대비 거침.
  → **개선**: 매칭은 무드(밀도 보장), **콜드스타트/랭킹은 공간·컬러 세밀축 + 실제 앨범 이미지**로. 사용자 체감="구체 이미지로 고름", 시스템=매칭 밀도 유지.
- **R5 🔴 메인 추천 취향 확장 회귀** — `usePlaceRecommendations`는 전 소비자 surface 사용 → 정렬/성능/0건 깨지면 광범위 회귀.
  → **개선**: 취향은 **부스트(tie-breaker)로만**, `partner_rank` 1차 정렬 보존, 취향0/매칭0이면 현행 동일, 피처플래그 + 캡처 시뮬.

### A.2 사장 부담·표현·이탈 리스크
- **R3 ⚠️ "포트폴리오 적극 등록" 강요가 1인 부담** — 네일·축의대·폐백엔 압박.
  → **개선**: 넛지 **업종별 가중**(§10 포폴 가중 열). "오늘 3개만" 단계적, 빨간 폭탄 금지.
- **R4 ⚠️ 통제 피커 = 표현 자유 상실.**
  → **개선**: **피커(매칭용 무드) + 자유 캡션(표현용, 매칭 미사용)** 병행(label vs value 분리).
- **W1 🔴 위저드 강제 = 기존 4천·바쁜 사장 이탈.**
  → **개선**: 위저드=신규 기본 경로, **기존/숙련자는 "전체 한 화면 편집"(현 `BusinessVendorEdit`) 토글 유지** — 같은 저장·완성도 공유. 단일경로 강제 금지.

  → **개선**: data_completeness **산식 변경은 별도 단계**로 격리 + 점수 분포 before/after 비교 + 점진 적용. 위저드 게이지(UX)와 추천 산식(백엔드)을 **분리**해 먼저 UX만, 산식은 검증 후.

- **S0**: 기존 태그 정규화 + 무태깅 백필 **필수 선행**(R1). **`etc` 스키마 결정**(M2) 추가.
- **S1.5**: 위저드는 **기존 저장·draft 100% 공유**(M1), 전체편집 토글 유지(W1), 예시 단일소스(W3).
- **S2/S3**: 완성도 **UX 게이지 ↔ 추천 data_completeness 산식 분리**(M3), 취향은 **부스트만**(R5), 측정 동시(R7).
