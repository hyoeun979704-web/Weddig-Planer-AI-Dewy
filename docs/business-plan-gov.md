## 1. 문제 인식 (Problem)

### 1.1 창업 아이템 개요

한국 결혼 준비 시장은 1인당 평균 6~12개월의 준비 기간과 3,000만~7,000만원 비용이 발생하는 고관여 의사결정 영역이다. 그러나 기존 플랫폼은 표준 결혼식 사용자를 대상으로 동일한 정보를 제공하며, **사용자의 페르소나·진행 단계·지역·예산·특이사항(임신·재혼·국제결혼·스몰웨딩 등)에 따른 차별화된 큐레이션이 부재**하다.

**듀이(dewy)는 사용자별 초개인화(Hyper-personalization) 큐레이션 엔진을 기반으로, 각 사용자에게 자신의 결혼 준비에 필요한 정보만을 선별 노출하는 통합 플랫폼**이다. 결혼은 첫 적용 도메인이며, 동일 엔진을 라이프스테이지(신혼·기념일·영유아 등) 전반으로 확장한다.

### 1.2 문제 정의

| 핵심 문제 | 영향 받는 사용자 |
|---|---|
| 정보 과다 → 의사결정 부담 | 표준 결혼식 (P1~P10, 약 50%) |
| 비표준 케이스 가이드 부재 | 스몰웨딩 (P11~P14), 셀프·노웨딩(P15~P16), 임신·재혼·국제결혼(P17~P20) |
| 지역·예산별 맞춤 정보 부재 | 지방 거주자, 저예산층 |
| 가족 갈등·멘탈 케어 도구 부재 | 전 페르소나 |

### 1.3 외부 환경

| 동인 | 내용 |
|---|---|
| 인구·사회 | 혼인 건수 감소(2014→2023 약 40%↓)에도 평균 결혼 비용은 증가, 사용자당 객단가↑ |
| 기술 | LLM·이미지 생성 AI 상용화 (gpt-image-2, Gemini), 결혼 도메인 미특화 영역 |
| 소비 | 모바일 의사결정 70%↑, 시각 자료 중심, 비교·후기 기반 |
| 사용자 | 스몰웨딩·셀프웨딩 3~4배 증가, 비표준 케이스 비중 확대 |

### 1.4 사회·경제적 기여

- **사용자**: 결혼 준비 의사결정 부담·정보 비대칭 완화
- **소상공인 사업자**: 셀프 입점·마케팅 분석 콘솔로 디지털 마케팅 진입 장벽 완화 (B2B 콘솔)
- **고용·창업 생태계**: 1인 풀스택 자동화 모델 검증, 향후 1인 창업·마이크로 SaaS 표준 제시 가능성

|---|---|---|
| 페르소나 기반 모드 | ✓ | ✗ |
| 초개인화 큐레이션 | ✓ | ✗ |
| AI 챗봇 (도메인 특화) | ✓ | ✗ |

## 3. 성장 전략 (Scale-up)

### 3.1 자금 운용 계획 (지원금 5천만원 가정)

| 항목 | 비중 | 금액 |
|---|---|---|
| 마케팅·광고 (CAC 검증, Phase 4 확대) | 40% | 2,000만 |
| API 비용 (AI Studio 운영) | 30% | 1,500만 |
| 인프라·도구·분석 (Supabase Pro, GA4·Mixpanel) | 10% | 500만 |
| 외주 (법무·세무·디자인) | 10% | 500만 |
| 비상 운영 자금 | 10% | 500만 |

> 1억원 확보 시 마케팅 50% + API 20% + 인플루언서 협업 10% + 인프라·외주 20% 구조로 재편 (본 계획서 §7바 참조).

### 3.2 시장 진입 및 성과 창출

| 분기 | 가입 | D7 활성 | 결제 전환 | 매출 (Base) |
|---|---|---|---|---|
| Q1 | 100명/주 | 30% | — | 0원 (베타) |
| Q2 | 200명/주 | 35% | 3% | 100~200만 |
| Q3 | 300명/주 | 40% | 5% | 250~400만 |
| Q4 | 400명/주 | 45% | 7% | 500~800만 |

**1년차 종합 (Base)**: 결제 사용자 약 250명, 매출 약 1,400만원, 순이익 약 945만원.
**BEP**: Phase 3 종료(Day 90 전후), 결제 사용자 50명 도달 시.

### 3.3 위기 대응 및 리스크 관리

| 핵심 리스크 | 완화 전략 |
|---|---|
| 사용자 획득(CAC) 통제 실패 | 오가닉 우선 검증, CAC≤LTV/3 채널만 유료 확대 |
| 결혼 도메인 1회성 → LTV 한계 | 라이프스테이지 확장 메커니즘으로 활성 기간 6→24개월 |
| 대형 플랫폼(카카오·네이버·웨딩북) 진입 | 도메인 컨텍스트·개인 데이터 락인, 초개인화 정확도 누적 |
| AI 환각·결과물 분쟁 | "참고용" 고지, 사용자 검수 전제, 면책·환불 명시 |
| 외부 AI API 변동 | 프로바이더 추상화, 토큰 단가 마진 버퍼 |
| 개인정보(얼굴 사진) | 비공개 버킷·30일 자동 삭제·동의 절차 |
| 단일 인력 의존 | 매뉴얼 문서화, 자동화 강화 |
