## TL;DR — "엉망"의 실체
- **진짜 불 = 기업회원 멀티지점 반쪽 이행**: 관리 하위 8페이지가 아직 `get_my_listing`(단일) → 멀티지점
  계정은 **엉뚱한 지점 데이터**를 본다(마토니 통합 계정 포함). + `BusinessDesigns` place_id 필터 누락(권한 누수 우려).
- **운영자 어드민**: 대부분 동작함. 하드버그 거의 없음 → 체감 "엉망"은 **누락 도구(페이지네이션·필터·일괄·
  프리뷰) + UX 마찰 + 미완 2개**(촬영시안 "준비중", 인스타카드뉴스 "1단계").
- **사용자**: 하드 P0 0. dead-end(연락처 없는 업체 문의/전화가 toast만), 결제 리다이렉트 타임아웃 부재, 빈상태 CTA 부족.
- **공통 부채**: 로딩/빈/에러 **상태 누락**이 전 표면에 퍼져 "조용히 빈 화면" → 사용자에겐 "안 됨"으로 보임.

## 우선순위 로드맵 (이 순서로)

### 🔴 P0 — 1주차 (기능이 실제로 깨져 보이는 것)
1. **멀티지점 스코프 일괄 적용**(Phase 2b): `useBranches`로 8페이지(Products·Coupons·Events·Gallery·
   Inquiries·Deliveries·Reviews·Designs) 선택 지점 기준 로드. **`BusinessDesigns`는 `.eq("place_id")` 필수**(현재 무필터).
2. **권한 방어 점검**: `designer_designs`·`vendor_deliveries`·`place_reviews` 쓰기 RLS가 소유자 한정인지 확인,
   아니면 RPC/RLS로 가드(데이터 누수 차단).
3. **어드민 파트너십 검토** `p_note: undefined` → `null` (AdminBusinessReview.tsx:99).

### 🟠 P1 — 2주차 (dead-end = "되는 척하는" 버튼)
4. **연락처 없는 업체 CTA**: '전화 문의'/'문의하기'가 toast만 뜨는 경우 → 채널 없으면 **버튼 비활성/숨김 +
   claim 유도**(PlaceDetailLayout, VendorDetailPage).
5. **결제 리다이렉트 안정화**: HeartCharge·Checkout 의 `openExternal` 리다이렉트에 타임아웃/에러 토스트
   (멈춘 스피너 방지) — 카카오 연동 심사와도 직결.
6. **기업회원 dead-end/빈상태**: 문의·리뷰 빈 상태에 다음 행동 안내, claim 버튼 q= 프리필.

### ⚠️ P2 — 3주차 (UX 상태 일관화 — "조용한 실패" 제거)
7. **로딩/빈/에러 3상태 표준화**: 사용자·기업·어드민의 silent-empty 쿼리에 에러 토스트+스켈레톤
   (BusinessLeads·Gallery 병렬쿼리·AdminDashboard 16쿼리·검색 빈상태 등).
8. **어드민 목록 사용성**: AdminUsers 페이지네이션+역할필터, AdminPlaces 검색 피드백, 일괄작업 건수경고.

### 🎨 P3 — 4주차 (UI 일관성)
9. **공통 컴포넌트화**: raw `<button bg-emerald-500>` → `<Button variant>`(AdminPlaceClaims 등), 카드/간격/
   그리드 토큰 통일(BusinessDashboard 모바일 4칼럼→2칼럼, 이미지 비율 표준화).
10. **safe-area 잔여**(#336 후속): InvitationStudio 내부 패널·SupportChat·MergeGame 등.

### ➕ P4 — 백로그 (필요한데 없는 기능)
- 기업회원: 항목 일괄삭제/순서변경, 앨범 수정/삭제, 문의 상태필터, 리드 CSV/전환 분석.
- 어드민: 역할필터·CSV·문의 담당자 배정·프로모 모바일 프리뷰·리포트 일괄처리·검토 통합 카드.
- 사용자: 빈 업체목록 "전국/견적" CTA, 커뮤니티 "첫 글 작성" CTA.
- 미완 2개(촬영시안·인스타카드뉴스) 완성 또는 메뉴 가리기.
