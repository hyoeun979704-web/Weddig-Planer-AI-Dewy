## 자동 fix (이 PR)

### Fix 1. MenuSection 에 "내가 만든 것" 그룹 추가
새 그룹 (웨딩 준비 / 내가 만든 것 / 계정 / 고객 지원) — "웨딩 준비" 와 "계정" 사이에 위치:
- 📧 **내 청첩장** → `/invitation/my`
- 👗 **드레스 시뮬레이션** → `/ai-studio/dress-tour/gallery`
- ✨ **메이크업 시뮬레이션** → `/ai-studio/makeup-room/gallery`

신규 사용자에게도 "여기서 무엇을 만들 수 있는지" 시각적 힌트 역할.

### Fix 2. QuickMenuGrid 에 "하트" 추가
4열 → 5열 그리드로 확장. 하트는 핑크 강조 색상:
- 찜 / **하트** / 포인트 / 쿠폰 / 주문내역

하트 = AI 시뮬·청첩장 발행 결제 수단. user_hearts.balance 표시.
탭하면 `/points` (충전 페이지) 로 이동.

## 결정 필요 (사용자 보고)

### F. 포인트(`user_points`) vs 하트(`user_hearts`) 통합

**현 상태**: 두 자산이 분리돼 있고 QuickMenuGrid 에 둘 다 노출. 사용자에게 헷갈림.

- `user_points` — 출석·리워드 등으로 적립, 1포인트 = 1원 가치, 환금성 강조
- `user_hearts` — AI 시뮬·청첩장 결제 전용, 패키지로 구매 (예: 1000원 = 100하트)

**선택지**:
- **F1** — 그대로 (현재). 포인트는 "출석 보상", 하트는 "AI 결제 단위" 로 명확히 분리되어 있어 보존 가치 있음
- **F2** — 포인트만 사용. AI 결제도 포인트로. 1AI = N포인트
- **F3** — 하트만 사용. 출석 보상도 하트로. 적립 단위가 더 작아져야 함 (1포인트 적립 → 0.1하트?)
- **F4** — 자동 변환 (포인트 → 하트 1:1 또는 N:1)

→ **추천 F1** (현재 분리가 의미 있음). 다만 마이페이지 도움말에 "포인트 vs 하트 차이" 작은 안내 박스 추가 검토.

### G. 운영자 대시보드 카드 위치

**현 상태**: admin role 인 사용자에게만 D-Day 카드와 PartnerLinkCard 사이에 보라/핑크 그래디언트 카드 노출. PremiumBanner 위치보다 위.

**관찰**:
- 운영자에겐 자주 가는 메뉴라 위에 있는 게 합리적
- 다만 시각적으로 D-Day 같은 일반 콘텐츠와 섞여서 "운영자 모드" 임이 명확하지 않음

**선택지**:
- **G1** — 그대로 (admin 본인이 알아서 인식)
- **G2** — 헤더 상단에 작은 "👑 관리자" 뱃지 추가, 카드는 그대로
- **G3** — 카드를 헤더 바로 아래로 올림 (D-Day 보다 위에)

→ **추천 G1** (admin 사용자 본인이 무엇인지 인식, 추가 시각 강조 불필요).

## 그 외 발견 (작은 것, 보고만)

- **GuestMyPage** 의 stats ("2,400+ 등록 업체", "15,000+ 이용 커플" 등) 는 하드코딩된 placeholder 수치. 실제 운영 데이터로 교체하거나 일관성 있는 mock 수치 유지 결정 필요
- **`/profile`, `/notifications`, `/settings`** 등 메뉴가 가리키는 페이지가 실제로 어떤 상태인지 별도 검토 필요. 마이페이지 자체와는 무관
- **PartnerLinkCard** 의 동작 (`hideWhenLoggedOut` prop) 은 정상이지만, 로그인된 사용자가 partner 없을 때 어떻게 표시되는지는 컴포넌트 내부 로직이라 검토 미진행
