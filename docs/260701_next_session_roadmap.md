## 1. 고도화 3축 (데이터 현실 기반)

### 축 A — 만든 루프 "활성화"(데이터 유입 넛지). 공급 무관·저난이도·고레버리지
인증배지·가능일·무드·claim 인프라는 있으나 데이터가 0. 데이터를 만들 **동선/넛지**를 붙인다.
- **A1 후기 유입 넛지**: contract/consult 행동로그(견적·문의·식장등록) 있는 사용자에게 "이 업체
  상담하셨네요 — 후기 남기면 **인증 배지**가 붙어요" 유도(마이페이지·문의 완료 후·홈). 인증 infra를
  실제 콘텐츠로 전환하는 유일한 길. → 후기 0 탈출.
- **A2 claim 유도 훅**: 미입점 홀 상세의 빈 가능일/문의 자리를 "이 홀 담당자세요? 가능일·문의를
  직접 받으세요 → 입점" 으로(2번 선택으로 미뤘던 것, 데이터가 0인 지금 재검토 가치). 공급 유입 동기.
- **A3 무드태그 넛지(파트너)**: 갤러리 앨범 무드 미입력 파트너에게 "무드 태깅하면 취향 맞는 신부에게
  우선 노출" 안내 → taste 연료 확보 → TASTE_BOOST 점진 ON 준비.

### 축 B — 소비자 신호 개인화 심화(공급 무관). 우리가 가진 데이터로 가치
벤더 데이터가 비어도 페르소나·예식일·예산·명단·진행단계는 **사용자가 준다**. 이걸 교차 활용.
- **B1 통합 "내 준비" 홈 인텔리전스**: 흩어진 신호(D-day·예산 건강도·체크리스트 진행·식대 추정·
  하객수·리워드 진행·가능일 매칭)를 홈 페르소나 대시보드에 **한 카드로 종합**. 이미 만든 훅
  (useMealCostEstimate·usePlanningRewards·usePersonaInsights·useBudget·useGuestList) 재사용.
- **B2 좌석배치 ②(이월 해제)**: 홀 구조(table_count/seats_per_table)+하객 명단(side·relationship)이
  이제 있으므로 **관계 기반 자동 배치 초안** 생성 가능. `seating_layouts`(JSON) 신규 + 드래그 수정.
  데이터 적어도 소비자 자가입력(테이블 수)으로 동작하게 설계.
- **B3 AI 플래너 신호 연동**: 식대 추정·예산 건강도·페르소나 코칭을 AI 플래너 컨텍스트로 주입해
  "맞춤 다음 액션" 품질↑(생성형 개인화, 차원14 깊이 ⑤).

### 축 C — 측정·완성·품질
- **C1 개인화 계측(필수 선행)**: 신규 카드(식대·가격가이드·인증배지·컨텍스트·리워드·가능일)에
  `trackEvent`(노출·클릭·전환) 심기. **뭐가 작동하는지 모르면 고도화 방향을 못 잡는다.** 차원14 "측정".
- **C2 read receipts / typing(2-A 완성)**: quote_messages.read_at UPDATE RLS 정책 + 열람 시 읽음
  표시. 라이브챗 체감 완성.
- **C3 a11y 배치**: 구코드 Label htmlFor·아이콘 aria-label·터치타깃 44px(감사 이월).
- **C4 부팅 워치독**: getSession hang 시 스피너 무한 방지(복원력 이월).

## 2. 다음 세션 실행 계획표 (우선순위 순)

> 원칙: **저난이도·공급무관·고레버리지 먼저.** DB 변경은 AGENTS 게이트(테이블 선확인→마이그→
> types.ts→클라, RPC 인자 교차확인) + 라이브 실측 + e2e. 각 항목 독립 커밋/PR 권장.

| 순위 | 항목 | 축 | 난이도 | 선행조건 | 검증기준(완료 정의) |
|---|---|---|---|---|---|
| 1 | **C1 개인화 계측** | C | S | 없음 | 신규 카드 노출/클릭 이벤트가 client 로그에 남음, PII 없음 |
| 2 | **A1 후기 유입 넛지** | A | S~M | 행동로그 조회(RLS) | 상담/거래 이력 사용자에 후기 CTA 노출, 작성 시 인증배지 부여 e2e |
| 3 | **B1 통합 "내 준비" 홈 카드** | B | M | 기존 훅 재사용 | 로그인 홈에 종합 카드 렌더, 빈 신호 폴백, 각 신호 클릭→해당 도구 이동 |
| 4 | **A2 claim 유도 훅** | A | S | 기존 claim 퍼널 | 미입점 홀 상세 빈 자리→입점 CTA, dead-end 0 |
| 5 | **C2 read receipts** | C | M | read_at RLS 마이그 | 상대 열람 시 "읽음" 표시, RLS로 참여자만 mark-read |
| 6 | **B2 좌석배치 ②** | B | L | 홀구조·명단(有) | seating_layouts 마이그, 관계 자동배치 초안+드래그, 소비자 자가입력 폴백 |
| 7 | **B3 AI 플래너 신호 연동** | B | M | AI 플래너 컨텍스트 | 식대·예산·페르소나가 플래너 추천에 반영, 비용/캐싱 상한 |
| 8 | **A3 무드 넛지 + taste 점진 ON** | A | M | 무드 데이터 유입 | 파트너 무드 넛지, 데이터 임계 시 TASTE_BOOST flag 게이트 ON |
| 9 | **C3 a11y 배치 / C4 부팅 워치독** | C | S~M | 없음 | 구코드 a11y 수정, getSession 타임아웃 폴백 |

**추천 시작점: 1(계측) → 2(후기 넛지) → 3(통합 홈).** 계측이 먼저여야 이후 고도화의 효과를 안다.
