### A1. 인가/RLS (P0·P1)
- 🔴 **P0 — `get_place_inquiry_stats` RPC 무방비**(`20260615015619_*.sql:18`): 소유자 체크 없이 **아무나
  특정 업체의 문의량·응답률·평균응답시간 조회** 가능(프론트만 claimed 게이트 → 직접 RPC로 우회).
  **경쟁사 영업 정찰**. → RPC 본문에 `place_owner = auth.uid() OR has_role(admin)` 게이트. (~1일)
- 🟡 P2 — `community_author_cards` SECURITY DEFINER 뷰가 RLS 우회로 anon에 `wedding_style/role` 노출
  (민감도 낮음, advisor ERROR). → PG17 `security_invoker=on` 또는 anon SELECT REVOKE.
- 🟡 P2 — `client_error_logs`·`product_clicks`·`service_waitlist` INSERT 개방(`USING true`) → 스팸/DoS.
  → 버스트 제한(IP/세션당) 또는 항목 크기 제한.

  전부 owner-scoped RLS + 적립/차감은 service-role RPC만.

### A2. 머니패스(KakaoPay) — 서버검증·멱등 ✅ / 승인후 정합 일부 미처리(deferred)

- 🟡 **P2(하향) — 광고 보상 포인트**: 재조사 결과 **서버 캡 존재**(`add_game_points`가 doubled별

- 🔴 P1 — `send_quote_message` **버스트 제한 없음**(업체가 닫힌 견적에 스팸). → 10분 5건(RSVP 패턴 복제).

## B. 국내법 (한국)
### B1. 🔴 AI 기본법 — 생성형 AI 결과물 표시 의무 (2026-01-22 시행, 발효 중)
- 「인공지능 발전과 신뢰 기반 조성 등에 관한 기본법」 시행. **AI사업자는 생성형 AI 결과물이 AI로
  생성됐다는 사실을 이용자가 인지 가능하게 표시**해야 함. 예술·창작물은 향유 저해 안 하는 방식 허용.
  **계도기간 1년+**(즉시 과태료는 아니나 출시 전 반영 정석).
- **Dewy 영향**: 드레스·메이크업·헤어·촬영시안·청첩장 AI 이미지 전부 대상인데 **무라벨**(프롬프트에
  "no watermark" 명시) → 위반 소지.
- ★ **결정(확정)**: 이미지 위 워터마크가 아니라 **결과 페이지에 "AI 생성" 안내 문구**를 표시(이용자
  인지 가능 방식 충족, UX 깔끔). → 재사용 컴포넌트 `<AiDisclosureNotice>`를 AI 결과 페이지 6종
  (드레스·메이크업·헤어·SDM·포토픽스·컨설팅)에 배치. (앞선 "워터마크 X"는 마케팅 워터마크 — 별개 의무)
- **타 업체**: Canva·Pinterest 등 라벨/배지·C2PA. 우리는 **페이지 안내 문구**로 채택.

### B2. 🟡 다크패턴 — 전자상거래법 6종 금지(2025-02) + 공정위 지침(2025-10)
- "숨은 자동갱신(forced continuity)" 명시 금지 → **U3: UpgradeModal "자동 결제 없음"(거짓) 수정 +
  자동갱신 명시 + 결제 전 사전 리마인더**가 법적 필수(반복 확인).

### B3. 🔴 마케팅/야간 수신 동의 (PIPA·정보통신망법)
- `marketing_consent_at`은 저장되나 **명시적 opt-in UI/게이트가 코드에 없음**(가입 암묵동의 의심).
  **야간(21~08시) 광고성 정보 수신 별도 동의 부재**. → 푸시/SMS 마케팅 전 별도 동의 모달.

### B4. 이미 충족(✅, `launch_readiness_audit` 확인)
