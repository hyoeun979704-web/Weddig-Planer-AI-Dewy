# 모바일 청첩장 에디터 경쟁사 분석 리포트 (2026-06-14)

> 목적: Dewy 사용자가 "청첩장 편집 UI가 너무 불편하다"고 피드백. 한국·해외 주요 청첩장
> 서비스의 **편집(에디터) UX**를 조사해 Dewy 청첩장 편집 UI 개선과 e2e 시뮬레이션 문서
> 고도화에 쓸 권고를 도출한다.
>
> **조사 방법·한계**: 웹 검색(2026-06 기준) + Dewy 코드 직접 확인. 각 서비스 에디터를
> 직접 손으로 조작해 본 1차 사용성 테스트는 못 했고(샌드박스 한계), 공식 페이지·블로그
> 후기·앱스토어 리뷰·비교 글에 의존했다. 일부 한국 서비스(메이크 make.kr, 핫프린트)는
> 편집 UX에 대한 신뢰할 만한 1차 자료를 찾지 못했다 — 해당 항목은 "확인 못 함"으로 표기.
> 결론 일부는 일반 UX 베스트프랙티스에 기반하며 그 부분은 명시했다.

---

## TL;DR (핵심 결론)

1. **한국 모바일청첩장 시장의 사실상 표준 = "폼 입력 + 고정 템플릿 실시간 미리보기"**.
   데어무드, 투아워게스트, 필카드, 잇츠카드, 디얼디어 등 시장을 끌고 가는 셀프 제작 서비스는
   거의 전부 자유 캔버스가 아니라 **섹션(블록) 단위 폼 입력**이고, 사용자는 텍스트·사진을
   "필드에 채워 넣으면" 옆/아래 미리보기가 즉시 갱신된다. "10분 완성", "클릭 몇 번", "직관적"이
   이들의 핵심 마케팅 문구다.
2. **자유 캔버스(드래그로 텍스트·사진 위치 잡기)는 디자인 툴(미리캔버스·캔바·Adobe Express)의
   영역**이고, 청첩장 "전용" 서비스는 의도적으로 이를 피한다. 자유도는 높지만 초보자에게
   어렵고 모바일 터치에서 특히 고통스럽기 때문.
3. **Dewy 현재 에디터는 청첩장 전용 서비스인데도 자유 캔버스(Konva)** — 슬롯을 드래그로
   옮기고, 회전 슬라이더·+/− 크기·방향키 nudge로 미세 조정하는 "디자인 툴" 패턴이다.
   이것이 "편집이 불편하다"는 피드백의 가장 유력한 구조적 원인일 가능성이 높다. 시장 표준과
   반대 방향에 서 있다.
4. 해외(Paperless Post·Minted)에서 **자유 캔버스 + 모바일**이 낳는 불만이 매우 구체적으로
   기록돼 있다: "텍스트 편집 시 **키보드가 청첩장을 가려서** 입력 내용이 안 보인다", "편집할
   텍스트를 손가락으로 정확히 집기 어렵다", "버벅임/지연", "**앱은 폰·태블릿 제작을 위해 만든 게
   아니다**". Dewy가 그대로 답습하기 쉬운 함정들이다.

## 1. 한국 서비스

### 1-1. 데어무드(theirmood.com) — "마음"류 셀프 제작의 대표
- **편집 방식**: 폼 입력 + 고정 템플릿 미리보기. 사진 최대 40장, **섹션(블록) 순서 자유 변경**,
  워터마크 제거(유료) 후 **무제한 수정·즉시 반영**.
- **인터랙션**: 자유 캔버스 아님. 사진 디테일/투명도 등 "삽입 이미지 디테일 조절"까지 세세하게
  커스텀 가능하다는 후기 — 단 이는 **필드 기반 옵션**이지 캔버스 드래그가 아님.
- **온보딩**: 마이페이지 → "제작하기" 버튼 → 즉시 제작(셀프). 가입·결제 장벽 낮음.
- **칭찬**: "디자인이 세련·고급스러워 받는 분들이 예쁘다고 함", "통계(하객 응답)가 직관적",
  엑셀 추출. **불평**: 자료상 편집 UX 불만은 거의 없음(폼 기반이라 마찰 자체가 적음).
- 출처: theirmood.com, slowabc.com 비교글, 다수 후기.

### 1-2. 투아워게스트(toourguest.com) — "감도 높은 초대 10분 완성"
- **편집 방식**: 폼 입력 + 실시간 미리보기. **UI가 직관적이라 "클릭 몇 번"으로 수정**,
  수정 즉시 반영. 평균 **10분 완성**을 전면에 내세움.
- **모바일/PC**: "PC·태블릿으로 만들면 미리보기가 더 빠르고 편하다"고 안내 — 즉 **세밀 편집은
  큰 화면을 권장**(자유 캔버스 서비스의 전형적 한계 인정).
- **시사점**: "10분", "클릭 몇 번", "직관적"이 곧 이 카테고리의 **성공 지표**. 속도·단순함이
  KPI다.

### 1-3. 필카드(directwedcard.com)·필메이커(feelmaker.co.kr) — 무료 셀프
- **편집 방식**: 폼 입력 + **실시간 미리보기**. 배경음악 설정, 이미지 갤러리, 동영상 삽입 등
  **기능을 "토글/필드"로 제공**. 무료 강조.
- **시사점**: 음악·갤러리·동영상·지도 같은 "청첩장 구성요소"는 캔버스 오브젝트가 아니라
  **구조화된 섹션/설정**으로 다룬다 — 사용자는 "무엇을 넣을지"만 고르고 "어디에 어떻게"는
  템플릿이 책임진다.

### 1-4. 잇츠카드(itscard.co.kr)·디얼디어(deardeer.kr) — 인쇄+모바일 겸업
- 인쇄 청첩장 업체가 모바일을 곁들이는 형태. 모바일 청첩장은 **템플릿 선택 → 폼 입력** 흐름.
  바른손카드도 동일 계열(모바일청첩장 무료 제공, 식전영상 무료 등 번들 마케팅 위주).
- **편집 UX 1차 자료는 빈약** — 다만 카테고리 패턴(폼 입력)은 동일.

### 1-5. 바른손카드 / 카카오톡 전송
- **바른손카드(모청)**: 모바일청첩장은 부가서비스(무료 번들). 편집 자체보다 "혜택·할인"
  중심 마케팅. 제작·편집은 마이페이지에서 폼 기반 수정.
- **카카오톡**: 카카오는 청첩장 "에디터"를 제공하는 게 아니라 **전송 채널**이다. 대부분
  서비스가 "카카오톡 전송하기" 버튼으로 공유 링크를 발송하는 방식(달팽 등 안내). 즉
  카카오는 편집 UX 비교 대상이 아니라 **공유 채널 요구사항**으로 봐야 한다.

### 1-6. 미리캔버스 / 캔바(한국 사용) — 자유 캔버스(디자인 툴)
- **편집 방식**: **완전 자유 캔버스**. 드래그&드롭, 색·폰트 변경, 사진 업로드, 투명도 조절,
  동영상 삽입, 매직 리사이즈. 템플릿 2만+(캔바 기준).
- **포지션**: 청첩장 "전용"이 아니라 범용 디자인 툴로 청첩장을 "만들 수도" 있는 것. 자유도는
  최고지만 **초보자 학습곡선·모바일 정밀편집 어려움**이 구조적 단점(아래 해외 사례와 동일
  계열의 마찰). 자유 캔버스 청첩장의 단점에 대한 직접 후기는 충분히 못 모았으나, 패턴상
  Paperless Post·Minted와 같은 마찰을 공유한다고 보는 게 합리적.

**한국 종합**: 청첩장 **전용** 서비스 = 폼 입력 + 고정 템플릿. **범용 디자인 툴** = 자유
캔버스. 사용자가 "예쁜 청첩장을 빨리"를 원하면 전자가 압도적으로 선택받는다.

## 2. 해외 서비스

### 2-1. Canva (웨딩 인비테이션)
- **편집 방식**: 자유 캔버스 드래그&드롭. 모든 요소(텍스트·그래픽·아이콘·애니메이션·사진)
  조정. 2만+ 템플릿, 매직 리사이즈(인쇄/디지털 변환), iOS·Android 앱.
- **강점**: 자유도·자산 라이브러리. **약점**: 모바일 정밀편집·터치 핸들 조작은 데스크탑 대비
  불편(일반적으로 알려진 디자인 툴 한계). 청첩장 특화 흐름(하객·RSVP)은 약함.

### 2-2. Paperless Post — 자유 캔버스 + 디지털 RSVP (**가장 교훈적**)
- **편집 방식**: 카드 자유 편집(텍스트 박스 선택 후 **클릭&드래그로 이동**), 봉투형 프리미엄
  연출, 큐레이션 라이브러리, 코인 기반 과금.
- **구체적 불만(앱스토어/Trustpilot)**:
  - **텍스트 입력 시 키보드가 청첩장을 가려 입력 내용이 안 보임.**
  - **편집할 텍스트를 손가락으로 정확히 집기 어렵다**, 화면 축소가 잘 안 됨.
  - **버벅임·지연(stalling/lagging)**.
  - **"앱은 폰·태블릿에서 제작하라고 만든 게 아니다"** — 사실상 모바일 미지원.
  - 한 텍스트 박스 안에서 여러 서식 혼용 불가.
- **시사점**: 자유 캔버스 청첩장 에디터가 모바일에서 무엇을 망치는지 거의 **그대로 체크리스트**.
  Dewy의 Konva 에디터가 똑같이 빠지기 쉬운 함정. (출처: justuseapp/Trustpilot 리뷰)

### 2-3. Minted — 큐레이션 디자이너 카드 + 제약된 에디터
- **편집 방식**: 디자이너 작품 큐레이션 강점, 하지만 **에디터는 다른 툴보다 제약적**이고
  **데스크탑 지향**. 디지털 카드는 인터랙티브·모바일 친화(보기 측면). 텍스트 교체 위주의
  "안전한" 커스터마이즈.
- **시사점**: "자유도 낮춰서 결과물 품질을 보장"하는 전략 — 한국 폼 입력 패턴과 같은 철학.
  "예쁨"은 템플릿이 책임지고 사용자는 내용만 채운다.

### 2-4. Zola — 모바일 퍼스트 웹사이트+초대
- **편집 방식**: 모바일 퍼스트, 무료 티어 강함, RSVP·하객관리 통합. 자유 캔버스보다
  **구성요소 기반 편집**(섹션 채우기)에 가까움.

### 2-5. with Joy (withjoy.com) — 무료 웨딩 사이트
- **편집 방식**: "단순·직관적 에디터"로 URL 커스텀·사진/영상·정보 수정. 엑셀 게스트
  임포트, 통합 RSVP(질문·답변). **커스터마이즈가 많다고 호평**. 폼/섹션 기반.

**해외 종합**: 결과물 품질을 중시하는 쪽(Minted·Zola·withJoy)은 제약된/구성요소 기반 편집,
자유도를 파는 쪽(Canva·Paperless Post)은 캔버스 — 그리고 **캔버스 쪽이 모바일 사용성
불만이 압도적으로 많다.**

## 3. 종합 결론 — 표준 패턴

| 축 | 한국 청첩장 전용 | 디자인 툴(캔바·미리캔버스) | 해외 큐레이션(Minted·Zola) |
|---|---|---|---|
| 편집 모델 | **폼 입력 + 고정 템플릿** | 자유 캔버스 | 제약된 텍스트 교체 / 섹션 |
| 사진 배치 | 슬롯에 "채우기" | 드래그로 자유 배치 | 슬롯/프레임 |
| 텍스트 | 필드 입력(위치 고정) | 자유 이동·서식 | 정해진 영역 편집 |
| 미리보기 | 실시간(옆/아래) | 캔버스 자체 | 실시간 |
| KPI | **속도·단순함(10분)** | 자유도 | 디자인 품질 |
| 모바일 적합도 | 높음 | 낮음(정밀편집) | 중간 |

**핵심 통찰**: 청첩장은 "디자인 작업"이 아니라 **"양식 채우기 + 결과 확인"** 작업으로
사용자가 기대한다. 한국 시장 1군은 전부 이 기대에 맞춰 자유도를 **의도적으로 줄였고**,
그것이 "직관적 / 10분 / 클릭 몇 번"이라는 평판으로 돌아왔다. **Dewy의 자유 캔버스(Konva)는
이 표준에서 벗어나 있으며, 사용자 불만의 구조적 원인일 가능성이 가장 높다.**

> 단, Dewy 캔버스는 **종이 청첩장(인쇄, 300dpi PDF)**도 함께 다룬다
> (`InvitationStudio.tsx`, `exportPdf.ts`). 인쇄용 정밀 레이아웃에는 캔버스 자유도가
> 정당화될 수 있다. 따라서 권고의 핵심은 "캔버스를 버려라"가 아니라 **"모바일 청첩장 경로는
> 폼 입력을 기본 모드로, 자유 캔버스는 종이/고급 사용자용 선택 모드로 분리하라"**다.

## 4. Dewy 청첩장 편집 UI 개선 권고 (우선순위)

> Dewy 현황(코드 확인): `InvitationStudio.tsx`는 `wizard → template → studio` 3단계.
> studio는 Konva 캔버스(`InvitationCanvas.tsx`)로 **선택 슬롯만 draggable**, 회전 슬라이더,
> +/− 크기, 방향키 1px/Shift+10px nudge, 텍스트는 Textarea/Dialog 입력, undo/redo,
> 정렬 가이드(`onGuideChange`), AI 추천 시트(8초 미입력). 즉 자유 캔버스 + 보조도구 다수.

### A. 빠른 개선 (구조 변경 없이, 1~2주급)
1. **[P0] "내용 채우기(폼)" 모드를 기본 진입 모드로 추가.**
   studio 진입 시 자유 캔버스 대신, 템플릿의 텍스트/사진 슬롯을 **리스트형 필드**(신랑·신부
   이름, 일시, 장소, 인사말, 대표사진…)로 먼저 보여주고 옆/위에 실시간 미리보기. 캔버스
   드래그는 "자유 편집" 토글로 숨긴다. 시장 표준 + 가장 큰 마찰 제거.
2. **[P0] 텍스트 편집 시 키보드가 대상 텍스트를 가리지 않게.**
   (Paperless Post 1번 불만) 편집 중인 슬롯을 키보드 위 안전영역으로 **자동 스크롤/스케일**,
   또는 별도 바텀시트에서 입력하고 미리보기는 그 위에 유지. 현재 Dialog/Textarea 입력이
   캔버스를 가리는지 e2e로 확인 필요.
3. **[P0] 터치 핸들(선택/리사이즈/회전) 히트영역 확대.**
   (Paperless Post 2번: "텍스트를 손가락으로 집기 어렵다") Konva `hitStrokeWidth`·anchor
   크기를 **최소 44×44pt 터치 타깃**으로. 미선택 슬롯도 한 번 탭으로 쉽게 선택되게.
4. **[P1] "이 사진 교체 / 이 글자 수정"을 슬롯 직접 탭 → 인라인 액션 시트로.**
   드래그·미세조정을 강요하지 말고, 슬롯 탭 시 "교체 / 수정 / AI추천 / 삭제"만 노출.
5. **[P1] 미세조정 도구(회전 슬라이더·+/−·방향키)는 "고급" 접기.**
   기본 화면에서 숨기면 초보자 인지부하 급감. 고급 사용자만 펼침.
6. **[P1] 저장/미리보기/공유를 항상 보이는 하단 고정 바로.**
   "내가 뭘 만들고 있고 어떻게 보낼지"의 안심감. 데어무드·투아워게스트의 "즉시 반영" 체감.
7. **[P1] 실행취소(undo)·되돌리기 가시성·접근성 강화.**
   이미 useUndoable 있음 → 버튼을 더 크게/항상 노출, 토스트에 "되돌리기" 액션 추가.

### B. 구조적 개선 (수 주~분기급)
8. **[P0-구조] 모바일 청첩장 = 폼/섹션 기반, 종이 청첩장 = 캔버스, 경로 분리.**
   `format=mobile`은 데어무드형 섹션 편집기, `format=paper`만 Konva 캔버스. 모바일은
   드래그 위치 자유도가 사실상 불필요(스크롤형 세로 레이아웃).
9. **[P1-구조] 모바일 청첩장을 "섹션(블록)" 모델로.**
   대표사진/인사말/예식정보/오시는길/갤러리/계좌/RSVP/방명록을 **블록**으로. 블록 on/off,
   순서 변경(데어무드 핵심 기능), 블록별 폼. 캔버스 좌표 대신 구조화 데이터.
10. **[P1-구조] 온보딩 마법사 강화: 최소 필수입력 → "완성 미리보기" 빠른 도달.**
    현재 wizard 단계 활용. "이름·일시·장소·사진 1장"만으로 **완성된 청첩장을 즉시** 보여주고,
    나머지는 선택. "10분 완성" 체감 설계.
11. **[P2-구조] 자동 정렬·스냅 강화로 "못생기게 만들 수 없게".**
    이미 정렬 가이드 있음 → 슬롯을 **그리드/세이프영역에 스냅**, 폰트·색은 템플릿 팔레트
    내로 제한(Minted 철학). 자유도를 안전하게.
12. **[P2] 저화질/세이프영역 경고를 인라인으로.**
    이미 `lowResPrintWarning`·`imageQuality` 있음 → 슬롯 위 배지로 즉시 노출(인쇄 품질 사고 예방).

## 5. e2e 시뮬레이션 체크포인트 (청첩장 에디터 검증용)

> AGENTS.md 규칙: "작동한다 ≠ 검증됨". 아래는 **실제 사용자 시점**으로 밟아야 할 항목.
> 각 항목은 "정적 통과"가 아니라 **클릭/터치 경로를 직접 시뮬레이션**해 확인할 것.

**진입·온보딩**
- [ ] `/invitation/new?format=mobile` / `?format=paper` 각각 진입 → wizard→template→studio 흐름 정상.
- [ ] 최소 입력(이름·일시·장소·사진1)만으로 "완성 미리보기"에 도달하는 시간/탭 수.
- [ ] 8초 미입력 AI 추천 시트가 의도대로 뜨고, 편집을 방해하지 않는가.

**텍스트 편집(최우선 — 해외 1순위 불만 영역)**
- [ ] 텍스트 슬롯 편집 시 **키보드가 해당 텍스트를 가리지 않는가**(가리면 P0 버그).
- [ ] 입력 중 미리보기가 실시간 반영되는가.
- [ ] 빈 텍스트/아주 긴 텍스트/이모지/줄바꿈에서 레이아웃이 깨지지 않는가(엣지케이스).
- [ ] 텍스트 슬롯을 **한 번에 정확히 선택**할 수 있는가(작은 글자도).

**사진 편집**
- [ ] 슬롯 탭 → 교체 흐름이 드래그 없이 가능한가.
- [ ] 저해상도 업로드 시 인쇄 품질 경고(`lowResPrintWarning`)가 실제로 노출되는가.
- [ ] 누끼/리터치/일러스트 변환 엣지펑션 경로가 실패해도 우아하게 처리되는가(빈 catch 금지).

**터치·제스처(모바일 실기기)**
- [ ] 리사이즈/회전 핸들 터치 타깃이 손가락으로 잡히는가(44pt 이상).
- [ ] 핀치 줌/스크롤과 슬롯 드래그가 충돌하지 않는가.
- [ ] 미선택 슬롯 위 탭이 의도대로 "선택"으로 동작하는가(드래그로 오인 안 됨).

**실행취소·안정성**
- [ ] undo/redo가 텍스트 타이핑·드래그·크기변경 각각에서 올바른 단위로 동작(coalesceKey 검증).
- [ ] 연속 조작 후에도 **버벅임/지연 없이** 미리보기가 갱신되는가(Paperless Post 불만).

**저장·공유·출력**
- [ ] draft 저장 후 `/invitation/:id/edit` 재진입 시 모든 슬롯이 그대로 복원되는가
      (좌표·폰트·이미지·애니메이션 — `readFaceLayout` 라운드트립).
- [ ] 카카오톡 등 **공유 링크 발송 경로**가 실제로 동작하고, viewer가 정상 렌더되는가.
- [ ] 종이: 300dpi PDF 추출(`exportInvitationPdfPages`)이 세이프영역/블리드 포함 정확한가.

**회귀·검증 한계 명시**
- [ ] 위 항목 중 샌드박스에서 e2e 불가한 것(실기기 터치·카카오 전송 등)은 "미확인"으로
      보고하고 실환경 확인을 요청(AGENTS.md 검증 규칙).
