# AI 플래너 — 페르소나 UX 검토

main `02a5263` 기준. 5명의 가상 페르소나로 AI 플래너(`/ai-planner`) 진입~
대화 흐름을 시뮬레이션해서 발견한 UX 이슈를 정리한다.

자동 개선 가능한 명백한 결함은 같은 PR 에서 적용했고, **사용자 결정이
필요한 항목은 마지막 섹션 "사용자 결정 필요" 에 따로 모았다.**

대상 파일:
- `src/pages/AIPlanner.tsx` — 페이지 UI
- `src/hooks/useAIPlanner.ts` — 대화 상태·전송·에러 처리
- `src/components/wedding-planner/ChatBubble.tsx` — 답변 말풍선(마크다운 렌더)

---

## 페르소나

### 1. 박서연 — 29세, 일반 결혼(general), AI 플래너 첫 진입
- 진입하면 환영 카드 + 빠른 질문 4종(웨딩홀·스드메·타임라인·예산)이 보임
- "예산 어떻게 짜야 해?" 직접 입력 → 듀이가 답하며 `[예산 페이지](/budget)`
  링크 제안 → **링크를 탭함**

**발견한 이슈**:
- 🔴 **챗봇 답변 속 앱 내부 링크가 전체 페이지를 새로고침함**. 답변에는
  `[예산 페이지](/budget)`·`[주문 내역](/orders)`·`[드레스 갤러리](/ai-studio/...)`
  같은 내부 링크가 핸들러 전반(`userActivityHandlers`·`diagnosticHandlers`·
  `aiStudioHandlers` 등)에서 광범위하게 사용되는데, `ChatBubble` 의
  `ReactMarkdown` 에 링크 렌더러가 없어 기본 `<a>` 가 SPA 라우팅을 안 하고
  브라우저 전체 새로고침을 일으킴 → 대화 상태 소실 + 앱 재로딩(느림)
- → **자동 fix 1 적용**

### 2. 김민준 — 32세, 신랑, 본인이 직접 준비
- AI 플래너 진입 → 첫 화면 인사말 "**안녕하세요, 신부님!**"
- 본인은 신랑인데 "신부님" 으로 불려 살짝 어색함

**발견한 이슈**:
- ⚠️ **환영 인사말이 "신부님" 으로 고정**(`STYLE_GREETING`). 인수인계 문서상
  호칭은 "신부님 또는 신랑님 (또는 별명)" 이지만, `weddingSettings` 에 성별/
  역할 필드가 없어 클라이언트가 호칭을 결정할 근거가 없음
- → **결정 필요 A** (성별 데이터 부재 — 중립 문구로 바꿀지 / 필드 추가할지)

### 3. 이지은 — 27세, 셀프웨딩(self), 카페에서 모바일로 사용
- 스타일 오버라이드로 "셀프촬영 로케이션"·"DIY 부케·소품" 카드가 노출 — 잘 맞음
- "셀프웨딩 촬영 명소 추천해줘" 전송 → 응답 스트리밍 중 지하철 진입으로
  네트워크 끊김

**발견한 이슈**:
- 🔴 **전송 실패 시 사용자의 질문이 조용히 사라짐**. `useAIPlanner` 의
  catch 가 토스트만 띄우고 방금 보낸 user 메시지를 `filter` 로 제거 →
  토스트가 사라지면 "내가 뭘 물어봤는지"·"왜 답이 없는지" 흔적이 없음.
  스트리밍이 중간에 끊긴 경우엔 user 메시지만 지워져 앞뒤 없는 부분
  답변 버블만 남는 경우도 발생
- → **자동 fix 2 적용**

### 4. 정해린 — 34세, 무료 사용자, 예산에 민감
- 빠른 질문 카드 중 "예산 플래너" 에 **PRO 뱃지**가 붙어 있음
- "유료겠지" 생각하고 탭했는데 `BudgetSurvey` 모달이 그냥 열리고, 제출하면
  결과까지 무료로 정상 출력됨

**발견한 이슈**:
- ⚠️ **"예산 플래너" 카드의 PRO 뱃지와 실제 동작 불일치**. 카드에는
  `premium: true` 로 PRO 뱃지가 붙지만 `handleQuickClick` 에 프리미엄 게이트가
  없어 무료 사용자도 모달·결과를 전부 사용 가능. 뱃지는 사용자에게 "막혀
  있음" 신호를 주는데 실제론 안 막힘 → 신뢰 저하/혼란
- → **결정 필요 B** (게이트를 걸지 / 뱃지를 뗄지 — 수익화 정책 결정)

### 5. 최서윤 — 26세, AI 플래너 자주 쓰는 헤비 유저
- 한참 대화 후 화면 상단의 회전 아이콘(대화 초기화)을 무심코 탭
- 확인 절차 없이 **전체 대화가 즉시 삭제됨**. 대화는 어디에도 저장돼 있지
  않아(새로고침에도 사라짐) 복구 불가

**발견한 이슈**:
- ⚠️ **대화 초기화 버튼에 확인/되돌리기 없음** + **대화 비영속**. 실수 탭
  한 번에 누적 컨텍스트가 날아감. 무료 한도를 써가며 쌓은 답변이라 손실 체감 큼
- → **결정 필요 C** (확인 다이얼로그 / 대화 영속화)

### (보조) 신규 무료 사용자
- 첫 진입 시 일일 무료 5회 한도가 어디에도 안 보임(`dailyRemaining` 은 첫
  호출 응답 헤더로만 채워져 첫 질문 전에는 `null`)
- 6번째 질문에서 처음으로 한도 모달을 만나 당황
- → **결정 필요 D** (한도를 사전 고지할지)

## 사용자 결정 필요

### A. 환영 인사말 호칭 "신부님" 고정 (페르소나 2)

**현 상태**: `STYLE_GREETING` 의 title 이 모두 "신부님"(또는 "스몰웨딩 신부님"
등) 으로 고정. `weddingSettings` 에는 성별/역할 필드가 없음.

**선택지**:
- **A1** — 중립 문구로 변경 ("안녕하세요!" / "결혼 준비 도와드릴게요"). 데이터
  변경 없이 즉시 적용 가능, 신랑·신부 모두 어색하지 않음
- **A2** — 결혼 정보에 본인 역할(신랑/신부) 필드 추가 → 호칭 동적 적용. 데이터
  모델·온보딩 변경 필요
- **A3** — 그대로 ("신부님" 유지 — 주 타깃이 신부라는 전제)

→ **추천 A1** (저비용·무난). 장기적으로 A2 가 가장 정확하지만 필드 추가가 필요.

### B. "예산 플래너" PRO 뱃지 vs 실제 무료 동작 (페르소나 4)

**현 상태**: 카드는 `premium: true` 로 PRO 뱃지 노출, 그러나 게이트가 없어
무료로 완전 사용 가능. (Premium 가치는 "예산 분석 리포트 PDF" 인데, 빠른
설문이 만드는 건 채팅 답변이라 별개)

**선택지**:
- **B1** — 뱃지 제거 (실제 동작에 맞춤 — 채팅용 예산 플래너는 무료). 가장 단순,
  현 동작과 일치
- **B2** — 게이트 추가 (무료 사용자가 탭하면 `UpgradeModal`). 뱃지 의미는
  살지만 무료 사용자 진입 장벽
- **B3** — 뱃지를 "PRO" 대신 "리포트는 PRO" 류로 정교화 (채팅 답변은 무료,
  PDF 리포트만 PRO 임을 명확히)

→ **추천 B1** (지금 무료로 동작하므로 뱃지가 오해를 부름). 수익화를 의도했다면 B2.

### C. 대화 초기화 확인 + 대화 영속화 (페르소나 5)

**현 상태**: 초기화 버튼이 즉시 전체 대화 삭제, 확인/되돌리기 없음. 대화는
메모리에만 있어 새로고침·재진입 시에도 사라짐.

**선택지**:
- **C1** — 초기화에 확인 다이얼로그 추가 (실수 방지, 가장 가벼움)
- **C2** — C1 + 대화를 `localStorage` 에 영속화 (새로고침 복원). 클라이언트만
  변경
- **C3** — 대화를 서버(Supabase)에 저장해 기기 간 동기화 + 히스토리. 비용·
  스키마 변경 큼

→ **추천 C1 우선** (실수 데이터 손실이 가장 큰 통증). 여유 있으면 C2.

### D. 무료 일일 한도 사전 고지 (보조 페르소나)

**현 상태**: `dailyRemaining` 은 첫 호출 응답 헤더로만 채워져, 첫 질문 전에는
한도(일 5회)가 화면에 안 보임. 6번째에서야 한도 모달을 처음 만남.

**선택지**:
- **D1** — 그대로 (한도를 미리 강조하면 무료 체감 저하 우려)
- **D2** — 환영 화면에 작은 안내 한 줄 ("무료 하루 N회 · Premium 무제한").
  단, 숫자는 서버 정책이라 하드코딩 위험 → 첫 응답 후 받은 `dailyRemaining`
  기반으로만 노출하거나 문구를 숫자 없이 처리
- **D3** — 첫 질문 직후 받은 잔여 횟수를 환영 영역에도 반영

→ **추천 D1 또는 D3** (D2 의 숫자 하드코딩은 정책 변경 시 어긋남 위험).

## 변경 사항 검증 체크리스트

이 검토 PR 머지 후:

- [ ] AI 플래너에서 "주문 내역 보여줘" 등 질문 → 답변 속 `[주문 내역](/orders)`
      링크 탭 시 전체 새로고침 없이 SPA 라우팅으로 이동
- [ ] 답변 속 외부 링크는 새 탭에서 열림 (`rel="noopener noreferrer"`)
- [ ] 비행기모드 등으로 네트워크 끊고 질문 전송 → 내 질문 버블 유지 +
      "답변을 가져오지 못했어요" 인라인 에러 버블 노출
- [ ] 에러 버블이 남은 상태에서 재질문 → 정상 응답 (에러 버블이 컨텍스트
      오염 안 시킴)
- [ ] 빠른 질문(웨딩홀·스드메·타임라인·예산) 모달 정상 동작 (변경 없음)

위에서 OK 면 사용자 결정 필요 사항(A~D)에 대한 답을 받고 다음 작업 결정.
