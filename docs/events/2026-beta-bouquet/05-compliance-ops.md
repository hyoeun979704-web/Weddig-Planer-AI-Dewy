# 컴플라이언스 · 운영 가드 (집행 전 필독)

> 선행: `README.md` §1~§4. 이 문서는 크리에이티브가 아니라 **"집행 직전 발목 잡히는 것"**을 막는 가드.
> 소재를 올리기 전 이 체크부터 통과시킨다.

## 1. 🔴 플레이스토어 후기 보상 — 정책 리스크 (결정 필요)

- **리스크:** Google Play는 **보상을 대가로 한 평점·리뷰 유도를 금지**한다. "플레이스토어 후기 써주면 하트 +50"이 이에 해당하면 **앱 노출 하락·정책 경고·최악의 경우 제재**로 이어질 수 있다.
- **권장안:**
  - 보상(하트 +50)은 **SNS 후기에만** 연결.
  - 플레이스토어 후기는 **무보상 자발 권유**로 분리("써주시면 큰 힘이 돼요" 수준).
  - 리뷰 유도는 **만족 모먼트**(청첩장 발행 직후·AI 생성 성공 직후)에 자연 노출(경쟁분석 권고와 동일).
- **결정 필요:** 현재 §2 스펙은 원본 모집글대로 "SNS+플스토어 후기 → 하트 +50". 위 권장안으로 바꿀지 결정 후 `README.md` §2 반영.

## 2. 메타/인스타 프로모션 면책

- 인증 이벤트 게시물에 면책 1줄 포함: **"본 이벤트는 Instagram/Meta가 후원·관리하지 않습니다."**
- 태그·해시태그 인증은 **공개 계정 한정** 명시(비공개 계정은 인증 확인 불가).

## 3. 경품(부케 할인권) 고지 의무

집행 시 아래를 이벤트 안내에 **명시**(분쟁·신뢰 문제 예방):
- 당첨 인원 · **선정 방식**(선착순/추첨) · 발표일 · 발표 방법(DM/공지)
- 할인권 **유효기간**(예: 발급 후 60일) · **1인 1매** · 중복 할인 가능 여부
- 제휴업체명 · 적용 조건 → **제휴 계약 확정 후** 기재(확정 전 공개 금지)

## 4. 개인정보

- **베타 이메일:** 수집목적(베타 등록)·보유기간(**종료 후 파기**)·동의를 모집 글에 고지.
- **당첨자 정보:** 할인권 지급에 필요한 최소 정보만, 목적 외 사용 금지.

## 5. 부케 이벤트 운영 디테일

- **당첨 검증:** 게임 점수 **서버 기록(`game_points`) ↔ 캡처 교차 확인**. 오피셜 태그 + 공개 계정 필수.
- **발표/지급:** 정해진 날짜에 DM/공지, 미응답 시 재선정 기한(예: 7일).
- **할인권 발급:** 1인 1매 코드, 기존 `business_coupons` 인프라로 관리·유효기간 설정.
- **상호 노출:** 제휴업체도 자사 채널에 공동 홍보(win-win) — 제휴 협의 시 포함.

## 6. 소재 제작 실무

- **자막 필수:** 숏폼·광고는 무음 시청이 다수 → 모든 영상에 한글 자막. 자막 안전영역 상·하 10% 확보.
- **음원:** 상업적 사용 가능 라이선스만(유튜브 오디오 라이브러리·릴스 추천 음원). 무단 음원 금지.
- **export 규격:** 릴스/쇼츠 9:16(1080×1920) · 유튜브 16:9(1920×1080) · 카드뉴스/팝업 4:5(1080×1350).
- **이미지 텍스트:** 생성 이미지엔 텍스트 미포함 → 한글 오버레이(오타·다국어 깨짐 방지, README §3).
