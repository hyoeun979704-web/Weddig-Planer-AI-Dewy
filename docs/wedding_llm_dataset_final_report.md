# 웨딩 전용 LLM 학습 데이터셋 최종 정리본

작성일: 2026-07-02  
상태: 레포 업로드 대상 최종 MD  
목적: 결혼/웨딩 도메인 LLM 학습을 위한 원천 데이터, 신뢰도 라벨, 추출 필드, 보강 우선순위, SNS 텍스트 추출 결과를 체계화한다.

## 1. 핵심 결론

이 데이터셋은 단순 웨딩 자료 모음이 아니라 웨딩 전용 LLM이 다음 능력을 학습하도록 설계된 구조화 데이터셋이다.

| 학습 능력 | 설명 |
|---|---|
| 비용 추출 | 식대, 보증인원, 드레스 피팅비, 예복 패키지, 청첩장 수량별 가격 등 추출 |
| 업체정보 이해 | 업체명, 카테고리, 주소, 전화, 공식 URL, SNS URL, 주차, 교통, 가격패키지 구조화 |
| 출처 신뢰도 판단 | 공식 사이트, 광고성 후기, 소비자 후기, 정책자료, SNS 카드뉴스를 분리 |
| 계약 리스크 판단 | 보증인원, 위약금, 환불불가, 추가금, 계약정책, 청약철회 위험 분류 |
| 특수 결혼 유형 대응 | 임신 중 결혼, 국제결혼, 재혼, 셀프웨딩, 스몰웨딩, 노웨딩 데이터 라벨링 |
| 모바일청첩장 조언 | QR, 계좌, 지도, RSVP, 청모, 혼주/직장용 분리, 하객 UX 조언 |
| 마케팅 문구 필터링 | 인기, 추천, 가성비, 최고, 프리미엄 같은 검증 어려운 표현 분리 |

현재 본 데이터셋의 중심은 Supabase `places` / `place_details`의 업체 마스터이며, 로컬 JSONL/MD는 신뢰도 규칙, 수집 큐, 추출 샘플, 학습 스키마를 보강하는 역할이다.

### 반복 2. 신뢰도 라벨 개선

광고성 게시물은 모두 저신뢰로 버리는 방식이 아니라, 객관정보와 주관후기를 분리하는 기준으로 개선했다.

개선된 신뢰도 체계:

| 라벨 | 의미 | 학습 사용 |
|---|---|---|
| `A public_policy` | 정부/공공기관/법령/소비자원 | 정책 QA, 계약 리스크 |
| `B+ official_site` | 업체 공식 홈페이지 가격, 주소, 전화, 사업자번호 | 업체정보, 가격 추출 |
| `B vendor_provided_fact` | 광고성 글 안의 가격, 구성, 위치, 주차 | 조건부 구조화 가능 |
| `C consumer_experience` | 개인 후기의 경험담 | 소비자 동향, 리스크 힌트 |
| `D marketing_claim` | 추천, 인기, 가성비, 최고, 프리미엄 | 일반 근거 제외 |
| `E unverified_comment` | 댓글, 검증 안 된 여론 | 학습 제외 또는 집계만 |

개선 결과:

| 문제 | 개선 |
|---|---|
| 광고성 후기 전체를 버리면 가격/주소까지 손실 | `ad_objective_fact`와 `ad_subjective_review` 분리 |
| 후기 감상과 업체 객관정보가 섞임 | `objective_facts`, `subjective_claims`, `marketing_claims` 필드 분리 |
| 추천/가성비 문구가 모델 근거로 쓰일 위험 | `marketing_claim_filtering` 태스크 생성 |

## 9. 운영상 주의사항

1. 네이버블로그/SNS 원문은 저작권과 개인정보 위험이 있으므로 원문 전체를 학습시키지 않는다.
2. 광고성 게시물은 폐기하지 않고 객관정보만 분리한다.
3. 가격은 항상 조건부로 저장한다.
4. 현재 확인된 가격은 수집일 기준이며, 실시간 견적을 보장하지 않는다.
5. 카페/비공개 SNS는 사용자가 접근 가능한 범위에서만 읽고 우회하지 않는다.
6. Google AI Overview는 인용하지 않는다.
7. 레포에는 사용자가 승인한 최종 MD만 등록한다.
