# 13 · 운영 워크플로우 (Dewy 인스타 자동화 전체 흐름)

> zip 의 콘텐츠 지침과 직전에 만든 1단계 어드민 인프라(`/admin/instagram-posts`, `instagram_post_drafts` 테이블)를 **하나의 운영 흐름**으로 묶어주는 문서.

## ■ 전체 그림 (5단계)

```
┌─────────────────────────────────────────────────────────────────┐
│ [1단계 ✅] 스키마 + 어드민 껍데기                                │
│   · instagram_post_drafts 테이블 (admin RLS)                    │
│   · /admin/instagram-posts 페이지 (수동 초안 추가)              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ [2단계 ⏳] Satori 카드 렌더러                                    │
│   · content/instagram/10_안전영역_디자인토큰.md 의 사양 그대로  │
│   · Edge Function: instagram-card-renderer                      │
│   · card_texts → PNG N장 → Supabase Storage 업로드              │
│   · 어드민에서 미리보기 + 카드 교체                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ [3단계 ⏳] AI 카피 생성 (Gemini)                                 │
│   · 시스템 프롬프트 = content/instagram/01~08 + 09 + 11         │
│   · Edge Function: instagram-draft-generator                    │
│   · 주제 → caption · hashtags · card_texts                      │
│   · 11번 문서의 광고·저작권 룰 강제                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ [4단계 ⏳] 자동 발굴 (pg_cron)                                   │
│   · content/instagram/12_데이터소스_매핑.md 의 규칙 그대로      │
│   · 매일 09:00 → 우선순위 큐 → 1주제 자동 draft 생성            │
│   · promotional_events > partner_deals > tips_* > season        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ [5단계 ⏳] Instagram Graph API 발행                              │
│   · status='scheduled' && scheduled_for <= now() 큐 처리        │
│   · Edge Function: instagram-publisher                          │
│   · media container → publish → permalink 기록                  │
│   · Meta App Review 통과 후에만 라이브                          │
└─────────────────────────────────────────────────────────────────┘
```

## ■ 운영자의 일상 흐름 (정상 운영 시)

**아침 (5분)**
1. `/admin/instagram-posts?filter=draft` 진입
2. 어젯밤 cron 이 만든 초안 1~3개 확인
3. 카피 다듬기 + 카드 미리보기 점검
4. OK 이면 **승인** (status='approved')
5. 발행 시간 설정 → **예약** (status='scheduled')

**점심·저녁 (자동)**
- 예약 시간 도달 → 5단계 publisher 가 자동 발행
- 실패 시 어드민에 알림 (`status='failed'` + `last_error`)

**주 1회 (30분)**
- 지난 주 발행물 인사이트 검토 (도달·저장·댓글)
- `09_시즌_발행캘린더.txt` 의 다음 주 캠페인 큐 확인
- 필요 시 manual 초안 1~2개 직접 작성

## ■ 콜드 스타트 (지금 — 1단계 완료 시점)

5단계까지 미완성이지만, **수동 발행 흐름** 으로 지금부터 운영 시작 가능:

1. `/admin/instagram-posts` → "초안 추가" → 주제 입력
2. zip 의 01~08 지식 + 09 캘린더를 참고해서 카피 작성 (운영자 또는 Claude Code 호출)
3. Canva / Figma 에서 10번 디자인 토큰대로 카드 PNG 수동 제작
4. PNG 를 어드민 페이지에서 업로드 (← 2단계 만들기 전엔 수동 업로드 UI 필요)
5. 카피·해시태그를 클립보드 복사
6. 인스타 앱에서 수동 게시
7. 게시 후 permalink 를 어드민에 기록 (`published_permalink` 컬럼)

⚠️ **현재 어드민 페이지에는 PNG 업로드·미리보기 UI 가 없음.** 2단계 작업 전에 임시로 추가하거나, `card_image_urls` 컬럼을 Supabase 콘솔에서 직접 입력하는 방식으로 시작.

## ■ Claude Code 에 줄 작업 지시 템플릿

```
content/instagram 의 01~13 문서들 참고해서 다음 카드뉴스를 만들어줘:

  주제: "{주제 한 줄}"
  축:   {축1·축2·축3·축4 중 1개}
  소스: {tip_blog row id / partner_deal row id / 또는 "manual"}
  발행 시점: {언제}

결과:
  1. card_texts JSONB (3~7장)
  2. caption (인스타 캡션, 한국어 다정 톤)
  3. hashtags (10~15개)
  4. 광고·출처 표기 자동 삽입 (11번 룰)
  5. SQL: instagram_post_drafts 에 INSERT 하는 statement

스크린샷이 있으면 /content/instagram/shots/ 에 있어.
```

## ■ Claude Code 가 절대 하면 안 되는 것

- 01~13 의 톤·디자인 토큰 임의 변경 ❌
- 가격·평균을 자료 없이 만들어내기 ❌
- 의료·시술·다이어트 단정 ❌
- 광고 콘텐츠를 광고 표기 없이 ❌
- 외부 이미지 직접 사용 (tips_instagrams thumbnail 등) ❌
- 공포마케팅·단정·과장 ❌
