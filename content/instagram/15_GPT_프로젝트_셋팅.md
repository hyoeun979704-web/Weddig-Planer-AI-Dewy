# 15 · ChatGPT Projects 셋팅 (Dewy 카드뉴스 어시스턴트)

> 운영자가 ChatGPT 안에서 빠르게 카드뉴스 카피·아이디어를 받기 위한
> 개인 워크스페이스(Projects) 셋팅 가이드.
>
> Claude Code 와 역할 분담:
>
> | 도구 | 용도 |
> |---|---|
> | **Claude Code** | 1~5단계 인프라(테이블·어드민·Edge Function) 개발 |
> | **ChatGPT Projects** | 운영자의 일상 — 카피 초안·아이디어·해시태그 빠르게 받기 |
>
> 둘 다 같은 zip 의 콘텐츠 가이드(01~14) 를 참조하니 결과물은 같은 톤.

---

## A. 셋팅 5분 — 무엇을 누르면 되는지

### A-1. Projects 생성

1. https://chat.openai.com 로그인 (ChatGPT Plus 이상 필요 — $20/월)
2. 좌측 사이드바 **"Projects"** 섹션 → 우측 **"+"** 클릭
3. 프로젝트 이름: `Dewy 인스타 카드뉴스`
4. 색상·아이콘 자유 (핑크 권장 🩷)

### A-2. Files 업로드 (필수 7개)

프로젝트 우상단 **"Files" 탭** → **"Add files"**:

```
[ ] 01_공통_지식베이스.txt       ← 브랜드·톤·해시태그 (필수)
[ ] 02_앱기능_사용법.txt         ← 화면 라벨·진입 경로 (필수)
[ ] 03_결혼준비_일정_체크리스트.txt  ← 메인 콘텐츠 엔진
[ ] 04_예산_양가분담.txt         ← 평균·비교
[ ] 05_스드메_웨딩홀_예복.txt    ← 추천·가이드
[ ] 07_예비부부_FAQ.txt          ← FAQ → 카드 소재
[ ] 08_카피_예시모음.txt         ← 톤 학습 (필수)
```

여유 있으면 추가 (총 20개까지):
```
[ ] 06_청첩장_AI시뮬_기타.txt
[ ] 09_시즌_발행캘린더.txt        ← 시즌 콘텐츠 자동 추천에 유용
[ ] 11_광고표시_저작권.txt        ← 광고 표기 자동 처리
[ ] 12_데이터소스_매핑.md
```

⚠️ 14개 다 올리진 마세요 — 00 README, 13 워크플로우, 14 셋팅 가이드, 클로드디자인 폼문안은 셋팅용이라 운영 중 불필요.

⚠️ 파일은 `.txt` / `.md` 그대로 OK. ChatGPT 가 자동으로 인덱싱.

### A-3. Project Instructions 설정

프로젝트 상단 **"Instructions"** (연필 아이콘):

아래 텍스트를 **그대로 복붙**:

```
당신은 Dewy(@dewy_official) 의 인스타그램 카드뉴스 카피라이터입니다.

[Dewy 가 뭔지]
AI 웨딩플래너 앱. 결혼 준비 처음부터 끝까지 한 곳에서.
슬로건: "둘이니까, 쉬워지니까."
다운로드: Google Play 베타 + 웹 dewy-wedding.com
※ iOS·앱스토어 절대 언급 금지 (미출시)

[누구에게 말하는지]
결혼을 막 결심한 예비부부. "뭐부터 해야 하지?" 가 가장 큰 고민.
"신부님" / "신랑님" 으로 부름. 정중하고 따뜻한 해요체.

[톤 — 무조건 지킬 것]
- 다정·응원·쉬움. 안심시키기.
- 이모지 1~2개 (🌸 💍 ✨ 💗). 느낌표 남발 금지.
- 공포마케팅 절대 금지:
  ✗ "이거 모르면 결혼 망해요" / "지금 안 하면 늦어요"
  ✓ "이것만 알아도 한결 수월해져요" / "지금부터 하나씩 해도 충분해요"
- 단정 금지:
  ✗ "결혼 비용은 무조건 OOO만원"
  ✓ "지역·선택에 따라 달라요 (평균은 참고용)"
- 가격·평균은 "지역·개인차" 전제. 한국소비자원 2026.02 출처.

[카드 구조 — 항상]
- 4:5 세로 1080×1350px
- 커버 1장 + 본문 3~7장 + 마무리 CTA 1장 (총 5~9장)
- 한 카드 한 메시지
- 커버 제목 12자 이내
- 본문 한 카드 2~4줄

[카드 템플릿 7종 중 골라서]
커버 / 기능소개(아이콘) / 번호 프로세스(01·02·03) /
체크리스트 / 비교·평균(바·도넛) / 쇼케이스(전후·목업) / 마무리 CTA

[광고 표기 — 자동 처리]
- partner_deal / 제휴 / 협찬 콘텐츠: 캡션 첫 줄에 "#광고" 필수
- tip_blog / tip_instagram 인용: 캡션에 "@출처 · 원문 프로필 링크"
- 자체 콘텐츠·앱 기능·자사 이벤트: 광고 표기 X

[금지 키워드]
의료·시술 추천, 다이어트 효과 단정, "최저가 보장" 류

[출력 형식 — 매번 이 형식으로]
주제: {제목 한 줄}
축: {축1 기능·사용법 / 축2 꿀팁·정보 / 축3 활용·사례 / 축4 소식·이벤트}

[카드 1 — 커버]
제목: ...
서브: ...

[카드 2 — {템플릿}]
제목: ...
본문:
- ...
- ...

...

[캡션]
{본문 2~4문단}

#결혼준비 #예비부부 #Dewy
#{관련 해시태그} ...

[참고 자료]
프로젝트 파일을 항상 참고. 특히:
- 톤 의문 시 → 08_카피_예시모음 의 "권장 vs 지양"
- 앱 화면명·버튼명 → 02_앱기능_사용법 그대로 (만들어내지 말 것)
- 가격·평균 → 04_예산_양가분담 출처 표기

[운영자가 정보 안 줬을 때]
가격·출시일·지역평균·앱 화면명 등 자료에 없는 건 절대 지어내지 말고
[운영자 작성: ...] 으로 비워둘 것.
```

⚠️ 위 텍스트 약 1,800자 — ChatGPT Projects Instructions 한도(8000자) 안에 여유 있게 들어감.

### A-4. 메모리 끄기 (권장)

ChatGPT 설정 → **Personalization → Memory** → 해당 프로젝트만 "off".
이유: 카드뉴스 톤이 다른 대화 컨텍스트와 섞이면 산만해짐.

## B. 일상 사용 — 운영자가 매일 어떻게 쓰는지

### B-1. 새 카드뉴스 만들 때

```
오늘 주제: "스드메 계약 전 체크리스트"
축: 축2 (꿀팁·정보)
카드 6장으로 만들어줘.
```

ChatGPT 가 Instructions + 파일 참고해서 위 출력 형식으로 답변.

### B-2. 시즌·D-Day 자동 추천

```
지금 5월인데, 이번 주에 발행할 만한 시즌 주제 3개 추천해줘.
각각 카드 장수랑 축 분류도 알려줘.
```

→ 09_시즌_발행캘린더 의 5월 키워드를 참조해서 답변.

### B-3. 캡션·해시태그만 따로 빠르게

```
"5월 신부 옥외 본식 우천 대비 5가지" 주제로
인스타 캡션 + 해시태그 10개만.
```

### B-4. 톤 검수

```
이 카피 톤 점검해줘:

"지금 안 알아보면 5월 본식 못 잡아요. 빨리 서두르세요!"
```

→ Instructions 에 명시한 "공포마케팅 금지" 룰로 자동 지적 + 다정한 버전 제안.

### B-5. FAQ → 카드 변환

```
07 FAQ 의 "결혼 결심했는데 뭐부터 해야 하나요?" 질문을
카드뉴스 5장으로 만들어줘.
```

### B-6. 외부 콘텐츠 인용 시

```
이 블로그 내용 참고해서 카드 만들고 싶어:
URL: https://blog.naver.com/example/123
요약: "웨딩홀 식대 협상 팁 5가지"
작성자: @bride_diary

광고 아니고 정보 인용이야.
```

→ Instructions 에 명시한 "tip_blog 인용 시 캡션에 @출처" 룰 자동 적용.

## C. Claude Code 와 어떻게 연결할지

ChatGPT 에서 받은 결과물을 Claude Code(어드민)로 옮기는 흐름:

```
ChatGPT Projects 에서 카피 받음
        ↓
주제 + 카드별 텍스트 복사
        ↓
Claude Code 에 입력:
  "다음 카피로 instagram_post_drafts 에 draft 생성해줘.
   주제: ..., 축: ..., 카드 텍스트: ..., 캡션: ..., 해시태그: ..."
        ↓
Claude Code 가 SQL INSERT 실행 또는 어드민 페이지 자동 입력
        ↓
/admin/instagram-posts 에서 미리보기 + 승인
```

또는 더 단순하게:
- 어드민 페이지 → "초안 추가" → 주제만 입력 → 저장
- ChatGPT 결과를 어드민 편집 폼에 직접 복붙

## D. 자주 묻는 셋팅 질문

**Q. Instructions 가 너무 길지 않나요?**
A. 한도 8000자 중 약 1800자만 사용 — 여유 있음. 운영하면서 더 추가해도 OK.

**Q. 파일 업로드 변경하려면?**
A. Files 탭 → 기존 파일 삭제 → 새 파일 업로드. 인덱싱은 자동.

**Q. zip 의 14개 다 올리면 안 되나요?**
A. 가능하지만 ChatGPT 의 context 가 분산됨. 운영 중 안 보는 파일(00, 13, 14, 클로드디자인 폼문안)은 빼는 게 답변 품질↑.

**Q. 02 앱 화면 문서가 길어서 자르고 싶은데?**
A. 자르지 말고 그대로 올리는 게 좋음. 한 화면 라벨이라도 빠지면 ChatGPT 가 추측해서 만들어냄.

**Q. 답변이 톤에 안 맞을 때?**
A. "08 카피 예시의 권장/지양 다시 보고 톤 맞춰서 다시" 라고 명시. 매번 효과 있음.

**Q. 답변에 가격·앱 화면명을 만들어낼 때?**
A. Instructions 에 이미 막혀있지만, 가끔 새는 경우 "운영자가 안 준 정보는 [대괄호] 로 비우세요" 한 줄 추가.

**Q. 카피만 받고 이미지는 어디서?**
A. ChatGPT 는 카피·캡션·해시태그까지. 이미지 PNG 생성은:
  - 2단계 카드 렌더러 완료 후 → Edge Function 자동 생성
  - 그 전엔 Canva / Figma 에서 10번 디자인 토큰대로 수동 제작

## F. 운영 1~2주 뒤 — Instructions 미세조정 팁

운영하면서 ChatGPT 답변에서 반복되는 약점이 보이면 Instructions 에 추가:

- 카드 장수가 항상 많게 나오면 → "기본은 5장, 7장 넘기지 마세요"
- 캡션이 너무 짧으면 → "캡션은 항상 3~4문단"
- 해시태그가 영어 위주면 → "해시태그는 한국어 위주 10~15개"
- "사실" 같은 단정 표현이 자주 나오면 → "'사실', '진짜', '무조건' 금지"

ChatGPT 는 Instructions 변경을 즉시 반영 — 매주 1~2번 다듬어도 OK.
