## 🔴 P0 — 즉시 (오늘 안에)

### A. Supabase 마이그레이션 적용

**문제**: `instagram_post_drafts` 테이블이 운영 DB 에 아직 적용 안 됨. 1단계 어드민 페이지 진입 시 PostgREST 422.

**어디서**: Supabase Dashboard → SQL Editor 또는 CLI

**무엇을**:
```bash
# 옵션 1: CLI (로컬 환경에 supabase CLI 있을 때)
supabase db push

# 옵션 2: Dashboard SQL Editor 에서 직접 실행
# supabase/migrations/ 폴더의 미적용 .sql 파일 내용을 순서대로 붙여넣기
# 특히:
#   20260528120000_payments_plan_type.sql
#   20260528130000_instagram_post_drafts.sql
```

**검증**:
```sql
SELECT column_name FROM information_schema.columns
WHERE table_name = 'instagram_post_drafts'
ORDER BY ordinal_position;
-- 결과 행 21개가 나와야 정상 (id, topic, source_type, ..., updated_at)
```

추가로:
```sql
SELECT column_name FROM information_schema.columns
WHERE table_name = 'payments' AND column_name = 'plan_type';
-- plan_type 한 행 나오면 정상
```

마이그레이션 적용 후 어드민 페이지 진입해서 "초안 추가" 한 번 눌러서 row 생성되는지 확인.

**어디서**: https://search.google.com/search-console

**무엇을**:

## 🟠 P1 — 이번 주

### D. Supabase Storage 버킷 생성 (2단계 카드 렌더러 동작 조건)

**어디서**: Supabase Dashboard → Storage → **New bucket**

**무엇을**:
```
Name:              instagram-cards
Public bucket:     ✓ 체크 (필수 — IG Graph API 가 fetch 함)
File size limit:   10 MB
Allowed MIME:      image/png, image/jpeg
```

생성 후 → Policies → 다음 SQL 추가:

```sql
CREATE POLICY "Admins manage instagram cards"
ON storage.objects FOR ALL
TO authenticated
USING (
  bucket_id = 'instagram-cards'
  AND public.has_role(auth.uid(), 'admin'::app_role)
)
WITH CHECK (
  bucket_id = 'instagram-cards'
  AND public.has_role(auth.uid(), 'admin'::app_role)
);
```

**검증**: 대시보드에서 테스트 PNG 1장 업로드 → 공개 URL 브라우저로 열림 확인.

### E. Slack Webhook 셋업 (발행 실패 알림)

**어디서**: 본인 Slack 워크스페이스 (없으면 5분 생성)

**무엇을**:
1. 워크스페이스 → "Incoming WebHooks" 앱 추가
2. 채널 선택 (예: `#dewy-alerts`)
3. Webhook URL 복사
4. Supabase Dashboard → Edge Functions → Secrets → 추가:
   ```
   SLACK_WEBHOOK_URL = https://hooks.slack.com/services/...
   ```

**검증**:
```bash
curl -X POST -H 'Content-type: application/json' \
  --data '{"text":"Dewy 알림 테스트 ✅"}' \
  $SLACK_WEBHOOK_URL
```
Slack 채널에 메시지 보이면 OK.

### F. Edge Functions 배포 (3개 신규)

**어디서**: 터미널 (로컬 supabase CLI 필요)

**무엇을**:
```bash
supabase functions deploy instagram-draft-generator
supabase functions deploy instagram-card-renderer
supabase functions deploy instagram-publisher
```

**검증 (3단계 — 즉시 동작 가능)**:
```bash
# 어드민 토큰 또는 service_role 키로
curl -X POST "https://<PROJECT_REF>.supabase.co/functions/v1/instagram-draft-generator" \
  -H "Authorization: Bearer <ADMIN_JWT>" \
  -H "Content-Type: application/json" \
  -d '{"topic": "5월 옥외 웨딩홀 추천 TOP 5"}'
# → caption, hashtags, card_texts JSON 반환되면 OK
```

**검증 (2단계 — Storage 버킷 필요)**:
```bash
# 위에서 만든 draft 의 id 사용
curl -X POST "https://<PROJECT_REF>.supabase.co/functions/v1/instagram-card-renderer" \
  -H "Authorization: Bearer <ADMIN_JWT>" \
  -d '{"draftId": "<UUID>"}'
# → instagram-cards/drafts/<UUID>/card-N.png 업로드 + URL 반환
```

**검증 (5단계 — Meta 토큰 필요, 다음 단계)**: 토큰 받기 전엔 503 "instagram_not_configured" 반환이 정상.

## 🔍 Claude 가 발견한 잔여 이슈

1. **타입 cast `(supabase as any)`** — AdminInstagramPostEdit.tsx 에서 임시 사용 중. 마이그레이션 적용 후 `npm run gen-types` (또는 동등 명령) 실행하면 정리 가능.
2. **`source_id` UUID vs text** — 마이그레이션은 UUID, 어드민 UI 는 text input. 비-UUID 값 입력 시 DB 거부. 운영자가 manual source 입력 시 UUID 형식 또는 빈값으로.
3. **App Review 시연 영상** — Storage·draft·렌더러까지 동작하는 상태에서 녹화해야 통과율↑.
4. **Pretendard 폰트 jsdelivr CDN** — 카드 렌더 시점에 CDN 응답 의존. 운영 시 caching 안정성 모니터.
