## A. Meta Developer App + Instagram Graph API 토큰

> 5단계(자동 발행)의 핵심 인프라. 현재 Dewy 인스타 계정 보유 가정.

### A-0. 사전 조건 (먼저 점검)

```
[ ] @dewy 인스타 계정이 Business 계정인가?
    인스타 앱 → 설정 → 계정 → "프로페셔널 계정으로 전환" → Business 선택
    (Creator 가 아니라 Business 여야 함)

[ ] Facebook 페이지가 있는가?
    없으면 facebook.com/pages/create → "비즈니스/브랜드" → "Dewy"
    페이지 카테고리: 앱 페이지 / 결혼 서비스 등

[ ] 인스타 ↔ 페이스북 페이지 연결됨?
    페이스북 페이지 → 설정 → "Instagram" → 계정 연결
    또는 인스타 앱 → 설정 → 계정 → "페이지 연결"
```

⚠️ Business 계정 + 페이지 연결이 없으면 Graph API 자체가 호출 안 됨. 이게 가장 자주 막히는 부분.

### A-1. Meta Developer App 생성

1. https://developers.facebook.com/apps/ 로그인
2. **"앱 만들기"** → 사용 사례: **"기타"** → 다음
3. 앱 유형: **"비즈니스"** 선택 → 다음
4. 앱 이름: `Dewy Wedding` (또는 원하는 이름)
   - 앱 이름에 "Instagram" / "FB" / "Facebook" 단어 금지 (Meta 정책)
5. 앱 연락처 이메일 입력 → 비즈니스 포트폴리오 선택 (없으면 새로 생성)
6. 생성 완료 → 대시보드 진입

### A-2. Instagram Graph API 제품 추가

대시보드 좌측 메뉴:
1. **"제품 추가"** 또는 "+" 아이콘
2. **Instagram** 카드 → **"설정"**
3. **"Instagram Graph API"** 선택 (Basic Display 아님 ⚠️)
4. **Facebook 로그인** 도 같이 추가 (토큰 발급에 필요)

### A-3. 권한 신청 (App Review 필요)

좌측 메뉴 → **"앱 검수"** → **"권한 및 기능"**:

```
[ ] instagram_basic           (기본 — 자동 승인)
[ ] instagram_content_publish (★ 카드뉴스 발행에 필수 — 검수 필요)
[ ] pages_show_list           (페이지 목록 — 검수 필요)
[ ] pages_read_engagement     (인사이트 — 선택, 나중에 받아도 OK)
```

각 권한 우측 **"요청"** 클릭 → 사용 사례 설명 입력 → 검수 제출.

**검수 제출 시 필요한 것**:
- 스크린 녹화 영상 (1~3분) — 어드민 페이지에서 카드 생성 → 발행 흐름 시연
- 사용 사례 설명: "Dewy 는 한국 예비부부를 위한 AI 웨딩플래너 앱이며, 공식 인스타 계정에 결혼 준비 가이드 카드뉴스를 자동으로 발행하기 위해 이 권한이 필요합니다"
- 개인정보처리방침 URL: `https://dewy-wedding.com/privacy`
- 앱 아이콘 1024×1024

⚠️ 영상 없이 신청하면 거의 100% 반려. 2단계(카드 렌더러) 완료 후 영상 찍어서 함께 제출 권장.

**검수 통과까지**: 5~14일, 가끔 한 달.

### A-4. 토큰 발급 — 검수 대기 중에도 가능

좌측 메뉴 → **"도구"** → **"Graph API Explorer"**:

#### Step 1. 단기 사용자 토큰

1. 우측 상단 앱 선택 → 본인 앱
2. **"Get User Access Token"**
3. 권한 체크:
   ```
   ✓ instagram_basic
   ✓ instagram_content_publish
   ✓ pages_show_list
   ✓ pages_read_engagement
   ✓ business_management
   ```
4. **"Generate Access Token"** → 페이스북 로그인 동의
5. 발급된 토큰 복사 (약 1시간 유효)

#### Step 2. 장기 사용자 토큰 (60일)

터미널에서:
```bash
curl -i -X GET "https://graph.facebook.com/v19.0/oauth/access_token?\
grant_type=fb_exchange_token&\
client_id={APP_ID}&\
client_secret={APP_SECRET}&\
fb_exchange_token={STEP1_TOKEN}"
```

- `APP_ID`, `APP_SECRET`: 대시보드 → "설정 → 기본설정" 에서 확인
- 응답의 `access_token` 이 60일 유효 토큰

#### Step 3. 페이지 액세스 토큰 (무기한)

```bash
curl -i -X GET "https://graph.facebook.com/v19.0/me/accounts?\
access_token={STEP2_TOKEN}"
```

응답에서 본인 페이지 객체의 `access_token` 추출.
**이게 무기한 유효한 페이지 토큰** ← 우리가 쓸 토큰.

#### Step 4. IG_BUSINESS_ACCOUNT_ID 조회

```bash
curl -i -X GET "https://graph.facebook.com/v19.0/{PAGE_ID}?\
fields=instagram_business_account&\
access_token={STEP3_PAGE_TOKEN}"
```

응답 `instagram_business_account.id` 가 IG Business Account ID (숫자 17자리).

### A-5. Supabase Secret 등록

Supabase 대시보드 → 프로젝트 → **"Edge Functions" → "Secrets"**:

```
IG_PAGE_ACCESS_TOKEN     = {Step3 페이지 토큰}
IG_BUSINESS_ACCOUNT_ID   = {Step4 IG Business ID, 숫자}
IG_APP_ID                = {앱 ID, 토큰 재발급용}
IG_APP_SECRET            = {앱 시크릿, 토큰 재발급용}
```

⚠️ **절대 코드에 하드코딩 금지**. .env.local 도 금지 (git ignore 되어도 실수로 커밋 위험).
⚠️ 페이지 토큰 분실 시 Step 2~3 재실행 → 새 토큰으로 secret 업데이트.

### A-6. 발급 확인 (curl 한 줄)

```bash
curl "https://graph.facebook.com/v19.0/{IG_BUSINESS_ACCOUNT_ID}?\
fields=username,followers_count&\
access_token={IG_PAGE_ACCESS_TOKEN}"
```

응답에 `"username": "dewy_official"` 같은 게 나오면 OK.
401/403 이면 토큰·권한 점검.

## B. Supabase Storage 버킷 (instagram-cards)

> 2단계 Satori 카드 렌더러가 PNG 를 저장할 곳.

### B-1. 버킷 생성

Supabase 대시보드 → **"Storage" → "New bucket"**:

```
Name:             instagram-cards
Public bucket:    ✓ (체크 — 인스타 Graph API 가 public URL 로 fetch 함)
File size limit:  10 MB
Allowed MIME:     image/png, image/jpeg
```

⚠️ Public 필수. private 이면 IG Graph API 가 이미지 못 읽어서 발행 실패.

### B-2. 폴더 구조 (관례)

```
instagram-cards/
  drafts/{draft_id}/        ← 미발행 초안 카드 (편집 중)
    card-1.png
    card-2.png
    ...
  published/{yyyy-mm}/      ← 발행 완료 카드 (월별 아카이브)
    {draft_id}/
      card-1.png
      ...
```

Edge Function 이 자동으로 이 구조로 업로드 — 운영자는 폴더 직접 안 만들어도 됨 (첫 업로드 시 자동 생성).

### B-3. RLS 정책 (Storage 정책)

Storage → instagram-cards 버킷 → **"Policies"** 탭 → **"New policy"**:

**정책 1: Admin 만 업로드/수정/삭제**
```sql
-- INSERT, UPDATE, DELETE 모두 동일 패턴
CREATE POLICY "Admins manage instagram cards"
ON storage.objects FOR ALL
TO authenticated
USING (
  bucket_id = 'instagram-cards'
  AND public.has_role(auth.uid(), 'admin'::app_role)
)
WITH CHECK (
  bucket_id = 'instagram-cards'
  AND public.has_role(auth.uid(), 'admin'::app_role)
);
```

**정책 2: 누구나 읽기 (public)**
- Public 버킷이라 별도 정책 불필요. 자동으로 anon 읽기 가능.

### B-4. 검증

대시보드에서 직접 테스트 파일 1장 업로드 →
공개 URL 클릭해서 브라우저로 열림 확인.

URL 형식:
```
https://{PROJECT_REF}.supabase.co/storage/v1/object/public/instagram-cards/test.png
```

이 URL 형식을 Edge Function 이 그대로 IG Graph API 의 `image_url` 파라미터로 넘김.

## C. 발행 실패 알림 채널

> 5단계(자동 발행) 가 실패했을 때 운영자에게 알리는 채널.
> 권장: **Slack Incoming Webhook** (5분 셋업, 무료, 가장 간단)

### 옵션 비교

| 채널 | 셋업 시간 | 모바일 알림 | 비용 |
|---|---|---|---|
| **Slack Webhook** ⭐ 권장 | 5분 | ✓ | 무료 |
| Discord Webhook | 5분 | ✓ | 무료 |
| 이메일 (Resend) | 15분 | ✗ (지연 가능) | 무료 100/일 |
| 카카오톡 (나에게 보내기) | 30분+ 비공식 | ✓ | 무료지만 정책 변경 위험 |
| Telegram Bot | 10분 | ✓ | 무료 |

### C-1. Slack Webhook 셋업 (권장)

1. 본인 워크스페이스가 있다면 사용, 없으면 https://slack.com/get-started 에서 무료 생성
   - 워크스페이스 이름: "Dewy 운영"
2. 워크스페이스 → **앱 추가** → "Incoming WebHooks" 검색 → 추가
3. 알림 받을 채널 선택 (예: `#dewy-alerts`)
4. **Webhook URL** 복사 (형식: `https://hooks.slack.com/services/T../B../...`)
5. Supabase Secret 등록:
   ```
   SLACK_WEBHOOK_URL = https://hooks.slack.com/services/...
   ```

테스트:
```bash
curl -X POST -H 'Content-type: application/json' \
  --data '{"text":"Dewy 알림 테스트 ✅"}' \
  $SLACK_WEBHOOK_URL
```

### C-2. Discord 대안

1. Discord 서버 → 채널 우클릭 → **"채널 편집" → "통합" → "웹훅"** → "새 웹훅"
2. 웹훅 URL 복사
3. Supabase Secret:
   ```
   DISCORD_WEBHOOK_URL = https://discord.com/api/webhooks/...
   ```

테스트:
```bash
curl -X POST -H 'Content-Type: application/json' \
  --data '{"content":"Dewy 알림 테스트 ✅"}' \
  $DISCORD_WEBHOOK_URL
```

### C-3. 알림 발송 패턴 (Edge Function 코드)

```ts
// supabase/functions/_shared/notify.ts (5단계 작업 시 추가)
export async function notifyAdmin(message: string, level: "info" | "warn" | "error" = "warn") {
  const url = Deno.env.get("SLACK_WEBHOOK_URL");
  if (!url) return; // 알림 채널 미설정 — 조용히 패스
  const emoji = { info: "ℹ️", warn: "⚠️", error: "🚨" }[level];
  await fetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text: `${emoji} Dewy 인스타: ${message}` }),
  }).catch((e) => console.error("notify failed:", e));
}
```

호출 예 (instagram-publisher 안에서):
```ts
catch (error) {
  await adminClient
    .from("instagram_post_drafts")
    .update({ status: "failed", last_error: String(error) })
    .eq("id", draftId);

  await notifyAdmin(
    `발행 실패 — draft ${draftId}\n주제: "${topic}"\n에러: ${error.message}`,
    "error",
  );
}
```

## ■ 자주 막히는 부분 FAQ

**Q. Instagram 비즈니스 계정인데 Graph API 가 401 을 반환해요**
A. 거의 99% "페이스북 페이지 ↔ 인스타 계정 연결" 안 됨. A-0 점검.

**Q. App Review 가 계속 반려돼요**
A. (1) 영상이 없거나 (2) 사용 사례 설명이 추상적 (3) 개인정보처리방침 URL 접근 안 됨.
   영상은 "운영자가 어떻게 발행하는지" 어드민 화면 위주로 1~3분, 한국어 자막 권장.

**Q. 페이지 토큰이 갑자기 401 이 됩니다**
A. 페이지 관리자에서 본인이 제거됐거나, 페이스북 비밀번호 변경 직후. A-Step2~3 재실행.

**Q. Storage 업로드는 되는데 IG 발행 시 "image_url could not be downloaded"**
A. 버킷 public 체크 안 됨. B-1 재확인.

**Q. Slack 무료 워크스페이스 90일 메시지 보관 한계 — 알림 사라져요**
A. 알림 채널은 보관용 아니라 OK. 실패 기록은 instagram_post_drafts.last_error 에 영구 저장.
