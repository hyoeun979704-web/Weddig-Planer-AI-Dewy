# 12 · 데이터 소스 → 카드 템플릿 매핑 (Dewy)

> Dewy DB 의 실데이터를 카드뉴스 소스로 자동 변환하는 규칙.
> `/admin/instagram-posts` 1단계 인프라의 `source_type` · `source_id` 컬럼과 직접 연결.

## ■ 매핑 일람표

| 소스 테이블 | `source_type` | 추천 카드 템플릿 | 광고 표기 | 시즌성 |
|---|---|---|---|---|
| `tips_blogs` | `tip_blog` | 체크리스트·번호 프로세스 | 출처 표기 | 상시 |
| `tips_instagrams` | `tip_instagram` | 쇼케이스(요약형) | 출처 + 멘션 | 상시 |
| `tips_videos` | `tip_video` | 번호 프로세스 | 채널 표기 | 상시 |
| `partner_deals` | `partner_deal` | 비교·평균 / 마무리CTA | **#광고 필수** | 시즌 가능 |
| `promotional_events` | `promotional_event` | 마무리CTA | Dewy 자체 → 불요 | 캠페인 |
| `places` | `place` | 쇼케이스(목업) | 광고 X (정보) | 시즌 강함 |
| (없음) | `season` | 커버 + 번호 프로세스 | 불요 | 시즌 의존 |
| (없음) | `manual` | 자유 | 운영자 판단 | 자유 |

## ■ 데이터 소스별 변환 규칙

### 1) tips_blogs (네이버 블로그 큐레이션)

```
입력 row: { url, title, description, blogger_name, blogger_link, post_date, categories }

→ 카드 변환:
  topic         = title (요약)
  source_type   = "tip_blog"
  source_id     = row.id
  card_count    = 4~6
  card_texts    = [
    { title: "{title 의 핵심 키워드}", body: "" },          # 표지
    { title: "이런 이야기예요", body: description 요약 },   # 인트로
    ... 핵심 포인트 N장 (Gemini 가 description 에서 추출) ...
    { title: "Dewy 가 이런 정보를 모아드려요", body: CTA } # 마무리
  ]
  caption       = "정보 출처: @{blogger_name} · 원문은 프로필 링크 참고\n\n{커버 카피 확장}"
  hashtags      = ["결혼준비", "예비부부", "Dewy", ...row.categories]
```

### 2) tips_instagrams (인스타 큐레이션)

```
입력 row: { url, title, description, author, thumbnail_url, categories }

→ 카드 변환:
  topic         = title
  source_type   = "tip_instagram"
  card_count    = 3~4 (요약 형식 — 원본 인스타와 차별화)
  caption       = "@{author} 님의 게시물 참고\n원문: {url}\n\n{본문}"

⚠️ 원본 thumbnail_url 직접 사용 금지 (저작권). 자체 카드만 발행.
```

### 3) tips_videos (유튜브 큐레이션)

```
입력 row: { video_id, title, channel_name, duration_seconds, categories }

→ 카드 변환:
  topic         = title
  source_type   = "tip_video"
  card_count    = 5~7 (영상 = 정보 많음 → 카드 늘림)
  card_texts    = [ 표지, 인트로, 단계 1~N, CTA ]
  caption       = "참고 영상: {channel_name} · YouTube\n자세한 건 영상에서 확인해요\n\n{본문}"

⚠️ 영상 썸네일 직접 사용 금지.
```

### 4) partner_deals (제휴 할인·쿠폰)

```
입력 row: { title, description, partner_name, discount_amount, valid_until, categories }

→ 카드 변환:
  topic         = "{partner_name} {할인 내용}"
  source_type   = "partner_deal"
  card_count    = 3
  card_texts    = [
    { title: "{partner_name} 특별 혜택", body: "" },
    { title: description, body: "마감: {valid_until} 까지" },
    { title: "Dewy 앱에서 받기", body: CTA }
  ]
  caption       = "#광고 · #{partner_name} 제휴\n\n{본문}\n\n혜택 받기: Dewy 앱 → 혜택 탭"
  hashtags      = ["광고", "결혼혜택", "Dewy", ...]
  scheduled_for = valid_until - 7일 (마감 1주일 전)
```

### 5) promotional_events (자사 이벤트·캠페인)

```
입력 row: { title, subtitle, cta_label, cta_path, starts_at, ends_at, target_personas }

→ 카드 변환:
  topic         = title
  source_type   = "promotional_event"
  card_count    = 3~4
  card_texts    = [
    { title: title, body: subtitle },
    { title: "이런 분께 추천", body: target_personas 풀이 },
    { title: "이벤트 안내", body: "{starts_at} ~ {ends_at}" },
    { title: cta_label, body: "Dewy 앱에서 → {cta_path}" }
  ]
  caption       = "{본문}\n\n{cta_label} → Dewy 앱"
  scheduled_for = starts_at (이벤트 시작일 0시)
  hashtags      = ["Dewy", "결혼준비", "이벤트"]
```

### 6) places (웨딩홀·스튜디오·드레스샵 등)

```
입력 row: { name, category, region, description, thumbnail_url(자체), avg_rating }

→ 카드 변환:
  topic         = "{region} {category 한글명} 추천 TOP N"  (여러 places 묶음)
  source_type   = "place"
  source_id     = 대표 row 1개 (실제로는 N개 묶음)
  card_count    = N+2 (표지 + 각 장소 1장 + 마무리)
  card_texts    = [ 표지, 장소1, 장소2, ..., 마무리 ]

⚠️ 특정 업체 단정 추천 금지 — "이런 옵션이 있어요" 톤
⚠️ 광고가 아니라 정보 큐레이션 → 광고 표기 X (단, partner 인 경우는 partner_deals 로)
```

### 7) season (시즌 자동 발굴 — DB 무소스)

```
입력: 현재 월 + 09_시즌_발행캘린더.txt 의 매핑

→ 카드 변환:
  topic         = "{월} {시즌 키워드} 가이드"  (예: "5월 신부 옥외 본식 가이드")
  source_type   = "season"
  source_id     = NULL
  card_count    = 5~7
  card_texts    = [ Gemini 가 시즌 캘린더 + 03~08 지식 베이스에서 생성 ]
```

## ■ Edge Function 자동 발굴 흐름 (4단계 작업 시 구현)

```
매일 09:00 (pg_cron) → instagram-draft-generator edge function:

  1. 다음 7일 안의 promotional_events 가 있나? → 있으면 우선 (캠페인 카드)
  2. partner_deals 중 valid_until D-7 이 있나? → 있으면 (마감 임박 광고)
  3. tips_blogs / tips_instagrams / tips_videos 중 최근 7일 추가됨? → 있으면 (큐레이션)
  4. 위 셋 다 없으면 → season 모드 (현재 월의 시즌 캘린더에서 1주제 픽)
  5. 선정된 소스로 위 매핑 규칙 적용 → instagram_post_drafts INSERT (status='draft')
  6. AI 카피 생성 (Gemini) → caption, hashtags, card_texts 채움
  7. 어드민에게 알림 (선택)
```

## ■ 어드민 페이지 표시

`/admin/instagram-posts` 목록에서 `source_type` 별 색상·아이콘 추천:

| source_type | 아이콘 | 색상 토큰 |
|---|---|---|
| manual | `Pencil` | 회색 |
| tip_blog | `FileText` | 파란 |
| tip_instagram | `Instagram` | 분홍 |
| tip_video | `Video` | 빨강 |
| partner_deal | `Tag` | 노랑 (광고 표시) |
| promotional_event | `Megaphone` | 보라 |
| place | `MapPin` | 초록 |
| season | `Sun` | 주황 |

→ 어드민이 한눈에 "이게 광고인지 / 외부 인용인지 / 자체 콘텐츠인지" 파악 가능.

## ■ Claude Code 활용 메모

- 새 draft 생성 시 source_type 에 맞는 변환 규칙을 자동 적용
- partner_deal · tip_* 은 광고 / 출처 표기 자동 삽입 (11번 문서 룰)
- card_texts JSONB 스키마는 `{ title?, body?, footer? }` 배열로 통일
- 매핑 결과를 Satori 카드 렌더러에 그대로 넘김 (2단계 작업 시)
